## 나는 누구인가

```
아직 작성 전입니다.
Part 05에서 인터뷰를 통해 나만의 CLAUDE.md를 만들게 됩니다.

지금은 기본 설정만 되어 있습니다:
- 한글로 답변
- 쉬운 말로 설명
- 시각적 결과물 우선
```

## 기본 규칙

- 모든 답변은 한글로 합니다
- 전문 용어는 쉬운 말로 풀어서 설명합니다
- 결과물은 눈에 보이는 형태(파일, 표, 리스트)로 만듭니다
- 코드보다는 결과물에 집중합니다
- 사용자에게 입력 가이드 줄 때는 "~해줘" 금지, **"~하려는데 어떻게 해?"** 패턴

## 플러그인·스킬 운영 규칙 ★

> 마켓플레이스 플러그인(skillers-suda·deep-research 등)은 좋지만 **내 워크스페이스 규칙을 모른다.**
> 아래 규칙으로 덧씌워, 반복작업 때마다 손보는 수고를 없앤다. (플러그인 파일은 업데이트 때 날아가므로 직접 고치지 않는다.)

### deep-research

- **출력 위치**: 결과는 현재 폴더가 아니라 **`50-my-work/Part{NN}-.../실습{NN}-.../`** 아래로 저장한다. 실습 맥락이 아니면 `50-my-work/_research/{주제}_{날짜}/`.
- **인터뷰 건너뛰기**: 반복하는 리서치는 매번 4문항 인터뷰를 받지 말고 **쿼리 프로파일**을 쓴다 — `30-templates/research-profiles/`에 `시장동향·기술비교·이슈추적` 3종. 자리표시자(TOPIC 등)만 바꿔 넣으면 Phase 1을 건너뛴다.
- **인용 검증**: Phase 6(품질 확인)에서 "링크 다 살아있다"고 말로만 넘기지 말고 **실측**한다 —
  `python3 20-references/tools/verify_citations.py <보고서폴더> --sources <sources.jsonl>`. 깨진 링크·고아 출처·미등록 인용을 잡는다.

### skillers-suda

- **이중 저장**: 스킬을 만들면 활성 위치 **`.claude/skills/{이름}/`** + 포트폴리오 사본 **`50-my-work/실습NN-.../skill-source/`** 양쪽에 둔다(강의 규칙).
- **능력 매니페스트**: 새 스킬을 만들면 `20-references/skill-capabilities.json`에 한 줄 추가한다 — 능력 플래그(external_send/billing/delete/pii/scraping/network_read)와 auto_invoke. **위험 능력(외부발송·과금·삭제)이 있으면 자동발동을 끈다.**
- **라이트/풀 모드**: 시작할 때 먼저 물어본다 — **라이트**(4관점 분석 + 파일 생성, eval 생략)냐 **풀**(eval 서브에이전트 + description 최적화 루프까지). 풀은 비싸므로 큰 개편일 때만.
- **기존 스킬 점검**: 이미 만든 스킬을 손볼 땐 무거운 풀 파이프라인 대신 **`/skill-checkup [스킬]`**(가벼운 4관점 정적 점검)을 쓴다.

### 자식 워크스페이스를 찍을 때 (Part 07 빌더)

- 마켓플레이스 플러그인은 복사 없이 **전역 상속**되어 능력 게이트를 우회한다. 자식 워크스페이스 로드아웃에 없는 플러그인(특히 nopal·git-teacher처럼 외부발송 능력)은 자식의 `.claude/settings.json` permissions **deny로 스탬프**한다. (설계도 v3.2 참조)

## 폴더 구조

10단위 넘버링 — 직관적이고 일관됨.

| 번호 | 폴더 | 용도 |
|---|---|---|
| `00-system` | 시스템 규칙 | 강의 워크플로우, 답변 규칙 (한국어 답변 등) |
| `10-curriculum` | 강의 자료 | 커리큘럼 PDF, Part별 가이드, 보충 자료 |
| `20-references` | 참고 예시 | CLAUDE.md / 커맨드 / 스킬 / Hook / MCP / 에이전트 예시 모음 |
| `30-templates` | 템플릿 | 결과물 양식 (보고서·카드뉴스·PRD 템플릿) |
| `40-mock-data` | 실습 데이터 | CSV / XLSX 샘플 (Part 03 데이터 분석 등) |
| `50-my-work` | **내 결과물 ★** | 모든 실습 결과는 여기에 저장됩니다 |
| `90-archive` | 아카이브 | 옛 버전 / 백업 |

## 결과물 저장 규칙 ★

**모든 실습 결과는 `50-my-work/Part{NN}-제목/실습{NN}-제목/`에 저장합니다.**

```
50-my-work/
├── Part02-시작하기/
│   ├── 실습03-첫실행-폴더정리/
│   │   ├── README.md          ← 무엇을 만들었는지 + 날짜·모드·모델
│   │   └── (결과물 파일들)
│   └── 실습04-모드소개-Alias/
├── Part03-체험하기/
│   ├── 실습05-데이터분석/
│   ├── 실습06-보고서작성/
│   └── ...
```

**스킬이 자동 처리**:
- 실습 시작 시 해당 폴더 자동 생성 (없으면 mkdir)
- 결과물 파일 + README.md (메타 정보) 함께 저장
- 진도 추적: `progress.json`의 `practice_completed`에 추가

**커스텀 자산** (Part 04~06에서 만드는 것들):
| 자산 | 활성화 위치 (Claude Code 인식) | 포트폴리오 사본 |
|---|---|---|
| 커스텀 스킬 | `.claude/skills/{이름}/` | `50-my-work/실습NN-.../skill-source/` |
| 커스텀 커맨드 | `.claude/commands/{이름}.md` | `50-my-work/실습NN-.../command-source/` |
| 커스텀 hook | `.claude/hooks/` | `50-my-work/실습NN-.../hook-source/` |

---

## 진도 트래킹

`progress.json`이 학생 진행 상태를 자동 기록합니다:
- `completed_parts`: 완료한 Part들
- `practice_completed`: 완료한 실습 번호들 (1~40)
- `skills_created`, `commands_created`, `hooks_created`: 만든 자산
- `level`: AI Starter (현재) → AI Intermediate → AI Advanced → AI Native
- `last_activity`: 마지막 활동 시각

## 강의 가이드 스킬 사용법

각 Part마다 가이드 스킬이 있습니다. 클로드코드에서 호출:

```
/part02     ← Part 02 시작하기 (실습 1-4)
/part03     ← Part 03 체험하기 (실습 5-13) — 준비 중
/part04     ← Part 04 강화하기 (실습 14-17) — 준비 중
...
```

스킬이 단계별로 안내해줍니다. 모르는 부분은 스킬 안에서 직접 질문하세요.
