## A. 즉시 수정 (설계 결함 — 반영 필수) ★P0

1. **자막은 ASS(libass) 단일 경로** — 이 ffmpeg static build에 drawtext 필터 자체가 없음(실측). ASS + Malgun Gothic 한글 렌더링 정상 실측. Alignment=2, MarginV=250(하단 세이프존), PlayResX/Y=출력 해상도 일치. [P1]
2. **발화시간 공식 교체** — 단일 상수(5.5 또는 3.7음절/초) 폐기. 검수자 5문장 실측 공식 채택:
   `예상 발화초 = 음절수 ÷ 6.0 + 1.1초(edge-tts 고정 묵음) + 0.25초 × 쉼표 수` (오차 <0.2초) [W1·W5·P3 수정판]
3. **장면별 mp3 뒤묵음 트림** — edge-tts가 파일마다 뒤 0.9초 묵음 부착 → 6장면이면 5.4초 죽은 공기. build_reel.py에 트림 단계 필수. [검수자 발굴 버그]
4. **오디오 리샘플링** — edge-tts 출력은 24kHz 모노(실측). `-ar 44100 -ac 2` 명시 필요. [P2]
5. **클립 정규화 체인** — concat demuxer 대신 filter_complex(scale+crop+setsar+fps)+concat. 해상도·fps 제각각인 한글/공백 파일명 클립 실측 통과, rename 불필요. [P4]

## B. 구조 추가 (스킬의 핵심 가치) ★P1

6. **Step 3 이원 모드** [G4+G5 축소안]:
   - **빠른 모드(기본)**: `data/trend-cache-{YYYYMM}.md`(TTL 14일) + 즉석 WebSearch, 에이전트 스폰 없음, 메인 스레드가 100점 루브릭 단일 패스 채점. 일상 대본 = 초 단위.
   - **딥 모드**: deep-research 재호출 + 검증. 발동 = 캐시 만료 / 사용자가 "제대로 조사" / 신규 테마 / 직전 2릴스 viral ratio<3x.
   - 검증관 3인 스폰은 딥 모드에서도 기본 아님 — (a)상위 2후보 점수차 ≤5점 또는 (b)트렌드vs계정적합 하드 충돌일 때만.
   - 루브릭(100점): 송신 유발력 40 / 계정 적합성 25 / 차별각 20 / 제작 실현성 15. ≥70 통과, 55~69 수정 재채점, <55 폐기.
   - deep-research 쿼리에 "DM 공유 유발(관계형) 소재 우선 + 가성비 직장인 필터"를 박음. 실패 시 폴백: WebSearch 직접 조사로 강등 + 사용자 고지.
7. **Step 6 '성과 회수' 신설** [G7] — 다음 실행 시 fetch_profile 재호출 → 신규 릴스 viral ratio·추정 신호를 프로파일 성과표에 append → "송신형 우위 가설(현재 n=1, 신뢰도 중간): 지지/반증 카운트" 갱신. 릴스 3개 쌓이면 쿼터·배점 재조정.
8. **프로파일에 승리 신호 지문 + 훅 라이브러리** [G1·G2] — 성과표를 조회수가 아니라 `viral ratio + 추정 신호(송신/저장/무신호)`로 기록 (홍콩 모녀 45x=송신 > 디즈니 4.1x=저장 > 캐리어 0.31x=무신호). 실측 릴스의 첫 문장을 승리 훅/사망 훅으로 분리 저장.

## C. 품질 규칙 (Step 2·4 명문화) ★P2

9. **타이밍표 컬럼 확장** [W1+W2+W3+G6 통합]: `장면 | 타임코드 | 나레이션 | 발음표기 | 음절수 | 예상발화(공식) | 장면지시문 | 자막 | 트리거 신호`
10. **TTS 작문 규칙** [W2·W6]: 숫자·외래어 한글 풀어쓰기("만사천오백구십 번", TTS 입력엔 한글만), 이모지 제거, 호흡 5~7음절 쉼표, 문장부호 생략 금지.
11. **분량·자막 상수** [W5 수정]: 총 분량은 공식 역산(30초 릴스 ≈ 나레이션 140~160음절, 묵음·쉼표 감안). 훅 8~10음절. 자막 줄당 12자·2줄·1.5초+, 나레이션 축약형(받아쓰기 금지).
12. **어미 분포표** [W4]: 반말 나레이션 어미는 실측 샘플 없음 → "미확인" 라벨 + 첫 대본 생성 때 사용자 확인 강제. 캡션 해요체("~더라구요")는 실측 반영.
13. **리텐션 6장치 체크리스트** [G6]: 송신 훅(수신자 호명) / 3초 오픈루프 / 3~5초 패턴 인터럽트 / 저장 미끼 프레임 1장 / 루프 클로징 / 송신 CTA("같이 갈 사람 태그")+댓글 질문. CTA는 본문 연결 구체 질문형만 [W7].
14. **주제 쿼터는 소프트로** [G3 수정]: "송신형 ≥1 권장" — 하드 쿼터는 G7 데이터 3건 쌓인 뒤 격상 검토.
15. **표절 가드레일** [G8]: 후보 카드에 `트렌드 소스 / 이 계정만의 각 / 예상 송신 트리거` 3필드 강제.

## D. 엣지 케이스 보강 (검수자 발굴 갭)

16. **프로파일 자동 만료**: 생성일 기록, 30일 경과 시 자동 재수집 제안 (사용자 기억 의존 제거).
17. **API 차단 폴백의 한계 고지**: 수동 캡션 붙여넣기 폴백엔 조회수가 없음 → 그로스 분석(신호 지문) 비활성 모드로 강등 + 사용자 고지. HTTP 상태코드 분기로 차단 감지 [P6].
18. **클립 0개/과다 대응**: 0개 → 스크립트+촬영 리스트(장면 지시문 기반)만 출력하는 "촬영 가이드 모드". 과다(30개+) → 장면 지시문 기준 1차 자동 선별 후 사용자 확인.
