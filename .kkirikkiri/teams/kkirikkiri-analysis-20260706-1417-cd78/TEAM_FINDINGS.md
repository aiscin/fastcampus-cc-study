**알고리즘 교차검증 출처:**

- sends per reach는 좋아요보다 신규 도달에 3-5배 가치. 릴스 조회의 대부분이 비팔로워 → 소형 계정에 유리. (influencermarketinghub.com/instagram-sends-per-reach-playbook, dataslayer.ai)

### P4 — 클립 정규화 체인 + 한글/공백 파일명 처리 (실측 완료, 정상 동작)
- **대상**: Step5 클립 조립 파이프라인
- **무엇을**: concat demuxer(텍스트 리스트 파일) 대신 **filter_complex 기반 정규화+concat**을 기본 경로로 채택
- **왜**: concat demuxer는 리스트 파일에 파일명을 적어야 해 한글·공백·괄호 섞이면 이스케이프가 까다로움. filter_complex는 `-i` 인자로 직접 경로를 넘기므로 셸 레벨 따옴표 처리만 하면 되고, 해상도/fps가 다른 클립도 한 번에 정규화 가능
- **어떻게** (실측 성공, 명령 그대로):
  ```
  ffmpeg -i "클립 1 (가로).mp4" -i "클립 2 세로.mp4" -filter_complex \
  "[0:v]scale=1080:1920:force_original_aspect_ratio=increase,crop=1080:1920,setsar=1,fps=30[v0];\
   [1:v]scale=1080:1920:force_original_aspect_ratio=increase,crop=1080:1920,setsar=1,fps=30[v1];\
   [v0][v1]concat=n=2:v=1:a=0[vout]" -map "[vout]" -pix_fmt yuv420p -c:v libx264 out.mp4
  ```
  실측: 640x360(24fps)와 1080x1920(25fps) 클립을 한글/공백/괄호 파일명(`클립 1 (가로).mp4`, `클립 2 세로.mp4`)으로 실행 → 정상 종료, `ffprobe` 결과 1080x1920/30fps/4.0초(2+2초) 확인. **파일명 사전 rename 불필요**(도메인 카드의 실패 패턴 예상과 달리, filter_complex 경로에선 문제 없었음 — concat demuxer 텍스트 리스트 경로를 쓸 경우에만 rename이 필요할 것으로 추정, 그 경로는 미실측).
  `force_original_aspect_ratio=increase,crop`은 "커버" 방식(꽉 채우고 잘라냄)이라 방향이 섞인 클립도 여백 없이 처리되지만, 클립 가장자리가 잘려나갈 수 있음(트레이드오프, 사용자에게 명시 필요).

4. **deep-research 재호출 성공 가정** — G4·G5가 딥모드에서 deep-research가 쓸만한 구조화 출력 반환을 전제. 실패/타임아웃/빈약결과 시 딥모드 폴백 경로 없음(빠른모드로 강등하는 규칙 미정의).
5. **web_profile_info가 조회수를 계속 반환** — 그로스 분석 전체(viral ratio)가 아이템별 조회수 존재에 의존. DESIGN은 "확인됨"이라지만, API 차단 폴백(캡션 수동붙여넣기)엔 조회수가 없어 → 차단 시 G1 신호분석 붕괴(엣지 매트릭스 참조).
6. **심판이 3점수를 "합산"(G5)** — 분담채점을 합산하는 건 랭킹엔 되지만, "독립 의견 수렴"이 아님. 설계 문구("블라인드 독립 3명")와 실제 메커니즘(분업) 불일치 — 문서 정합성 결함.
