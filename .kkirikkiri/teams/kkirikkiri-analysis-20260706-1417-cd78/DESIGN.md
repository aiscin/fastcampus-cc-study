## 설계안 본문 — 5단계 워크플로우

### Step 1: 계정 데이터 수집 (script)
- `scripts/fetch_profile.py`가 인스타 웹 프로필 API로 캡션·조회수·게시물유형 수집
- 프로파일 문서가 이미 있으면 스킵, 사용자가 "프로파일 갱신"이라 하면 재수집
- API 차단 시 fallback: 사용자가 최근 캡션 9~12개 수동 붙여넣기

### Step 2: 톤앤매너 프로파일 생성 (prompt → generate)
- `data/tone-profile.md` 생성·저장 (1회 생성, 이후 모든 단계가 참조)
- 내용: 나레이션·자막=반말 / 캡션=해요체 구분, 이모지·해시태그 패턴, 훅 패턴, 성과 상위/하위 주제 분석

### Step 3: 주제 선정 (api_mcp → review)
- 내 성과 패턴 + 여행 트렌드 검색 결합 → 후보 3~5개 (예상 훅 문장 포함) → 사용자 선택
- **[사용자 추가 요구 — 구체화 필요]**: 트렌드 검색을 deep-research 스킬 재호출로 수행하고,
  전문가 에이전트 팀이 후보를 검증한 뒤에 스크립트 생성으로 넘어가는 절차를 워크플로우에 포함하고 싶다.
  → 어느 지점에서 deep-research를 호출하고, 검증 에이전트는 몇 명·어떤 관점·어떤 기준으로 걸러야 하는지 구체안 필요

### Step 4: 릴스 나레이션 스크립트 생성 (generate)
- 훅(첫 1.5초)–본문–CTA 구조, 반말 나레이션
- 장면별 자막 타이밍표 + 해요체 캡션 + 해시태그
- md 파일로 `50-my-work/reels/{날짜-주제}/`에 저장

### Step 5: 영상 조립 (script → review)
- 사용자가 촬영 클립 파일 제공 → 장면-클립 매핑 확인
- `scripts/build_reel.py`: edge-tts 나레이션 mp3 생성 + ffmpeg 컷 조립 + 자막 번인(맑은 고딕) + 오디오 믹스 → 1080x1920 세로 mp4
- 클립 개수·길이가 스크립트와 안 맞으면 매핑 단계에서 조정 제안

### 진입점 유연성
- "주제 추천해줘" → Step 3부터 / "○○ 대본 써줘" → Step 4부터 / "클립 편집해줘" → Step 5부터
- 프로파일(data/tone-profile.md)은 어느 진입점에서든 자동 참조

### 설정 기본값
- 계정: travel_yonori / TTS: 선히 / 저장: 50-my-work/reels/{날짜-주제}/ / 릴스 길이: 30~60초
