## 3. 팀장 파고들기 4문항 답변

**Q1. 위임(표준 10) — "증거 있을 때만"이 지켜지나? 축 ⑤ 비었나?**
- 원칙은 정확히 인용(87). 그러나 빌더 자신의 위임 설계(카드 게이트·살 4종·출력 계약)는 통째로 비어있음.
- 판정: 표준 10 §2 "카드 0종 기본값"에는 부합 → **원칙 위반 아님**. 다만 "닥터 4종"(31)이 서브에이전트 카드인지 스킬인지 미결정 상태가 진짜 갭. 대량 탐색(수백 파일 대조 = 표준 10 §1-1 컨텍스트 격리)은 harness-audit 닥터가 서브에이전트여야 정당한데, 설계도는 이를 스킬(121)로 뭉뚱그림.

**Q2. 검증(축 ⑦) — 점검 기준·통과선·재점검 루프 실재? 결정론 채점 가능?**
- **불가.** /audit는 "의도↔결과 갭 점검"(100)이라는 문구와 "내비 재안내" 비유뿐. 표준 09의 grep 판정·P0/P1/P2 항목·통과선·회귀가 하나도 정의 안 됨. 현재는 LLM 감상 채점 형태 → 표준 09 §1 위반 소지. **이름만 있다.**

**Q3. 모순 — "안전장치 처음엔 최소"(99) vs "지시+강제+스킬 한 번에"(121)?**
- **표면적 모순 존재.** 121 "한 번에"는 하네스(L5)를 처음부터 full로 까는 것으로 읽혀 strength ladder(약→강·증거만큼, 표준 01 §3)와 99에 충돌.
- 완화 요인: 실제 강제 항목은 2개(40-output 밖 차단·PII, 33-36)로 최소이고 PII는 표준상 "처음부터" 허용. 즉 실질 위반은 경미하나 **문구 정합은 필요** — "강제는 아키타입별 비가역 자원 기본값만, 나머지는 사고 시 승격"으로 121을 재서술하면 해소.

**Q4. 과설계 — 카탈로그 10종·6칸·2단·5단계, Clip 1 "설계도 한 장"치고 과한가?**
- 6칸 폴더·5단계 레이어 매칭·2단 노출은 **알맹이**(결정 엔진·라우팅 핵심) — 과설계 아님.
- 카탈로그 10종은 "골격만 정의, 템플릿은 Clip 2"(167)라 상시 컨텍스트 비용은 아님. 그러나 **Clip 2에서 10종 전부 + 닥터 4종 + /audit + /new-workspace 구현은 한 클립 범위 초과** → 표준 01 §4 "선제 과설치" 위험. 이미 "핵심 6종 / 선택 3 / 메타 1"(169·180·189) 우선순위가 있어 완화 — Clip 2 MVP는 6종 中 1-2개 + 닥터로 좁혀야 함.

## 6. 2라운드 재심 — L5 강제 재점수 (guard-doc·breach-doc 반증 반영)

세 반증(경계 모순 / 판정 로직 공백 = 강제 아님 / 스킬 자동발동 무가드)을 설계도 줄번호로 재검증한 결과 **L5=88은 유지 불가**.

### 무엇이 무너졌나 (근거)
- **경계 모순 (치명)**: 강제 #1 = "40-output/ 밖 수정 차단"(34) ↔ "실제 워크스페이스는 40-output 밖 독립 경로에 생성"(120). 빌더의 핵심 동작(찍어내기)이 방어선 밖에서 일어남. 정직한 훅 → 핵심기능 사망 / 예외 허용 → 임의 경로 덮어쓰기. **자기모순 가드.**
- **판정 로직 공백 = 지시로 격하 (표준 05·표준 09 §6)**: 34-36은 "무엇을 막나"만 있고 "어떻게(경로 화이트리스트·훅 이벤트·exit코드)"가 없음. rm-to-trash(141)는 Bash 전용 → Edit/Write로 밖을 고치면 발화조차 안 함. 선언은 "강제 층"(33·121)이나 산출물은 지시뿐 → **하네스 계약 불일치**.
- **내 1라운드 오류(자기검증)**: L5=88 근거 중 "되돌림 X 판정 로직(92-94)"을 L5 강제 강점으로 계상한 것은 **카테고리 오류** — 92-94는 "무엇을 강제할지 결정하는 L2 사고틀"이지 L5 집행이 아니다. 분리하면 실제 집행 내용은 [모순 가드 1 + 무기전 PII 가드 1 + 스킬복사 무가드]로 얇다.
- **스킬 자동발동 무가드 (breach)**: 스킬 복사(122) + description 유지로 자동발동(139) → 능력(외부발송·과금·PII)은 오는데 가드는 안 옴. 인터뷰 되묻기(124)는 "빈칸"에만 걸려 꽉 채운 위험 요청서는 무사통과.

### 살아남는 강점 (하한을 42보다 위로 올리는 요소)
- PII/시크릿 **처음부터 차단** 자세(36) = 표준 01:70 준수 — 올바른 집행 *결정* (단 기전 미명시).
- 훅 순서(약→강, 98) 원칙 건재.

### 재점수: **L5 = 46** (기존 88 → 46, guard-doc 42에 근접)
guard-doc 42 대비 +4는 "처음부터 PII" 자세와 훅 순서 원칙이 *결정*으로는 옳기 때문. 그 이상 못 주는 이유: 2개 명시 가드 중 1개(경계)가 자기모순, 나머지 1개(PII)도 무기전, 스킬복사 가드 미동반.

### P0냐 P1이냐 — 등급 가르는 판정 (팀장 질문 직답)
**P1 실패로 분류한다. P0 낙제 아님.** 결정론적 근거:
- 표준 09 §2 P0 예시는 *존재* 검사(instruction surface 존재 · .gitignore .env 패턴 · 정체성 섹션). 설계도는 PII 차단 *의도*를 line 36에 **명시** → 존재검사 통과.
- "판정 로직 없어 지시로 격하 + 경계 모순"은 표준 09 §6의 **하네스 계약 불일치(강도 선언 ≠ 산출물)** — *일관성* 결함이지 *부재* 결함 아님. 재범위지정("output_root ∪ target_workspace_path 밖 차단")으로 수리 가능 → P1 심각도.
- **경고**: Clip 2가 이 설계 그대로 빌드하면 임의 경로(~/.claude/·남의 워크스페이스) 덮어쓰기 = **런타임에선 P0급 비가역 사고**. 즉 *청사진 고도 = P1, 빌드 고도 = P0 위험*. 등급은 청사진 채점이므로 P1로 계산하되 이 승격 경로를 헤드라인으로.
