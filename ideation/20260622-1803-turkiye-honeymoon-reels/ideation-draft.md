# 아이데이션 작업본 — 튀르키예 신혼여행 여행 릴스 스크립트 스킬

생성: 2026-06-22 18:03 · ideation-funnel

## 1. 원문 (사용자 한 줄)
"튀르키예 신혼여행에 대한 여행 릴스 스크립트 작성 관련해서 스킬을 생성하고 싶다. 어떻게 하면 좋을지 구상하고 싶다."

## 2. 좁힌 의도 (Step 1)
- **무엇**: 튀르키예 신혼여행 영상 소재로 인스타그램 릴스(9:16, 15~30초) 스크립트를 뽑아주는 스킬
- **플랫폼**: 인스타그램 릴스
- **톤**: 정보·꿀팁 중심 (가볼 곳·맛집·동선 등 실용 정보를 짧게)
- **왜**: 신혼여행 영상/사진은 쌓이는데, 릴스용 대본 짜기가 막막해서 자동화하고 싶음
- **누구**: 본인(신혼여행 브이로그 올리는 개인)

## 3. 레퍼런스 스캔 (Step 2)
- 릴스 스크립트 작성 스킬: **없음** → 새로 만들 가치 있음
- 재활용 후보: `deep-research` — "정보·꿀팁" 소재(맛집·동선·명소)를 리서치하는 부분에서 호출 가능
- 결론: **새로 만들되, 정보 수집은 deep-research를 끼워 차별화**
- 외부 딥리서치(잘되는 정보형 릴스 구성, 등급 B 중심) → 훅 2~3초·길이 15~30초·점프컷 3~5초·자막 중앙48~60px/씬당2~4초·해시태그 3~5개. 원자료: `resource/` 참조

## 4. 후보 (Step 3)
1. **릴스 대본 생성기** [간단] — 장소/주제 입력 → 훅+자막+해시태그가 든 9:16 대본 1편. 결과물: 텍스트 대본
2. **리서치 결합형 대본기** [보통] — deep-research로 맛집·동선·꿀팁 먼저 수집 → 그걸 녹인 정보형 릴스 대본. `deep-research` 재활용
3. **시리즈 대본 패키지** [복잡] — 신혼여행 일정(여러 도시) → Day별 릴스 대본 여러 편 + 게시 순서·캡션까지 한 번에

## 5. 선택 (Step 4)
→ **② 리서치 결합형 대본기** 선택. deep-research로 정보 수집 후 정보형 릴스 대본 생성.

## 5.5 전문가 팀 검토 (Step 4.5)
전문가 4명(마케터·카피라이터·영상기획자·비평가) 병렬 검토. 원본: `resource/expert-review-2026-06-22.md`
채택: WebFetch→**캡션 붙여넣기 주입력 승격** / **표 강화**(샷타입·타임코드) / **훅 2안 선택** / **CTA 3종** / 스코프 유지+신선도 경고 방어. 보류(v2): 영상 길이별 씬 분기·기촬영 모드.

## 6. 명세 (Step 5)
- **한 줄 정의**: 튀르키예(또는 임의) 여행지 입력 → deep-research로 꿀팁·맛집·동선 수집 → 정보형 인스타 릴스(9:16, 15~30초) 대본을 씬별 표로 생성
- **입력**: 여행지/도시 + (선택) 영상에 담긴 장면·일정 메모
- **출력**: 마크다운 **씬별 표** — 컬럼 = `씬# | 화면(샷) | 자막(훅 포함) | 내레이션` + 표 아래 `해시태그`·`캡션` 블록
- **핵심 기능 (우선순위)**
  1. ★MVP **내 계정 톤앤매너 분석** — `https://www.instagram.com/travel_yonori`의 릴스를 WebFetch로 분석(훅 패턴·자막 길이·말투·이모지·해시태그 습관). WebFetch 막히면 사용자가 릴스 캡션/스크립트 붙여넣기로 폴백. 결과 = "내 톤 프로필" 요약
  2. ★MVP **잘되는 타 계정 템플릿 도입** — 정보형 여행 릴스로 잘되는 계정들의 구성 템플릿(훅 공식·씬 구성·자막 리듬)을 참고로 가져와, 내 톤 프로필과 결합한 "맞춤 대본 템플릿" 확정
  3. ★MVP deep-research 호출해 해당 여행지 정보·꿀팁 수집
  4. ★MVP 수집 정보 + 맞춤 템플릿으로 9:16 릴스 대본 생성, 씬별 표로 출력
  5. ★MVP 해시태그 + 게시 캡션 초안 생성 (내 해시태그 습관 반영)
  6. (나중) 촬영 샷리스트, 여러 편 시리즈 패키지, 톤 프로필 캐싱(매번 재분석 안 하게)
- **톤**: 정보·꿀팁 + **내 계정(@travel_yonori) 톤앤매너**에 맞춤
- **제약**:
  - 릴스 1편 길이에 맞게 자막은 씬당 짧게, 정보는 출처 불명확하면 단정 금지(deep-research 검증분만)
  - **WebFetch 리스크**: 인스타 로그인월/JS로 본문 수집 실패 가능 → "내 릴스 캡션 붙여넣기" 폴백 필수
  - 타 계정 템플릿은 **구조만 참고**(표현 그대로 베끼지 않음 — 표절·톤 붕괴 방지)
