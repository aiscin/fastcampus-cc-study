# 레퍼런스 리서치 — 잘되는 정보형 인스타 릴스 구성 (2026)

리서치: 2026-06-22 · ideation-funnel Step 2 (deep-research 방법론) · 정보 기준 시점 2026-06

## 핵심 한 줄
정보형 릴스는 **첫 2~3초 훅 → 3~5초마다 점프컷 → 씬당 짧은 자막(무음 시청 전제) → 1~3개 정밀 해시태그**가 검증된 공식이다.

## 구성 요소별 베스트프랙티스 (신뢰도 등급 포함)

| 요소 | 권장 | 신뢰도 | 출처 |
|------|------|:---:|------|
| 훅(첫 화면) | 2~3초 안에 결론·약속·시각적 패턴 인터럽트 먼저 (셋업 X) | B | invideo, prodshort |
| 길이 | **15~30초**가 완성률·발견성 sweet spot | B | invideo |
| 길이(충돌) | 30~90초가 "참여도 최고"라는 견해도 있음 ⚡ | C | prodshort/templify 요약 |
| 페이싱 | 3~5초마다 점프컷 → 약 32% 높은 참여 | C | prodshort 요약 |
| 자막 길이 | 씬당 1개 아이디어, 2~3블록으로 쪼개기 (벽글 금지) | B | opus.pro |
| 자막 타이밍 | 화면당 2~4초 노출, 말 리듬에 맞춤 | B | opus.pro |
| 자막 폰트·위치 | 48~60px, 볼드 산세리프, **중앙 1/3**(UI 겹침 회피) | B | opus.pro |
| 캡션 | 첫 100자(약 10~12단어)가 "더보기" 전 승부처. 구조 = 훅+키워드+결과 → 근거 → CTA 1개 | B | templify/truefuture 요약 |
| 해시태그 | **3~5개 정밀 태그**(누구를 위한 콘텐츠인지 AI에 알림). 범용 대량 태그 회피 | B | invideo |
| 접근성 | 다수가 무음 시청 → 자막·온스크린 텍스트가 메시지 본체 | B | invideo, opus.pro |

## ⚡ 충돌 해소
길이 15~30초(invideo) vs 30~90초(요약 매체)가 갈림. → **정보형 꿀팁 릴스는 15~30초** 채택: 짧을수록 완성률↑(알고리즘 신호)이고, 꿀팁은 압축이 미덕. 90초는 스토리·브이로그형에 적합.
