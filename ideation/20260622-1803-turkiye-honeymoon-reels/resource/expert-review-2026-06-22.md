## 팀 구성 (archetype 동적 매칭)
- 📊 숏폼 마케터 (Analyst) — 도달·참여·알고리즘
- ✍️ 여행 카피라이터 (Writer) — 자막·내레이션·캡션
- 🎬 릴스 영상 기획자 (Designer) — 씬 구성·촬영 실행
- 🔍 콘텐츠 비평가 (Critic, refute) — 약점·실패 시나리오

## 핵심 의견

| 전문가 | 한줄 |
|------|------|
| 마케터 | 계정 분석 범위 불명확 + 훅 검증 루프 부재가 핵심 리스크 |
| 카피라이터 | "정보는 맞는데 내 계정 같지 않다"가 될 위험 (감정 레이어 부재) |
| 영상기획자 | 계획용 대본 → 현장 실행 도구로 격상 필요 (기촬영 대응·샷타입) |
| 비평가 | 인스타 크롤링이라는 기술적 허구 위 4개 파이프라인 = 완성 불가 위험 |

## 합의 / 충돌
- **강한 합의**: ① WebFetch 톤분석 → 캡션 붙여넣기 ② 씬 수를 영상 길이로 분기 ③ 기촬영 영상 대응
- **충돌**: 비평가 "스코프 절반 잘라라(MVP)" vs 마케터 "deep-research 신뢰 차별점 살려라"

## 사용자 채택 결정
- ✅ **WebFetch → 캡션 붙여넣기** (주 입력으로 승격)
- ✅ **표 강화** (샷타입 약어 + 타임코드 컬럼)
- ✅ **훅 2안 생성 후 선택**
- ✅ **CTA 목적별 3종** (저장·팔로우·댓글)
- ✅ **스코프 유지 + 방어장치** (deep-research·타 계정 템플릿 살리되 정보 신선도 경고 등 방어 적용)
- ⏸ 미채택(보류): 영상 길이별 씬 수 자동분기 / 기촬영 모드 분기 → v2 후보로 기록
