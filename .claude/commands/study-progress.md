# /study-progress

강의를 들으며 만든 **실제 실습 결과물**을 직접 열어 검토하고,
어디까지 왔는지 진단하고, 다음에 뭘 하면 되는지 안내합니다.

> `/my-progress`는 기록(progress.json)만 봅니다.
> 이 커맨드는 **결과물 폴더를 직접 열어** 기록과 실제가 맞는지까지 봅니다.

## 검토 대상 (두 곳 모두 스캔)

| 워크스페이스 | 담당 Part |
|---|---|
| `50-my-work/` | Part 04~05 (실습 14~22) |
| `fastcampus-cc(part1-3)/50-my-work/` | Part 02~03 (실습 3~13) |

## 동작 (4단계, 가볍게)

### 1️⃣ 스캔
- 위 두 폴더의 `Part*/실습*/` 디렉터리를 모두 나열한다
- 각 실습 폴더 안에 **파일이 있는지 / README.md가 있는지** 확인한다

### 2️⃣ 검토 (가볍게)
각 실습을 아래 3단계 중 하나로 분류한다. 깊게 파지 말고 한 줄 코멘트만:

| 표시 | 기준 | 코멘트 |
|---|---|---|
| ✅ | README.md + 결과물 파일 있음 | 한 줄 칭찬 또는 가벼운 개선 포인트 1개 |
| ⚠️ | 폴더는 있는데 비었거나 README만 있음 | 뭘 채우면 되는지 한 줄 |
| ⬜ | 폴더 자체가 없음 (아직 안 함) | 코멘트 없음 |

### 3️⃣ 진단 (기록 vs 실제)
- `progress.json`의 `practice_completed`를 읽는다
- **기록엔 "완료"인데 폴더가 비었거나 없는 실습**을 콕 집어 알려준다 (가장 중요)
- 반대로 폴더는 있는데 기록에 빠진 것도 알려준다

### 4️⃣ README 자동 작성 (★ 보완)
검토에서 ⚠️로 분류된 실습 중 **"결과물 파일은 있는데 README.md가 없는"** 폴더를 처리한다.

1. 그런 폴더를 모아 사용자에게 알린다. 예: `"README 없는 실습: 실습10 · 실습XX — 작성해드릴까요?"`
2. 사용자가 동의하면, **각 폴더의 실제 파일을 직접 열어 보고** README.md를 작성한다.
   - 결과물 파일 이름·종류·개수를 실제로 확인한 뒤 쓴다 (아래 템플릿)
   - **있는 내용만 적는다. 지어내지 않는다.** 모르는 메타(날짜·모델 등)는 `(확인 필요)`로 남기거나 사용자에게 한 번 묻는다.
3. **결과물 파일이 0개인 빈 폴더**(예: 빈 실습 폴더)는 README를 지어내지 말 것.
   → "이 폴더는 결과물이 없어요. 뭘 만든 실습인지 알려주시면 채울게요"라고만 안내한다.

### 5️⃣ 안내
- `progress.json`의 `current_part` + 아래 커리큘럼 맵으로 **다음 실습 1개**를 정한다
- 지금 보완하면 좋은 것 **1~2개만** 콕 집어준다 (전부 나열 X)

## README 템플릿 (자동 작성용)

기존 실습 README와 같은 톤. 폴더 내용을 보고 채운다:

```markdown
# 실습NN — (제목, 폴더명 기반)

> 날짜: (알면 적고 모르면 확인 필요) · 모델: (확인 필요) · 도구: (파일 종류로 추정)

## 무엇을 만들었나

(폴더 안 결과물을 보고 1~2문장으로 요약)

## 결과물

| 파일 | 설명 |
|---|---|
| `파일명` | (무슨 파일인지 한 줄) |

## 배운 것 / 회고

(비워두고 "← 직접 한 줄 채워보세요"로 표시 — 회고는 본인 몫)
```

## 규칙

- **한글로**, 쉬운 말로. 표·체크리스트 같은 눈에 보이는 형태로.
- 가볍게 — 실습마다 한 줄 코멘트면 충분. 장문 분석 금지.
- 잔소리 금지. 잘한 건 칭찬, 개선은 "이러면 더 좋아요" 톤.
- 개선 제안은 전체에서 **1~2개만** 추려서. 모든 걸 다 고치라고 하지 않는다.
- README 자동 작성은 **실제 파일을 본 근거로만**. 회고/배운점은 본인 몫이라 비워둔다.
- 빈 폴더는 README를 지어내지 않는다 — 뭘 만든 실습인지 물어본다.
- **역할 경계**: 나는 검토·작업 담당. 동기부여·"잘하고 있나" 같은 코칭은 `practice-coach` 에이전트로 안내한다.
- 두 워크스페이스 모두 비었으면: "아직 결과물이 없네요. /part02 부터 시작해볼까요?" 안내.
