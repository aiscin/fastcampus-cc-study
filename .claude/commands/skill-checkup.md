# /skill-checkup — 스킬 가벼운 정기 점검

기존 스킬을 **읽고 분석만** 한다. skillers-suda의 풀 파이프라인(eval 서브에이전트·description 최적화 루프)은
새 스킬 제작이나 큰 개편용이라 무겁다. 이 커맨드는 **이미 만든 스킬의 유지보수**를 위한 가벼운 경로다.

## 대상 찾기

`$ARGUMENTS`가 비었으면 `.claude/skills/` 목록을 보여주고 어떤 스킬을 점검할지 물어본다.
경로면 그 SKILL.md를, 이름이면 `.claude/skills/<이름>/SKILL.md`를 Read로 읽는다.

## 4관점 정적 점검 (에이전트 스폰 없이 직접)

읽은 SKILL.md를 아래 4관점으로 각각 진단한다. skillers-suda의 4인 페르소나와 같은 축이되,
서브에이전트를 띄우지 않고 직접 분석한다(비용·속도).

| 관점 | 무엇을 보나 | 낙제 신호 |
|---|---|---|
| 🎯 기획자 | 목적이 한 문장으로 좁혀지나, "안 하는 것" 경계가 있나 | 목적이 뭉뚱그려짐, 경계 없음 |
| 👤 사용자 | description 트리거가 실제 발화를 커버하나(한/영), 자연 호출되나 | 트리거 빈약, undertrigger 위험 |
| 🔧 전문가 | 단계 타입 적절, 외부 결과에 review 단계 있나, 스크립트 경로 안전한가 | api/rag 뒤 review 없음, 하드코딩 경로 |
| 🔍 검수자 | 엣지 케이스·실패 처리, 위험 능력에 확인 장치, 본문 길이(1,500~2,000단어) | 실패 무처리, 위험 능력 무방비, 본문 과다 |

## 능력 매니페스트 대조

`20-references/skill-capabilities.json`에서 이 스킬 항목을 찾아, 실제 SKILL.md 내용과 대조한다:
- 매니페스트에 없는 스킬이면 → 능력을 판정해서 **추가 제안**(external_send/billing/delete/pii/scraping/network_read).
- `risk_capabilities`(external_send·billing·delete)가 있는데 `auto_invoke: true`면 → **자동발동 차단 제안**.
- 매니페스트와 SKILL.md 실제 능력이 어긋나면 → 불일치 지적.

## 출력

대화로 요약 + 우선순위 붙인 개선안. 형식:

```
📋 {스킬명} 점검 결과

🎯 기획자: {한 줄}  →  {발견}
👤 사용자: {한 줄}  →  {발견}
🔧 전문가: {한 줄}  →  {발견}
🔍 검수자: {한 줄}  →  {발견}

🔖 능력 매니페스트: {일치/불일치/추가 필요}

✅ 지금 고칠 것 (P0):
1. ...
🔧 여유 될 때 (P1):
1. ...
```

## 고칠지 물어보기

개선안을 보여준 뒤 AskUserQuestion으로 확인받는다 — "개선안 적용 / 일부만 / 분석만".
"적용" 선택 시에만 Edit로 SKILL.md를 수정하고, 덮어쓰기 전 변경 내용을 미리 보여준다.
큰 개편(단계 재설계·새 스크립트)이 필요하면 skillers-suda 풀 파이프라인으로 넘기라고 안내한다.
