# fashion-trend — KREAM 키워드 트렌드 리포트

> 키워드 한 마디 → 남녀 인기순 수집 → 검증된 틀로 분석 → HTML 리포트.
> 결정론 단계(수집)는 `scripts/crawl_kream.py`가, 분석·리포트는 이 문서(AI)가 맡는다.

---

## 0. 무엇을 받나 — 키워드 + 성별

사용자가 키워드를 줬으면 그대로 쓴다. 안 줬으면 기본 **'반팔티'**.
성별 범위가 모호하면 **AskUserQuestion**으로 묻는다 (질문/옵션을 텍스트로 나열하지 말고 도구 호출):

- 질문: "어느 쪽 인기순을 볼까요?" (header: "성별")
- 옵션: "둘 다 비교 (추천)" / "남성만" / "여성만"

키워드도 불명확하면 같은 호출에 "어떤 키워드로?" 질문 추가 (예시 칩: 반팔티·맨투맨·후드·청바지).

---

## 1. 수집 (스크립트 — 결정론)

`scripts/crawl_kream.py`를 Bash로 실행한다. **직접 크롤링 코드를 짜지 말 것** — 스크립트가 같은 입력에 같은 데이터를 보장한다.

```bash
python3 "{스킬경로}/scripts/crawl_kream.py" "{키워드}" --gender {male|female|both} --outdir "{저장폴더}/artifacts"
```

- `--gender both`가 기본(남녀 비교). 한쪽만 요청 시 male/female.
- `--outdir`: 결과를 둘 곳. 강의 워크스페이스면 `50-my-work/Part06-스킬만들기/실습28-웹크롤링스킬` 아래 `artifacts/`, 그 외엔 현재 작업 폴더 기준 `./fashion-trend/{키워드}/artifacts`.
- 스크립트는 stderr에 진행, stdout에 JSON(저장 경로·개수)을 출력 → 이 경로를 Read로 읽어 분석한다.
- **수집 방식**: curl_cffi safari TLS 위장 + self-referer. KREAM 봇 차단이 이 방식에 1회 성공(검증됨). 만약 실패(status≠200)하면 insane-search 플러그인으로 폴백 안내.

> 의존성(curl_cffi·beautifulsoup4)은 스크립트가 자동 설치 시도. 막히면 `pip install curl_cffi beautifulsoup4 --break-system-packages`.

## 2. 분석 (AI) — ★검증된 틀만 쓴다★

이 데이터의 **함정**을 반드시 지키며 분석한다 (전문가 토론 + 적대 검토로 확정된 원칙):

> **관심·리뷰·거래는 "상품별 누적 총량"이라 성별로 분리되지 않는다.**
> 두 파일은 같은 상품을 정렬만 다르게 본 것 → 같은 상품의 숫자는 동일하다.
> **"남녀 거래량이 같다 = 젠더리스"는 발견이 아니라 항등식.** 절대 쓰지 말 것.

### 성별 비교로 유효한 신호 = 딱 둘
| 유효 | 무효 (쓰지 말 것) |
|---|---|
| ① **순위 위치** (정렬축 = 성별 선호) | 남녀 거래량/관심량 "차이" (= 같은 상품이면 동일) |
| ② **TOP50 등장 구성(집합)** — 어떤 상품·브랜드가 뜨나 | 남녀 전환율·퍼널 비교 (지표가 성별 분리 안 됨) |

### 비교 분석 (both일 때) — 아래에서 데이터로 채운다
1. **브랜드 구성 차이** — 남녀 각 TOP 브랜드 SKU 수
2. **카테고리 성격** — 상품명 키워드 분포(예: 저지·레플리카 vs 그래픽·로고·하트)
3. **집합 분해** — 공통 / 남성전용 / 여성전용 개수 + 전용 브랜드
4. **집합 engagement 프로파일** — 남녀 TOP50 각각의 거래·관심 **평균** 비교 (집합이 다르므로 유효. "더 많이 산다" 아님 — 집합 성격 차이로 서술)
5. **순위 역전** — 공통 상품의 |남순위−여순위| 큰 것 (★거래량 아닌 *순위*로만 서술)
6. **가격 포지셔닝** — 상위권 가격 범위·평균

### 종합 분석 (남녀 합산 또는 단일)
- **성별 아키타입** (both일 때) — 한 줄 요약 (예: 남=저지 / 여=하트·로고티)
- **가격대 분포** — 볼륨 스위트스팟 구간
- **콜라보/신생 브랜드 신호** — 상품명·집합에서 관찰
- **(단일 성별·남성 데이터 한정)** 관심↔거래 격차로 "찜만 받는 화제성 상품" 식별 — ★"전환율" 금지, "관심 대비 거래 비율"로. 카운터는 독립이라 포함관계 아님.

> 계산은 Bash로 Python 인라인 집계(브랜드 Counter·집합 연산·평균·가격 bin)를 돌려 **실제 숫자 근거**를 뽑는다. 추정·기억으로 숫자를 만들지 말 것.

## 3. 리포트 (AI) — 다크 에디토리얼 HTML

`{저장폴더}/trend-report/{YYYY-MM-DD}-{키워드}-{성별}.html` 로 저장한다.

- **상단에 데이터 한계 배너 필수** — "거래 동일=발견 아님 / 성별 신호=순위+집합 / KREAM 인기순 한정, 시장 전체 아님"
- 남=쿨톤(파랑), 여=웜톤(핑크) 대비. 모바일 반응형. tabular-nums.
- 각 섹션: 제목 + 한 줄 리드 + 데이터(표·바·통계박스) + 인사이트 블록(근거 숫자 포함)
- 하단: 분석 방법 + **걸러낸 항목(전환율·퍼널 비교)** + "구매 권유 아님" 고지
- 단일 성별이면 비교 섹션 빼고 종합·내부 분석 중심으로 구성.

날짜 파일명 규칙(원본·리포트 둘 다 `{날짜}` prefix) → **주기 실행 시 트렌드 변화 추적** 가능. `/schedule`로 "매주 월요일 자동 수집"을 걸면 순위 변동이 날짜별로 쌓인다.

## 4. 발행 후

저장 경로 안내 + 핵심 인사이트 3~5줄 요약. WSL이면 `explorer.exe "{경로}"`로 여는 법 안내.
