## 핵심 패턴 — 자동 셋업 → 메타 흐름 안내 → 영상 따라하기 → WRAP

```
[자동 셋업]      스킬이 mkdir + (필요 시) Part 5 자산 활성화 확인 + 결과 한 줄 보고
   ↓
[브리핑 + 안내]  스킬 정의·메타 흐름 3단계·핵심 메시지 + 단계 흐름 (한 메시지)
   ↓
[영상 따라]      사용자가 영상 보면서 진행 (스킬 SLEEP)
   ↓ (트리거: 완료 / /wrap / 끝 / 다음 클립)
[WRAP]          파일 검증 + README 자동 + progress.json + 다음 안내
```

### 자동 셋업 의무 (모든 클립)

각 클립 reference 파일에 `## 자동 셋업` 섹션이 있다. 스킬이 클립 reference를 로드한 직후:

1. **reference의 `## 자동 셋업` Bash를 즉시 실행**
   - `mkdir -p ~/fastcampus-cc/50-my-work/Part06-스킬만들기/실습NN-제목/`
   - 만든 스킬은 CLAUDE.md 규칙대로 강의 워크스페이스 `~/fastcampus-cc/.claude/skills/{스킬명}/`(활성)에 두고, 원본 사본은 `50-my-work/Part06-스킬만들기/실습NN-제목/skill-source/`에 저장 (my-cc-study 아님)
   - 필요 시 Part 5 자산(GWS CLI · trash-guard 등) 활성화 한 줄 확인
2. **셋업 결과를 한 줄로 보고**
3. **`## 브리핑` (스킬 정의 + 메타 흐름) + `## 단계 안내` 섹션을 한 메시지로 출력** + SLEEP 모드

## 메타 흐름 — 모든 실습 공통 (Clip 2부터)

```
[1단계] 워크플로우 잡기 (방법론 조사)
  "[하려는 일 구체적으로] 만들려는데, 어떻게 워크플로우를 구성해야 할까?"
  → 클로드코드가 그 작업에 대해 직접 찾아보고 단계별로 정리해줌 (몇 단계 나올지는 조사 결과에 따라)
       ↓
[2단계] 보완 + 실행 가능 단계 정의
  정리된 단계들 같이 살펴보고 보완 → 클로드코드가 실제로 할 수 있는 단계로 다듬음
  (AI에게 맡길 단계 vs 코드로 분리할 결정론 단계 구분도 여기서)
       ↓
[3단계] 스킬로 만들기
  정의된 흐름을 그대로 SKILL.md로 정리 → 다음부터 자연어 한 마디로 호출
       ↓
[4단계] 만든 스킬로 테스트·검증
  실제 호출해서 description 매칭 + 결과 확인 + 부족한 부분 손보기
```

**핵심** — 그냥 "X 스킬 만들어줘" 하지 마세요. **"[하려는 일] 만들려는데, 어떻게 워크플로우를 구성해야 할까?"**로 출발해야 본인 한계가 박히지 않습니다. SKILL.md 작성 자체는 클로드코드가 하지만 어떻게 시작 프롬프트를 던지느냐가 결과를 가른다.
