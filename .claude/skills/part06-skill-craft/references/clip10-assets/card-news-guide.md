# 카드뉴스 제작 가이드 (도메인 레퍼런스)

> 이 문서는 **"카드뉴스를 어떻게 잘 만드나"** 만 다룬다. 스킬·파이프라인을 어떻게 구성할지는 담지 않는다 — 그건 실습에서 직접 정한다.
> 채널: 인스타그램 캐러셀(주력) + 네이버 블로그(OSMU).
> 출처: 한국어 자료 딥리서치 4종 → 4전문가(구조·카피·마케팅·현실점검) 교차검증 → 빈틈 보강. 2026-05-25.
> 모든 수치는 인스타 4:5(1080×1350) 기준. 이미지 없이 텍스트만 보고도 따라 만들 수 있게 px·pt를 명시.

---

## 1. 표준 구조 — 3구간

```
┌─ 표지 1장 ──┬─ 본문 3~10장 ──┬─ 마무리 1장 ─┐
│ 3초 후킹    │ 한 장 = 한 메시지 │ 요약 + CTA   │
└──────────┴────────────────┴────────────┘
총 장수 = 표지1 + 본문N + 마무리1. 권장 10장 내외(인스타 캐러셀 최대 20장).
```

| 구간 | 장수 | 역할 |
|---|---|---|
| 표지(Cover) | 1장 | 첫인상·후킹. 주제를 한눈에. 클릭/스와이프 유도 |
| 본문(Body) | 3~10장 | "1장 = 1 메시지". 소제목 + 내용 |
| 마무리(Closing) | 1장 | 요약 + CTA + 브랜드 노출 |

- **초반 1~2장에 후킹·핵심 집중** — 첫 장 반응이 없어도 두 번째 슬라이드가 재노출되므로 1~2장 안에 승부.
- **정보 단위 = 카드 단위.** "전세대출 6월 변경점 3가지"면 변경점 1개 = 본문 1장 → 표지1 + 본문3 + 마무리1 = 5장.

---

## 2. 제작 순서 5단계

```
① 유형 확정(§3 결정트리) → ② 프레임·톤·후킹·CTA 픽(§3 매칭표)
→ ③ 카피 초안(§3 프롬프트) → ④ 카드 잇기(§3 연결어) → ⑤ 캔버스 배치(§4 정량값)
```

## 3. 주제유형별 템플릿 (정보형 / 홍보형 / 뉴스형)

원 리서치의 가장 큰 공백이 "내 주제는 어느 틀을 쓰나"였다. 아래 결정 트리로 유형을 먼저 정하고, 매칭표에서 한 세트를 가져온다.

### ① 유형 분기 결정 트리
```
Q1. 목적이 "독자가 지금 무엇을 하게(클릭·신청·구매·저장)" 만드는 것?
   ├─ 예 → Q2   └─ 아니오(사실·지식 전달) → ▶ 정보형
Q2. 파는 게 "상품/서비스/이벤트"? 아니면 "시의성 소식·이슈"?
   ├─ 상품·서비스·이벤트 → ▶ 홍보형   └─ 새 소식·발표·이슈 → ▶ 뉴스형
```

### ② 유형 3종 완성 매칭표
| 칸 | 정보형(지식·노하우) | 홍보형(상품·전환) | 뉴스형(소식·이슈) |
|---|---|---|---|
| 카피 프레임 | 정보형(도입→정보1~N→정리) + 표지에 AIDA의 A | PAS(문제→증폭→해결) 또는 AIDA 풀세트 | 정보형 + 두괄식(결론 먼저) |
| 톤 | 절제된 친근체, 과장 금지(토스형) | 베네핏·행동촉구체, 이모지 적극 | 사실 명료체, 단정·추측 금지 |
| 표지 후킹 | 숫자/베네핏("60시간 정리한 N가지") | 타깃 호명+갭("광고비 쓰는데 문의 0건?") | 갭+시의성("6월부터 바뀌는 OO 3가지") |
| CTA | 저장("나중에 보려고? 📌") | 링크/DM("전체는 프로필 링크") | 공유/팔로우("친구 태그") |
| 총 장수 | 8~10장 | 5~7장(전환 빨리) | 6~8장 |
| 시퀀스 | 표지→(목차)→정보1~N→정리→CTA | 표지(문제)→공감·증폭→해결/상품→근거(후기)→혜택→CTA | 표지(핵심 두괄)→배경→변경점1~N→영향→CTA |
| 본문 강조 | 소제목·번호·키워드 | 베네핏·숫자를 강조색으로 점프 | 변경 전/후 대비, 날짜·수치 |
| 이미지 비중 | ≤30%(텍스트·도표) | 40%(상품·후기) | ≤35%(인포그래픽) |
| 동선 | F패턴 | Z패턴 허용·강조 점프 | F패턴+상단 헤드라인 |

> 레이아웃 정량(pt·그리드·색)은 세 유형 공통(§4). 달라지는 건 시퀀스·강조·이미지 비중·CTA뿐.

### ③ 표지 후킹 7유형 (실제 예시 문구)
결과("하루 만에 천만 원 날렸습니다") / 투자자원("60시간 들여 정리한") / 성과수치("구독자 0→1,000명") / 타깃호명("3년 차 직장인이라면") / 호기심갭("명품은 왜 비쌀까?") / 베네핏("성과 2배 12원칙") / 공감·역발상("아 맞아, 이거 하고 싶던 말!").
- 표지 제목이 성과의 **8할**. 3초 안에 사로잡아야. 카피 30자 내외, 굵고 크게.

### ④ 카드 간 전환 문구(연결어) — 스와이프 유인(체류시간 직결)
| 패턴 | 문구 | 위치 |
|---|---|---|
| 궁금증 미완결 | "그런데 진짜 문제는 따로 있어요 →" / "여기서 대부분이 실수합니다" | 정보1→2 |
| 숫자 예고 | "그 이유, 3가지로 정리했어요" | 도입→본문 |
| 반전 예고 | "그런데 반대로 하면?" / "사실은 정반대였습니다" | 전환 지점 |
| 정리 신호 | "그래서 결론은 →" / "딱 한 줄로 줄이면" | 본문→마무리 |

> 표지에만 쓰던 후킹을 본문 카드 사이에 반복 적용하는 것. 정보형이 가장 약했던 부분 = 다음 장 넘김 유인.

### ⑤ AIDA 카피 세트 예시 (홍보형 — 온라인 글쓰기 클래스)
| 단계 | 카드 | 카피 ✓ |
|---|---|---|
| A 주목 | 표지 | "하루 10분, 30일 만에 글이 팔리기 시작했습니다" |
| I 흥미 | 본문1 | "비결은 재능이 아니라 '한 문장 공식' 하나였어요" |
| D 욕구 | 본문2~3 | "수강생 1,200명이 평균 2주 만에 첫 의뢰(후기 캡처)" |
| A 행동 | 마무리 | "오늘 자정까지 50% — 프로필 링크에서 바로 시작" |
나쁜 예 ✗: A에 "글쓰기 클래스 오픈!"(베네핏 없음) / D에 "다양한 커리큘럼 제공"(추상).

### ⑥ AI에게 카피 시킬 때 프롬프트 골격
```
당신은 SNS 카드뉴스 카피라이터입니다.
- 주제:{주제} 타깃:{타깃} 유형:{정보형|홍보형|뉴스형}(트리 결과) 톤:{매칭표 톤}
- 카드 수: 표지1 + 본문{N} + 마무리1
규칙: 본문 1장=메시지 1개, 1장당 3줄·30자 내외.
출력: 카드별 [번호|소제목|본문 3줄] 표 + 표지 후킹 3개 변형(A/B용).
```
정보형: "도입→정보1~N→정리. 표지 숫자·베네핏. CTA 저장." / 홍보형(PAS): "P(고통)→A(방치 손실, 공포 금지)→S(구체 해결). CTA 링크/DM." / 뉴스형: "결론 먼저 두괄식. 사실만. 연결어. CTA 공유/팔로우."

## 4. 본문 카피 원칙 + 레이아웃 정량

### 본문 카피 (토스 6원칙 — 좋은/나쁜 예)
| 원칙 | 나쁜 예 ✗ | 좋은 예 ✓ |
|---|---|---|
| 핵심 하나만 | "소비 궁합 알아보고 지원금 받기" | "문제 10개 찍고 결과 보기" |
| 확실한 보상 | "최대 100만 원 받기" | "최소 100원 뽑기" |
| 행동 가볍게 | "3분 만에 가입하기" | "3분 만에 준비하기" |
| 정보 성격 먼저 | "10월 받을 대출 확인하기" | "이달의 대출 목록 도착 — 모아보기" |
- 제1원칙: **한 장 = 하나의 메시지.** 장점 나열 금지.

### 폰트 pt 위계 (1080×1350)
| 위계 | pt | px(=pt÷0.75) | 비고 |
|---|---|---|---|
| 표지 타이틀 | 68pt | 91px | 5~13자. 길면 60→54→48 오토핏 |
| 표지 서브 | 34pt | 45px | |
| 본문 소제목 | 62pt | 83px | |
| CTA 버튼 | 52pt | 69px | weight700, 강조색 배경+흰 글자 |
| 본문 | 46pt | 61px | 기본값 |
| 캡션 | 32pt | 43px | 하한선(이보다 작게 금지) |
폰트: Pretendard / Noto Sans KR, 최대 3종. 행간 = 폰트 × 1.618(본문 46pt → 74pt).

### 그리드 (4컬럼)
```
│←75→│220│16│220│16│220│16│220│←75→│  마진75·거터16·4컬럼220
콘텐츠 안전폭 = 1080 − 75×2 = 930px · 상150/하250 · 8px 배수 정렬
```

### 색 60-30-10
메인(배경) 60% / 보조 30% / 강조(CTA·핵심숫자) 10%↓. 10% 초과 시 강조력 소실. 한 페이지 2~3색.

### ★ 카피↔레이아웃 정합 — "30자 3줄이 930px에 몇 pt로 맞나"
전제: 안전폭 930px / 한글 1글자 ≈ 폰트px / 1줄 ≈ 10자(30÷3).
| 본문 폰트 | px | 930px 글자수 | 10자/줄? |
|---|---|---|---|
| 46pt(권장) | 61px | 15.2자 | ✅ 여유 |
| 32pt(하한) | 43px | 21.7자 | ✅ |
| 70pt(상한) | 93px | 10.0자 | ⚠ 딱 맞음(더 크면 넘침) |
결론: 본문 **46~70pt, 기본 46pt.** 세로 행간 74×3 ≈ 296px < 본문 영역 410px → 충족.

### 카드별 좌표 (1080×1350)
```
[표지]                  [본문 F패턴]            [마무리]
y150 (이미지 상단40%)    y150 소제목62pt(x75)    y150 요약46pt
y690 타이틀68pt(좌상)         본문46pt ──▶              ───
     서브34pt                본문46pt ──▶        y1000 CTA버튼(강조10%)
y1100 로고 우하          y1100 캡션32pt          y1100 로고
좌우 x:75~1005(폭930) · 텍스트 시작 x75,y150 · 이미지 ≤40%
```

## 5. 이미지 만들기 원칙

카드뉴스는 텍스트 + 이미지다. 이미지를 만들/고를 때:
- **텍스트는 이미지에 굽지 말고 별도 레이어로.** 이미지엔 배경·오브젝트만 두고, 헤드라인 자리(예: 상단 1/3)는 비워 둔다 — 수정·다국어·A/B가 코드로 가능해진다.
- **톤 일관성(Consistency)** — 전 카드가 같은 화풍·조명·색·질감을 공유해야 한 묶음으로 보인다. 카드마다 "매체·아트디렉션·무드·조명·색팔레트·질감·배경밀도·대비·여백전략·제외요소"를 똑같이 유지.
- **데이터·정보형이면 사진보다 도표·픽토그램·인포그래픽** 스타일이 어울린다. (이번 KREAM 분석 같은 주제)
- **저작권 가드** — 실존 인물·브랜드 로고·특정 작가 화풍 모방 금지. 발행 전 유사성 점검.
- 이미지 조달 폴백: 직접 생성 → 미리캔버스·캔바 → 무료소스(Unsplash·Pexels·Pixabay) → 아이콘(Flaticon·공유마당·눈누, 출처표시 의무 확인).

### 이미지 생성 프롬프트 (AI 이미지 모델용)

좋은 이미지 프롬프트는 1~3문장이면 충분하다. **순서**: 배경/장면 → 주제 → 핵심 디테일 → 구성(여백 위치) → 제약. 그리고 **STYLE_LOCK(불변값)**을 한 번 정의해 매 카드에 그대로 반복해야 톤이 흔들리지 않는다.

```
[STYLE_LOCK — 1편당 1회 정의, 매 카드에 그대로 복붙]
"Monochrome editorial illustration, soft studio lighting, fine paper texture,
 minimal background, high contrast, generous negative space.
 No readable text, no letters, no numbers, no logos, no watermark."

[카드별 프롬프트 골격]
"Draw a 4:5 vertical card-news background. {배경/장면}. {주제 오브젝트}.
 {핵심 디테일}. Keep the {상단 1/3 등} clean as empty negative space for a headline overlay.
 {STYLE_LOCK 그대로}."
```

**예시 (KREAM 반팔티 분석 — 카테고리 카드):**
> "Draw a 4:5 vertical card-news background. A clean flat-lay of two folded short-sleeve t-shirts side by side, one sporty jersey style and one minimal designer style. Soft top-down light, fine paper texture. Keep the upper third clean as empty negative space for a headline overlay. Monochrome editorial illustration, high contrast, generous negative space. No readable text, no letters, no numbers, no logos, no watermark."

- **텍스트는 이미지에 넣지 말 것**(HTML 레이어가 처리) — 꼭 필요할 때만 정확 문자열을 따옴표로 주고 "render verbatim", 폰트·크기·색·위치 지정.
- 데이터/정보형 카드는 사진풍보다 **도표·픽토그램·인포그래픽** 톤으로 STYLE_LOCK을 잡는다.
- **정본(전체 시스템 프롬프트·작성 순서·일관성 규칙)**: 같은 폴더의 `image-prompt-generator.md` 참조.

## 6. 배포 + 성과 벤치마크

| 채널 | 규격 | 분량 | 알고리즘 |
|---|---|---|---|
| 인스타 캐러셀 | 4:5 = 1080×1350, 첫 장 1:1 안전영역 | 실전 5~10장(최대 20) | 저장·공유·체류 가중, 해시태그 ≤5, 업로드 후 30분~2h 골든타임 |
| 네이버 블로그(OSMU) | 이미지 6~13장 | 본문 1500~2000자 | C-rank+D.I.A, 공유 > 공감·댓글 |

- 발행 시간(한국): 출근 07~09 / 점심 12~13 / 주중 15~18시. 요일 금>월.
- **벤치마크**: KPI 저장 > 공유 > 댓글 > 좋아요. 참여율 1~3% 양호 / 3~5% 우수(분모=도달). 캐러셀은 전 포맷 참여율 1위(평균 0.52%). **분모(팔로워 vs 도달)를 먼저 통일**하고 비교.
- OSMU: 카드 원본(4:5) → 인스타 그대로(첫 장만 1:1 안전영역) / 텍스트 풀어 네이버 블로그 본문 / 정적 카드에 모션·자막 → 릴스 9:16.
