# CLAUDE.md란?

> 한 줄 요약: AI에게 주는 나만의 업무 매뉴얼 — "이렇게 일해줘"라는 지침서

## 1. 한 줄로 말하면

CLAUDE.md는 클로드코드가 일할 때 지켜야 할 규칙을 적은 파일입니다. 
마치 신입 사원 입사 매뉴얼처럼, 회사(프로젝트)의 일하는 방식을 미리 알려주는 거죠.

## 2. 왜 필요한가

- **매번 같은 설명을 반복할 필요 없음** — 예: "우리 프로젝트는 폴더 구조가 이렇고, 파일명은 이렇게 짓는단다"
- **실수를 줄일 수 있음** — 예: "항상 테스트를 먼저 쓰고, 코드는 그 다음"
- **프로젝트 규칙을 한눈에 파악** — 새로운 작업 시작할 때 매번 설명할 필요 없음

## 3. 어디에 만드나 (위치/파일명)

| 위치 | 적용 범위 | 특징 |
|---|---|---|
| `~/.claude/CLAUDE.md` | 모든 프로젝트 | 전역 설정 (개인용) |
| `프로젝트/.claude/CLAUDE.md` | 이 프로젝트만 | 팀과 공유 가능 |

같은 이름으로 2개 파일이 있으면? **프로젝트 폴더의 파일이 우선 적용됩니다.**

## 4. 가장 단순한 예시

```markdown
# 우리 프로젝트 규칙

## 폴더 구조
- src/ → 소스 코드
- tests/ → 테스트 파일
- docs/ → 문서

## 이름 짓는 법
- 파일명: 소문자-대시로 연결 (예: user-manager.py)
- 폴더명: 소문자로 (예: api_handlers)

## 작업 순서
1. 요구사항 읽기
2. 테스트 먼저 짓기
3. 코드 작성
4. 테스트 실행해서 확인
```
