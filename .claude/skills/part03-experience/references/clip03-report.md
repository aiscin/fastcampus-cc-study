## 자동 셋업 (진행 안내 출력 직전 스킬이 즉시 Bash로 실행)

> **이 단계는 명령 안내가 아니라, 스킬이 직접 Bash 도구로 실행한다.**
> 워크스페이스가 어디에 있든(홈/Documents/Desktop 등) 자동 감지. 절대 경로 하드코딩 금지.

```bash
# 1) 워크스페이스 루트 자동 감지 — CLAUDE.md 있는 곳까지 부모로 올라감
WS="$(pwd)"
while [ "$WS" != "/" ] && [ ! -f "$WS/CLAUDE.md" ]; do
  WS="$(dirname "$WS")"
done

# 폴백: CLAUDE.md 못 찾으면 40-mock-data 있는 곳을 워크스페이스 루트로
if [ "$WS" = "/" ]; then
  WS="$(pwd)"
  while [ "$WS" != "/" ] && [ ! -d "$WS/40-mock-data" ]; do
    WS="$(dirname "$WS")"
  done
fi

if [ "$WS" = "/" ]; then
  echo "⚠ 워크스페이스 루트를 못 찾았어요. CLAUDE.md 있는 폴더에서 claude 실행해주세요."
  exit 1
fi

echo "워크스페이스: $WS"

# 2) 작업 폴더 + 이전 클립 결과물 경로 (CLAUDE.md 표준: 50-my-work/Part{NN}-제목/)
DEST="$WS/50-my-work/Part03-체험하기/실습06-보고서작성"
SRC="$WS/50-my-work/Part03-체험하기/실습05-데이터분석"
MOCK="$WS/40-mock-data"

mkdir -p "$DEST"
cd "$DEST"

# 3) Clip 2 결과물 통째 복사 (이어서 하는 작업)
if [ -d "$SRC" ]; then
  cp -r "$SRC"/* ./ 2>/dev/null
  rm -f README.md   # Clip 3 자체 README는 WRAP에서 새로 생성
fi

# 4) sample_csv 폴백 — Clip 2 안 했으면 강의 자료에서
if [ ! -d ./sample_csv ] && [ -d "$MOCK" ]; then
  mkdir -p sample_csv
  cp "$MOCK"/*.csv sample_csv/ 2>/dev/null
fi

echo "=== 결과 ==="
ls -la
```

**셋업 결과 한 줄 안내** (위 Bash 출력의 `WS` 경로를 그대로 인용):
- Clip 2 결과물 통째 복사 성공 → "✓ `{WS}/50-my-work/Part03-체험하기/실습06-보고서작성/` 준비 완료. Clip 2에서 만든 charts/ + sample_csv/ + 분석 답변 모두 가져왔어요."
- charts/ 없이 sample_csv만 (Clip 2 미실행) → "✓ 폴더 준비 완료. sample_csv는 강의 자료에서 가져왔는데 Clip 2 결과물(charts)은 없네요. STEP 4에서 차트가 필요하면 새로 만들거나 텍스트로 갑니다."
- 둘 다 없음 → "⚠ Clip 2 결과물 + 강의 자료(40-mock-data) 둘 다 못 찾았어요. CLAUDE.md 있는 폴더에서 claude 실행했는지 확인해주세요."
