## v6.0 교수법 매핑

v6.0 커리큘럼의 4단계 교수법을 3턴에 매핑합니다:

```
강의 4단계:   설명   →   프리뷰   →   직접실습   →   비교검토
               ↕           ↕            ↕              ↕
3턴 매핑:  ← Phase A →          ← Phase B →   ← Phase C →
```

## 각 블록은 반드시 3턴에 걸쳐 진행한다

```
┌─ Phase A (설명 + 프리뷰 — 강의 영상과 동기화) ─────────────────┐
│ 1. progress.json의 current_block 필드를 확인한다               │
│    → 값이 있으면 세션 재진입 처리 (아래 재진입 로직 참고)          │
│ 2. references/에서 해당 블록 파일의 EXPLAIN 섹션을 읽는다        │
│ 3. 핵심 개념을 설명한다                                         │
│ 4. 이번에 만들 결과물을 미리 안내한다 (프리뷰)                    │
│ 5. "강의 영상에서 프리뷰를 확인하세요" 안내                       │
│ 6. ⛔ 여기서 반드시 STOP. 아래 종료 문구 출력 후 턴을 종료한다.   │
│                                                               │
│ ❌ 절대 하지 않는 것: AskUserQuestion 호출                       │
│ ❌ 절대 하지 않는 것: 퀴즈 출제, QUIZ 섹션 읽기                  │
│ ❌ 절대 하지 않는 것: "해봤나요?" 질문                           │
└───────────────────────────────────────────────────────────────┘

  ⬇️ 수강생이 "완료", "다음" 등을 입력한다

┌─ Phase B (직접 실습 — 수강생이 직접 수행) ─────────────────────┐
│ 1. references/에서 해당 블록 파일의 EXECUTE 섹션을 읽는다        │
│ 2. 외부 작업 블록 여부 확인 (아래 외부 작업 처리 규칙 참고)        │
│ 3. 이전 결과물 필요 여부 확인 (아래 폴백 규칙 참고)               │
│ 4. AskUserQuestion으로 맞춤 질문 (목표, 주제 등)                │
│ 5. 답변 기반으로 실습 가이드 제공                                │
│ 6. 실습 실행 서포트                                             │
│ 7. progress.json의 current_block을 현재 블록으로 업데이트        │
│ 8. ⛔ STOP — "완료되면 '다음'이라고 입력하세요"                  │
│                                                               │
│ ❌ 절대 하지 않는 것: 퀴즈 출제                                  │
│ ❌ 절대 하지 않는 것: Phase C 내용 미리 진행                     │
└───────────────────────────────────────────────────────────────┘

  ⬇️ 수강생이 "완료", "다음" 등을 입력한다

┌─ Phase C (비교 검토 — 퀴즈 + 강사 버전 비교) ──────────────────┐
│ 1. references/에서 해당 블록 파일의 QUIZ 섹션을 읽는다           │
│ 2. AskUserQuestion으로 이해도 퀴즈 출제                         │
│ 3. 정답/오답 피드백                                             │
│ 4. 결과물 확인 + 강사 버전 비교 안내                             │
│ 5. 개선 포인트 1개 제안                                         │
│ 6. 레벨 체크 (달성 조건 충족 시 레벨업 알림)                     │
│ 7. progress.json 업데이트 (블록 완료 처리)                      │
│ 8. 다음 블록 이동 확인                                          │
└───────────────────────────────────────────────────────────────┘
```

## 핵심 금지 사항

1. **Phase A에서 AskUserQuestion을 호출하지 않는다**
2. **Phase A에서 퀴즈를 내지 않는다**
3. **한 턴에 Phase 2개 이상을 진행하지 않는다**
4. **Phase B에서 퀴즈를 내지 않는다**

## Phase A 종료 시 필수 문구

```
---
강의 영상에서 프리뷰를 먼저 확인하세요.
준비되면 "완료" 또는 "다음"이라고 입력해주세요.
```

이 문구 이후에 어떤 도구 호출이나 추가 텍스트도 출력하지 않는다.

## 세션 재진입 로직

스킬 시작 시 `progress.json`의 `current_block` 필드를 확인한다.

```
current_block 값이 있는 경우:
  → AskUserQuestion으로 안내:
    "Block N에서 멈추셨네요. 이어서 진행할까요?"
    옵션:
    - "네, Block N부터 이어서요" → 해당 블록 Phase A부터 진행
    - "아니요, 처음부터 시작할게요" → Block 0 Phase A부터 진행
    - "다른 블록으로 갈게요" → 블록 선택 메뉴 표시

current_block 값이 null인 경우:
  → 일반 시작 (Block 0 Phase A부터)
```

## 외부 작업 블록 처리 규칙

Claude가 직접 실행할 수 없는 작업이 포함된 블록의 Phase B 처리:

**외부 작업 유형:** 소프트웨어 설치, API 키 발급, 계정 생성, 브라우저 조작, 터미널 직접 실행

```
처리 순서:
1. 작업 내용을 단계별로 명확히 안내한다
2. "위 단계를 직접 수행해주세요" 안내
3. AskUserQuestion으로 완료 여부 확인:
   - "완료했어요" → 결과 확인 후 Phase B 마무리
   - "막혔어요" → 어디서 막혔는지 확인 후 해결 지원
   - "나중에 할게요" → current_block을 현재 블록으로 저장 후 STOP
```

## 이전 결과물 폴백 규칙

순차 블록 진입 시 이전 블록의 결과물이 필요한 경우:

```
확인 순서:
1. 이전 블록 결과물이 my-work/에 있는지 확인
2. 있으면: 해당 파일을 기반으로 진행
3. 없으면: mock-data/에서 대응하는 샘플 파일 자동 제공
   → "이전 실습 결과물이 없어서 샘플 데이터로 진행합니다" 안내
4. mock-data/에도 없으면: 해당 블록에서 직접 생성 후 진행
```
