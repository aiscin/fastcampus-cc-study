# 대화 패턴 3종

> Part 1에서 소개하고, 모든 Part에서 반복 적용합니다.

---

## A. 탐색형 -- "이것도 돼?"

```
목적: 가능성 확인, 정보 수집
예시:
  - "이 파일 분석해줄 수 있어?"
  - "이런 것도 가능해?"
  - "이거 어떻게 하는 거야?"

언제 쓰나: 새로운 기능을 처음 접할 때, 뭘 할 수 있는지 탐색할 때
```

## B. 구축형 -- "이렇게 만들어줘"

```
목적: 결과물 생성, 구조화
예시:
  - "주간보고서 만들어줘"
  - "이 데이터로 비교표 만들어줘"
  - "커맨드 하나 만들어줘"

언제 쓰나: 구체적인 결과물이 필요할 때, 무언가를 처음부터 만들 때
```

## C. 개선형 -- "이 부분 고쳐줘"

```
목적: 기존 결과물 수정, 품질 향상
예시:
  - "이 부분 좀 더 자연스럽게 고쳐줘"
  - "표 형식을 바꿔줘"
  - "이 스킬에 블록 하나 추가해줘"

언제 쓰나: 이미 만든 것을 발전시킬 때, 피드백 반영할 때
```

---

## 전체 흐름

```
아이디어 → 탐색(A) → 구축(B) → 피드백 → 개선(C) → 완성
```

모든 Part에서 이 흐름이 반복됩니다. 결과물만 달라질 뿐 과정은 같습니다.
