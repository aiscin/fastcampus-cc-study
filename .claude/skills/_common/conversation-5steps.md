# 대화 패턴 5단계

> v6.0 커리큘럼 기반 (2026-03-28)
> 기존 conversation-patterns.md(3종 체계)를 대체합니다.

---

## 핵심 선언

**뭘 만들든 방식은 같다.**

데이터 분석 보고서를 만들든, 포트폴리오 웹사이트를 만들든, 업무 자동화 에이전트를 만들든 — 대화 방식은 동일합니다. 결과물만 달라질 뿐, 과정은 항상 이 5단계를 따릅니다.

---

## 5단계

```
1단계: 던지기
2단계: 구체화하기
3단계: 만들기
4단계: 검증하기
5단계: 개선하기
```

---

### 1단계: 던지기 — "이런 거 하려는데"

```
핵심: "해줘"가 아니라 "하려는데 어떻게?"
목적: 의도와 방향을 상의한다. 바로 실행을 요청하지 않는다.

예시:
  "지난달 매출 데이터를 분석하려는데, 어떻게 시작하면 좋을까?"
  "포트폴리오 웹사이트 만들고 싶은데, 뭐부터 준비해야 해?"
  "매일 아침 뉴스 정리를 자동화하고 싶은데, 어떤 방식이 있어?"
```

---

### 2단계: 구체화하기 — "이런 조건으로"

```
핵심: 범위, 형식, 제약 조건을 구체화한다.
목적: AI가 원하는 것을 정확히 이해하게 만든다.

예시:
  "데이터는 CSV 파일이고, 결과는 차트로 보고 싶어. 월별 비교로."
  "1페이지짜리 심플한 디자인. 이름, 프로젝트, 연락처만 들어가면 돼."
  "매일 오전 9시에 실행되면서, 슬랙으로 결과를 보내줬으면 해."
```

---

### 3단계: 만들기 — "이걸로 만들어줘"

```
핵심: 충분히 구체화된 후에 실행을 요청한다.
목적: 결과물을 생성한다.

예시:
  "좋아, 그럼 이 CSV 파일로 월별 매출 비교 차트 만들어줘."
  "방금 얘기한 조건으로 포트폴리오 HTML 파일 만들어줘."
  "그 방식으로 뉴스 정리 자동화 스크립트 만들어줘."
```

---

### 4단계: 검증하기 — "이거 맞아?"

```
핵심: 결과물을 확인하고 수정을 요청한다.
목적: 원하는 결과와 실제 결과를 비교한다.

예시:
  "3월 데이터가 빠진 것 같은데, 다시 확인해줘."
  "색깔이 너무 진한데, 좀 더 밝게 해줄 수 있어?"
  "슬랙 메시지 형식이 마음에 안 드는데, 이렇게 바꿔줄 수 있어?"
```

---

### 5단계: 개선하기 — "여기 고쳐줘"

```
핵심: 반복적으로 개선한다. 완성은 한 번에 오지 않는다.
목적: 결과물을 계속 발전시킨다.

예시:
  "이번엔 전년도 데이터도 같이 넣어서 비교해줘."
  "모바일에서도 잘 보이게 반응형으로 바꿔줘."
  "뉴스 외에 날씨도 추가해줘."
```

---

## 흐름 정리

```
던지기(1) → 구체화하기(2) → 만들기(3) → 검증하기(4) → 개선하기(5)
                                              ↑___________________________↓
                                                  (반복)
```

완성될 때까지 4→5→4→5를 반복합니다.

---

## 바이브코딩 vs 에이전트 활용

**대화하는 방식은 동일합니다.** 다만 무엇을 만드느냐에 따라 명칭이 다릅니다:

| 구분 | 만드는 것 | 예시 |
|------|----------|------|
| **바이브코딩** | 앱, 웹페이지 등 실행 가능한 결과물 | 포트폴리오 사이트, 대시보드 앱 |
| **에이전트 활용** | 업무에 쓰는 자동화, 에이전트 | 뉴스 정리 자동화, 보고서 생성 스킬 |

어느 쪽이든 5단계 흐름은 같습니다.
