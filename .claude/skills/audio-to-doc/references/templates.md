# 유형별 출력 템플릿 — audio-to-doc

전사·화자분리 결과를 읽고 **유형을 판별**한 뒤, 해당 유형 양식으로 마크다운 문서를 만든다.
모든 문서 상단에는 공통 헤더(한줄 요약·날짜·화자/참석자·길이)를 둔다.

## 공통 헤더 (모든 유형)

```markdown
# {제목}

> **한줄 요약** — {핵심 한 문장}
> **날짜** {YYYY-MM-DD} · **유형** {판별된 유형} · **화자 {N}명** · **분량** {약 N분}
```

---

## 유형 판별 기준

| 유형 | 신호 |
|------|------|
| 회의 | 다수 화자 + "안건·결정·액션·다음 회의" 키워드 |
| 강의/컨퍼런스 | 단일(혹은 주도) 화자 + 개념 설명·예시 전개 |
| 팟캐스트/대담 | 2-3 화자 + 자유로운 주제 이동 |
| 인터뷰 | 질문자 ↔ 답변자 Q&A 반복 |
| 발표/세미나 | 단일 화자 + 주장-근거 구조 + 말미 Q&A |

판별이 애매하면(예: 발표+Q&A 혼합, 비공식 잡담) **사용자에게 판별 결과를 먼저 보여주고 확인**받은 뒤 진행한다.

---

## 1) 회의 — 안건 / 결정 / 액션

```markdown
## 안건
- {논의된 안건들}

## 결정사항
- {확정된 내용}

## 액션 아이템
- [ ] {담당자} — {할 일} (기한: {있으면})
- [ ] ...

## 논의 요약
{쟁점별 2-3줄}

## 전체 발언 (타임라인)
- [MM:SS] {화자}: {발언}
```

## 2) 강의/컨퍼런스 — 핵심 개념 + 섹션 요약

```markdown
## 핵심 개념
- **{개념}** — {한 줄 정의}

## 섹션별 요약
### [MM:SS] {소주제}
{2-3줄 요약}

## 기억할 문장
> {인상적인 인용 1-2개}
```

## 3) 팟캐스트/대담 — 주제별 구간 + 화자별 의견

```markdown
## 주제별 구간
### [MM:SS] {주제}
- {화자 A} 의견: {요약}
- {화자 B} 의견: {요약}

## 화자별 핵심 입장
- **{화자 A}** — {한 줄}
- **{화자 B}** — {한 줄}
```

## 4) 인터뷰 — Q&A + 하이라이트

```markdown
## Q&A
**Q. {질문}**
A. {답변 요약}

## 발언 하이라이트
> [MM:SS] "{인상적인 답변}"
```

## 5) 발표/세미나 — 주장-근거-Q&A

```markdown
## 핵심 주장
- {주장}

## 근거
- {주장}을 뒷받침하는 데이터·사례

## Q&A
**Q. {질문}** → A. {답변 요약}
```

---

## 부록 — 출력 규칙
- 액션 아이템은 **체크박스 `- [ ]`** 로 (Notion·Obsidian 붙여쓰기 용이).
- 화자가 1명이면 화자 분리 섹션은 생략하고 "단일 화자" 표기.
- 화자 3명 이상이면 분리 정확도가 떨어질 수 있어 문서 말미에 한 줄 고지.
- 원문 전사는 `transcript_raw.txt` 에 이미 보존됨 — 문서에는 요약 중심으로.
