## 자동 셋업

스킬 호출 직후 아래 Bash를 즉시 실행 (사용자에게 명령을 보여주는 게 아니라 스킬이 직접). **clip-03에서 만든 빈 위키에 라이브용 영화 raw를 넣어준다.**

```bash
if [ -d "$HOME/fastcampus-cc" ]; then
  ROOT="$HOME/fastcampus-cc"
elif [ -d "$HOME/Desktop/fastcampus-cc" ]; then
  ROOT="$HOME/Desktop/fastcampus-cc"
else
  ROOT="$(pwd)"
fi

# LLM Wiki = 별도 워크스페이스 (fastcampus-cc 밖)
if [ -d "$HOME/llm-wiki" ]; then WIKI="$HOME/llm-wiki"
elif [ -d "$HOME/Desktop/llm-wiki" ]; then WIKI="$HOME/Desktop/llm-wiki"
else WIKI="$HOME/llm-wiki"; fi
mkdir -p "$WIKI/raw" "$WIKI/wiki"          # clip-03 뼈대 없으면 최소 골격 보강
SEED="$ROOT/.claude/skills/part07-realworld/seed-movie-wiki/raw"   # 시드는 강의 워크스페이스의 스킬에 있음

if [ -d "$SEED" ]; then
  # 라이브 정리용 — 서로 연결되는 15편만 llm-wiki/raw/에 (전체 1,000편은 SEED에 그대로 있음)
  for id in 0001 0002 0003 0004 0026 0051 0052 0055 0081 0091 0096 0098 0102 0163 0175; do
    cp "$SEED/$id-"*.md "$WIKI/raw/" 2>/dev/null
  done
  echo "✓ 라이브용 영화 raw $(ls "$WIKI/raw"/*.md 2>/dev/null | wc -l | tr -d ' ')편을 llm-wiki/raw/에 넣었습니다 (정리는 아직)"
  echo "  전체 1,000편 raw는 $SEED 에 있음 (대량 빌드·공개용)"
else
  echo "⚠ 영화 raw 시드(seed-movie-wiki/raw)가 없습니다 — 강사 준비 필요"
fi

WORK_DIR="$ROOT/50-my-work/Part07-실전/실습35-위키짓기"
mkdir -p "$WORK_DIR"
echo "✓ $WORK_DIR 진도 폴더 준비 완료"
```

> ⚠ **강사 준비(1회)**: 대량 공개용으로 스킬의 `seed-movie-wiki/raw` 1,000편을 **미리 컴파일한 위키**(그래프 HTML[**시각화 전용 — 검색 아님**] + 대표 페이지 몇 장)를 강사 PC에 만들어 둔다. **전체 빌드 wiki는 스킬에 넣지 않는다 — raw에서 직접 짓는 게 실습의 핵심.** (clip-05 의미검색 데모용 `descriptions.jsonl`은 스킬 `seed-movie-wiki/`에 포함)

## 브리핑 (스킬이 시작 시 먼저 띄워줌)

> 학습자가 영상 보기 전에 한 번 읽고 시작.

### 오늘은 영화 자료로 손에 익힌다 (먼저 안내)

| 항목 | 내용 |
|---|---|
| 무슨 자료 | **영화 감상·인물·작품 자료**를 가져다 씁니다 (자동 셋업이 `llm-wiki/raw/`에 15편 넣어둠) |
| 왜 영화 | 누구나 아는 도메인이라 정리 결과를 **바로 알아봄**. 본인 주제는 똑같은 방식으로 나중에 적용 |
| clip-03과 관계 | clip-03에서 만든 **빈 위키 뼈대에 자료를 채우는 날**. 설계는 본인 주제로 했어도, 손에 익히는 건 영화로 |

### 오늘 도는 루프 — 4동사 중 3개

| 동사 | 하는 일 |
|---|---|
| 투입(ingest) | raw에 영화 자료 넣기 (셋업이 이미 함 — 훑어보기) |
| 정리(compile) | AI가 raw 읽고 wiki 페이지 + **frontmatter(description·출처)** + `[[링크]]` 생성 |
| 질문(query) | 위키에 묻고, **좋은 답은 다시 위키에 저장**(복리) |
| (점검은 clip-05) | 규모 커지면 깨지는 곳 고치기 |

### 라이브 + 대량 공개 (둘을 섞음)

| | 내용 |
|---|---|
| **라이브** | 영화 raw 15편을 직접 정리시켜 위키가 생기는 걸 따라치기 |
| **대량 공개** | 강사가 raw 1,000편을 미리 빌드해 둔 영화 위키를 열어 "이만큼 쌓이면 이렇게 된다" |

> 천 편을 라이브로 정리할 순 없으니, 소량은 직접 / 대량은 미리 빌드한 걸 보여준다. (둘 다 raw에서 빌드한 결과)

### 핵심 메시지

> 정리시킬 때 **각 페이지 맨 위에 한 줄 설명(description)과 출처**를 같이 남기게 한다. description은 나중에 검색의 기반(clip-05). 자료 많으면 **100편씩 배치**로.
> 질문해서 나온 좋은 답은 버리지 말고 **위키에 다시 저장** — 탐색도 자산이 된다(복리).

## 절차 — 투입·정리·질문 (영상 보면서 진행)

> **복사용 입력 안내**: 아래 "실행 입력"은 AI에게 직접 시키는 명령이라 "~해줘" 형태다. 평소 대화는 "~하려는데 어떻게?"로 시작하고(도입 멘트), AI가 알려준 방법을 실행 입력으로 따라친다 — 이 2단을 강사가 말로 짚어준다.

### Phase 1. 투입(ingest) — 영화 자료 훑어보기 (4분)

raw에 이미 들어간 영화 자료를 "정리 말고 훑어만" 보게 한다.

**실행 입력:**
```
llm-wiki/raw/ 폴더의 영화 자료들을 읽고, 어떤 주제(인물·작품·장르)가 반복되는지 먼저 훑어줘. 정리는 아직 하지 마.
```

| 확인 | AI가 반복 등장하는 인물·작품을 목록으로 보고 (예: 봉준호, 기생충…). 아직 wiki/ 비어 있음 |

### Phase 2. 정리(compile) — 위키 페이지 만들기 ★ (10분)

clip-03에서 만든 `CLAUDE.md` 규칙(description·출처·같은 이름·링크)이 매번 같게 적용된다.

**도입 멘트 ("~하려는데 어떻게?"):**
```
raw에 있는 영화 자료를 인물·작품별 위키 페이지로 정리하고 싶은데,
서로 연결도 하고 나중에 검색도 잘 되게 하려면 어떻게 하면 좋을까?
```
↓ AI가 방법을 알려주면 → 실행 입력으로 진행

**실행 입력 (라이브):**
```
raw 영화 자료를 읽고 wiki/에 주제별 페이지(인물·작품)를 만들어줘.
각 페이지 맨 위에 frontmatter로 description(본문 전체를 바탕으로 한 줄 요약)과 sources(출처)를 넣고,
관련 페이지끼리 [[ ]] 링크를 걸어줘.
```

| 확인 | `wiki/`에 인물·작품 페이지 생성 / 각 페이지 frontmatter에 `description:`·`sources:` / 본문에 `[[링크]]` |

> **description 품질 확인**: 생성된 description이 1~2문장 이상인지 한 번 훑어본다 — 너무 짧으면 clip-05 "설명으로 골라 읽기"가 부정확해진다. (검색 품질 = description 품질)

**대량 공개:** 강사가 raw 1,000편을 미리 빌드해 둔 영화 위키 열기.
> "소량은 방금 직접 만들었고, 1,000편을 다 정리하면 이렇게 인물·작품이 엮인 위키가 됩니다.
> 다만 이 정도로 커지면 '전체 읽기'가 삐걱대는 곳도 생기는데 — 그건 다음 클립에서 솔직하게 봅니다."

### Phase 3. 질문(query) → 답을 다시 쌓기 ★ (10분)

**실행 입력 (질문):**
```
이 위키에서 "봉준호 영화의 공간 연출"을 정리해줘. 근거가 된 페이지를 출처로 같이 알려줘.
```
| 확인 | 위키 페이지를 근거로 답 + 출처 |

**실행 입력 (재저장 — 복리):**
```
방금 답을 wiki/에 새 페이지로 저장하고(frontmatter description·출처 포함), 관련 인물·작품 페이지에 [[링크]] 걸어줘.
```
| 확인 | `wiki/`에 새 정리 페이지 추가 + 링크 연결. 탐색이 자산으로 쌓임 |

### Phase 4. (선택) 그래프 (4분)

**실행 입력:**
```
wiki/의 [[ ]] 링크 관계를 점·선 그래프 HTML 한 파일로 만들어줘. 브라우저에서 드래그·줌 되게.
```
> 옵시디언 없이도 클로드코드가 그려준다.

## WRAP 자동 처리

### 1. 결과물 검증

```bash
ROOT="${ROOT:-$HOME/fastcampus-cc}"
if [ -d "$HOME/llm-wiki" ]; then WIKI="$HOME/llm-wiki"
elif [ -d "$HOME/Desktop/llm-wiki" ]; then WIKI="$HOME/Desktop/llm-wiki"
else WIKI="$HOME/llm-wiki"; fi
echo "raw: $(ls "$WIKI/raw"/*.md 2>/dev/null | wc -l | tr -d ' ')편"
echo "wiki 페이지: $(ls "$WIKI/wiki"/*.md 2>/dev/null | wc -l | tr -d ' ')장"
```

wiki 페이지가 0이면 Phase 2에서 멈춘 것 — 재시도 권유. description이 0이면 규칙이 안 먹은 것 — 다시 시켜준다.

## 진행 원칙

- **자동 셋업 후 SLEEP**: 브리핑+진행 안내를 한 메시지로 출력하고 영상 따라 진행
- **영화 자료로 따라치기**: 본인 주제는 같은 방식으로 나중에 — 오늘은 영화로 손에 익힘
- **실습 진행 방식**: 강사 1줄 입력 → 학생 1줄 입력 → 결과 같이 확인. STOP 분리 X
- **"~해줘" 금지 / "~하려는데 어떻게?" 강제** (memory: feedback_ask_how_pattern) — 단 "실행 입력"은 실제 동작 프롬프트라 명령형. **이 2단(도입 멘트 → 실행 입력)을 강사가 말로 짚어준다.**
- **description·출처는 frontmatter에** — 별도 sources.md 파일 X. description은 본문 전체 기반 한 줄
- **대량은 미리 빌드 공개 / 라이브는 15편**: 천 편 라이브 금지. 빌드 결과물은 스킬에 안 넣음(raw에서 빌드)
- **출처·복리 강조**: 정리엔 description·출처, 질문 답은 다시 저장
- **자유 진행 중 개입 X**: 명시적 도움 요청에만 응답
- **AskUserQuestion 사용 X**: 회고도 자유 입력
- **IDE는 안티그래비티**: VS Code 언급 X
- **정량 수치 단정 금지**: "토큰 N배 절감" 같은 단정 X
