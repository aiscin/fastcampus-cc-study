# 08. 구축 워크플로우 — 빈 폴더에서 운영까지

> 01~07이 "갖춰야 할 것"이라면, 이 문서는 "만들어가는 순서"다.
> 핵심 설계: **대화는 LLM이, 생성은 결정론이.** 요구사항 수집은 대화로 하되,
> 파일 생성과 검증은 재현 가능한 스크립트가 한다.

## 0. 전체 흐름

```
Phase 0  현황 감사 ── 기존 워크스페이스/충돌/인접 워크스페이스 확인 → 신규/개선/검증 분기
Phase 1  인터뷰 ──── 목적·도메인·운영환경 수집 (질문 설계는 §2)
Phase 2  설계 ────── 작업 유형 결정 + 커맨드/구조/하네스 강도 설계
Phase 3  생성 ────── 결정론 생성 (스크립트 또는 체크리스트 기반 수동)
Phase 4  검증 ────── 표준 준수 채점 (→ 09)
Phase 5  인수인계 ── 시작 방법 안내 + 초기 상태 기록
Phase 6  진화 ────── 운영 피드백 → 실측 검증 → 표준 반영 → 회귀 (→ §4)
```

## 1. Phase 0 — 감사 없이 시작하지 마라

새로 만들기 전에 반드시 확인:
1. **같은 목적의 기존 워크스페이스**가 있는가 → 신규가 아니라 개선 분기
2. **인접 워크스페이스**가 있는가 → 경계 한 줄 + 단방향 참조 규칙을 설계에 포함
3. **과거 결정 기록**(변경이력, 일탈 로그)이 있는가 → 같은 실수 반복 방지

감사 결과는 사용자에게 보여주고 분기를 합의한 뒤 진행한다.

## 2. Phase 1 — 인터뷰 질문이 곧 하네스 사양서다

각 질문은 장식이 아니라 **특정 산출물을 결정**한다. 질문 ↔ 산출물 매핑:

| 질문 | 결정되는 산출물 |
|------|----------------|
| "무슨 작업을 하나요?" | 작업 유형 → 폴더/커맨드/워크플로우 원형 |
| "도메인을 한 줄로?" | L0 정체성 문장 |
| "(위임 역할별) 언제 부르나요? '~해줘' 2~3개" | 서브에이전트 카드 description — **답 못 하면 카드 생성 안 함** |
| "혼자 쓰나요, 여럿이?" | 위임 구조 경량/full + 세션 커밋 루틴 |
| "권한 승인 생략 모드로 쓸 건가요?" | Yes → 가드 훅 **필수** (allowlist 무효 환경) |
| "수정하는 외부 정본(시트/DB)이 있나요?" | 위험 명령 차단 패턴 + **recovery 계약** (예방+복귀 한 쌍) |
| "PII/시크릿이 생길 경로는?" | gitignore 3중 방어 + git init 안전 순서 (→ 06) |

원칙:
- 질문은 구조화 도구(선택지 제시)로, 옵션마다 설명과 추천 표기를 붙인다.
- 전문 용어 대신 일상 비유 ("아키타입" → "에이전트 유형").
- 사용자가 모르겠다고 하면 안전한 기본값으로 — 인터뷰는 강요가 아니다.

## 3. Phase 3~4 — 생성과 검증의 분리

- **생성은 결정론으로**: 같은 입력 → 같은 출력. LLM이 직접 파일을 만들면 회차마다 구조가
  달라지고 검증 기준이 흔들린다. 인터뷰 답변을 구조화 데이터(JSON 등)로 고정하고,
  스크립트(없으면 최소한 고정 체크리스트)가 파일을 만든다.
- **생성 직후 즉시 검증**: 만든 사람(세션)이 따끈할 때 채점해야 결함 수정 비용이 최소다.
- 인터뷰 답변 원본은 `_meta/`에 보존한다 — 재생성 가능성(결정성)의 근거.

## 4. Phase 6 — 진화 루프 (생성보다 이게 본체다)

```
운영 → 피드백/사고 보고 → ★피드백 자체를 1차 데이터로 재검증★
  → 표준 반영 → 기존 산출물 회귀 테스트 → 변경이력 기록
```

세 가지 철칙:
1. **피드백 문서도 검증 대상** — "한 번도 안 쓰였다"는 보고는 세션 기록 등 1차 데이터로
   확인 후 반영하라. 측정 도구가 꺼져 있었다면 그 0건은 무효다.
2. **표준 변경엔 회귀가 따른다** — 변경 후 기존 산출물을 재생성/재검증해 등급 유지 확인.
3. **변경이력 없으면 드리프트다** — 표준·생성기·검증기를 고칠 때마다 한 줄 기록.

## 스스로 점검

- [ ] 같은 인터뷰 답변으로 재생성하면 같은 워크스페이스가 나오는가?
- [ ] 생성 직후 검증이 자동으로 따라붙는가?
- [ ] 최근 한 달간 운영 피드백이 표준에 반영된 사례가 있는가?
