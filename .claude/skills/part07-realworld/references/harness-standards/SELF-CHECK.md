# SELF-CHECK — 레이어별 통합 점검 워크시트

> 01~10의 "스스로 점검"을 레이어별 한 장으로 모은 실습지.
> **핵심 12**는 구축 시 반드시 적용, **심화 11**은 운영이 자리 잡은 뒤 점검.
> 각 항목 옆 괄호는 근거 문서.

## 핵심 12 — 구축 시 반드시

### 뼈대: 정체성 (L0)
- [ ] 1. 이 워크스페이스는 **하나의 특화 에이전트**인가 — 목적이 섞여 있지 않은가 (01)
- [ ] 2. 허용/금지 목록이 있고, 금지에는 이유가 붙어 있는가 (02)

### 뼈대: 구조 (L1)
- [ ] 3. 폴더 넘버링이 작업물 생애주기(작업 중 → 완성)를 표현하는가 (04)

### 뼈대: 지침 표면 (L2)
- [ ] 4. CLAUDE.md+rules에 **매 세션 불변식만** 있는가 — 예산 상한을 정했는가 (07)
- [ ] 5. 변경이력이 정본 분리 + 최근 3행 구조인가 (07)

### 하네스: 라우팅 (L3)
- [ ] 6. 모든 커맨드 description에 트리거 발화('~해줘')가 2개 이상 있는가 (03)
- [ ] 7. 비슷한 커맨드 쌍마다 NOT-trigger 경계가 있는가 (03)
- [ ] 8. 비가역 행위 커맨드에 자동발동 차단이 걸려 있는가 (03)

### 하네스: 위임 (L4)
- [ ] 9. 모든 서브에이전트 카드가 "언제 부르나" 게이트를 통과했는가 — 기본은 0종 (10)

### 하네스: 기계 강제 (L5)
- [ ] 10. "위반=사고"인 규칙이 문서가 아니라 훅으로 강제되는가 (05)
- [ ] 11. PII 커밋 게이트가 있고 우회 수단이 없는가 — gitignore 3중 방어 + init 순서 (06)

### 하네스: 운영 (L6)
- [ ] 12. 세션당 1커밋(복구 지점) 루틴이 있는가 (05)

## 심화 11 — 운영이 자리 잡은 뒤

- [ ] 13. 인접 워크스페이스와의 경계 한 줄 + 단방향 참조 규칙 (01, 06)
- [ ] 14. 메타 기록(`_meta/`)이 산출물과 분리되어 있는가 (04)
- [ ] 15. 폴더 주석에 수량 하드코딩이 없는가 (02)
- [ ] 16. 핵심 커맨드에 후속 작업 키워드(다시/고쳐/이어서)가 있는가 (03)
- [ ] 17. 카드마다 산출물 계약(형식·근거·핸드오프)이 명시되어 있는가 (10)
- [ ] 18. 가드 훅이 차단 4요소(차단+이유+절차+우회 마커)를 갖췄는가 (05)
- [ ] 19. Stop 게이트가 있고 루프 가드(stop_hook_active)를 처리하는가 (05)
- [ ] 20. 훅 설정 자체를 보호하는 장치(ConfigChange 가드)가 있는가 (05)
- [ ] 21. recovery 계약(실패 루프/재시도 한계/되돌리기)이 사고 전에 작성돼 있는가 (05)
- [ ] 22. 같은 규칙이 3곳 이상에 전문 복사돼 있지 않은가 — 정본+포인터 (07)
