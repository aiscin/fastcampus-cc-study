# 03. 라우팅 — description이 전부다

> 커맨드, 스킬, 서브에이전트가 "언제 발동하는가"를 결정하는 유일한 신호는 **description**이다.
> 본문이 아무리 훌륭해도 description에 라우팅 신호가 없으면 단 한 번도 호출되지 않는다.

## 1. 실측에서 일반화한 핵심 사실

같은 워크스페이스 안에서:
- 트리거 발화가 든 description을 가진 **커맨드들은 자연어 자동 라우팅이 실제로 작동**했고 (오발동 0),
- `"{역할}을 담당. 생성기가 합성한 카드"` 식 보일러플레이트 description을 가진
  **서브에이전트 카드들은 수십 회의 위임 기회에서 단 한 번도 선택되지 않았다.**

차이는 개수도, 본문 품질도 아니었다. **description 품질이 라우팅의 전부다.**

## 2. Pushy description 공식

```yaml
---
description: "[핵심 작업]과 [후속 작업] 요청에 사용합니다.
  '[사용자 발화 1]', '[사용자 발화 2]' 같은 요청에 대응합니다.
  단, [제외 경계]는 /다른커맨드를 사용합니다."
---
```

세 부품이 모두 있어야 한다:
1. **트리거 발화** — 사용자가 실제로 말할 자연어. "~해줘" 형태 2~3개.
2. **후속 작업 키워드** — 최초 생성만 쓰면 유지보수 요청에서 빠진다.
   재실행("다시 만들어") / 수정("고쳐", "다듬어") / 보완("추가", "보강") / 이어서("계속") / 축소("간단히").
3. **NOT-trigger 경계** — "단, X는 /other를 사용합니다." 비슷한 커맨드끼리의 충돌을 description 안에서 해소.

## 3. 좋은 예 / 나쁜 예 (가상 — 베이커리 운영)

```yaml
# 좋음
description: "단건 주문을 기록하거나 수정할 때 사용합니다. '오늘 주문 넣어줘',
  '아까 그 주문 수량 고쳐줘', '주문 취소 처리해줘' 같은 요청에 대응합니다.
  단, 주간 집계는 /settle을, 재고 확인만은 /stock을 사용합니다."

# 나쁨
description: "주문 관리"
```

## 4. 서브에이전트 카드에도 똑같이 적용하라

서브에이전트는 커맨드보다 더 자주 이 원칙이 무시된다. 카드 생성 전 게이트 질문:

> **"이 역할을 언제 부르는가?"에 '~해줘' 발화 2~3개로 답할 수 있는가?**

- 답할 수 있으면 → 그 발화를 description에 넣고 카드를 만든다.
- 답할 수 없으면 → **카드를 만들지 않는다.** generic 카드는 dead code + 상시 컨텍스트 비용이다.

카드 description에는 추가로 **산출물 계약**(이 역할이 무엇을 돌려주는지)을 한 구절 넣는다 —
호출하는 쪽(leader)이 결과 형태를 예측할 수 있어야 위임이 습관이 된다.

```yaml
# 서브에이전트 카드 description 예 (가상)
description: "주문·재고·정산 기록의 3중 대조를 담당. '장부 맞는지 확인해줘',
  '재고랑 주문 대조해줘' 같은 요청일 때 위임합니다. 단, 단건 기록·조회는
  직접 처리합니다. 산출물: 불일치 목록 + 심각도 표."
```

## 5. 위험도 차등 — 자동발동을 막아야 하는 것

발동이 잘 되는 것만큼, **되면 안 될 때 안 되는 것**도 라우팅 설계다.

- 비가역·확정 행위(월말 마감, 규칙 확정, 외부 발송)는 자동발동 차단 옵션
  (Claude Code: `disable-model-invocation: true`)을 걸어 사용자가 명시 호출할 때만 돌게 한다.
- 일상 커맨드는 자동발동 허용 — 차단을 남발하면 라우팅 자체가 죽는다.

## 6. 한국어 팁

- description은 YAML 큰따옴표 스칼라에 들어가는 경우가 많다 — 트리거 발화 인용은
  **작은따옴표**로 (중첩 큰따옴표 = 파싱 깨짐).
- 템플릿 치환으로 description을 합성한다면 조사(을/를) 처리에 주의 — 받침 판별이 번거로우면
  **어순 회피**("담당 영역: {X}")가 더 안전하다. `{X}을(를)` 병기 노출은 미완성 티가 난다.
