# 02. 지침 표면(Instruction Surface) — CLAUDE.md 작성 원리

> CLAUDE.md는 에이전트의 헌법이다. 매 세션 무조건 로드되므로(→ 07 컨텍스트 예산),
> "무엇을 쓰나"만큼 "무엇을 쓰지 않나"가 중요하다.

## 1. 역할

| 역할 | 내용 |
|------|------|
| 컨텍스트 | 이 워크스페이스의 목적·배경·제약 |
| 규칙 | 작성/작업 규칙, 금지 사항, 품질 기준 |
| 워크플로우 | 작업 순서, 커맨드 사용법 |
| 참조 매핑 | 폴더 구조와 각 폴더의 역할 |

## 2. 권장 구성 (위에서 아래로)

```markdown
# {이름}
> 한 줄 정의 — 이 에이전트가 무엇인지

## ⚠️ 정체성 (허용/금지)     ← L0. 가장 먼저, 가장 단단하게
## 핵심 원칙 (테이블, ≤7개)
## 폴더 구조 (트리 + 역할 주석)
## 워크플로우 (다이어그램)
## 커맨드 목록 (테이블)
## 트리거 경계 (should / NOT)  ← 03 참조
## 품질 규칙
## 변경 이력 (최근 3행 + 정본 포인터)  ← 07 참조
```

## 3. 작성 원칙

1. **불변식만** — 매 세션 필요한 것만 남긴다. 작업별 절차는 스킬로, 상황 의존 정보는 훅으로 (→ 07).
2. **구체적 수치/예시** — "적절히", "충분히" 금지. "재시도는 3회까지", "표는 7행 이하"처럼.
3. **금지는 이유와 함께** — "X 금지 (Y가 깨지기 때문)" 형태가 모델 준수율을 높인다.
4. **개수 하드코딩 금지** — 폴더 트리 주석에 "커맨드 11종" 같은 수량을 쓰지 마라.
   실제와 즉시 어긋나는 드리프트의 단골 원인. 개수 대신 역할을 서술한다.
5. **중복 금지** — 같은 규칙을 CLAUDE.md와 rules와 커맨드에 풀어 쓰면 한 곳만 고쳐져 드리프트한다.
   정본 1곳 + 위반 시 사고급인 불변식만 사용처에 1줄 인라인 + 나머지는 포인터 (→ 07).

## 4. 예시 (가상 — 온라인 베이커리 운영 에이전트)

```markdown
# bakery-ops
> 온라인 베이커리의 주문·재고·발주를 기록하고 정산하는 운영 에이전트.

## ⚠️ 정체성
✅ 허용: 주문/재고 기록, 주간 정산 집계, 발주서 초안
❌ 금지: 결제 처리 실행, 고객 개인정보가 담긴 파일의 git 추적,
        레시피 개발(별도 워크스페이스 — 단방향 참조만)
```

마지막 줄에 주목 — 인접 워크스페이스(레시피 개발)와의 **경계 한 줄**이 L0에 들어간다.
경계가 없으면 에이전트는 옆 동네 일을 끌어와 정체성이 무너진다.

## 5. Scale Modes — 같은 커맨드도 작업 무게별 차등

모든 작업에 같은 강도의 절차를 강제하면 가벼운 작업이 절차 비용에 짓눌리고,
무거운 작업이 가벼운 절차로 부실해진다. 워크플로우에 3단 모드를 선언한다:

| 모드 | 대상 | 절차 |
|------|------|------|
| **Lite** | 단건 기록, 빠른 조회, 짧은 산출물 | 최소 단계 — 기록하고 끝 |
| **Standard** | 일반 작업 단위 | 기본 워크플로우 전체 |
| **Full** | 장기 프로젝트, 구조·검증이 중요한 작업 | 전체 + 검증·진행률·프레임워크 |

커맨드별로 기본 모드를 지정하고(예: 단건 기록 커맨드=Lite, 마감 커맨드=Full),
CLAUDE.md에 "어떤 작업이 어떤 모드인가"의 기준을 한 절로 둔다.
모드 구분이 없으면 에이전트는 모든 요청을 같은 무게로 처리한다 — 보통 과하게.

## 6. 안티패턴

| 안티패턴 | 문제 | 해결 |
|----------|------|------|
| 너무 김 | 핵심을 놓침 | 07의 예산 원칙으로 다이어트 |
| 너무 짧음 | 컨텍스트 부족 | 구체 예시 추가 |
| 모호한 표현 | 해석 차이 | 수치/예시 |
| 변경이력 본문 누적 | 상시 컨텍스트 비대 | 정본 분리 + 최근 3행 |
| 수량 하드코딩 | 즉시 드리프트 | 역할 서술 |

## 스스로 점검

- [ ] 정체성 섹션만 읽고 "이 에이전트가 안 하는 일"을 3개 말할 수 있는가?
- [ ] CLAUDE.md에서 "이번 주에만 유효한 정보"를 찾을 수 있는가? (있다면 잘못된 위치)
- [ ] 같은 규칙이 두 곳 이상에 전문으로 적혀 있는가? (있다면 정본+포인터로)
