# 10. 위임 설계 — 서브에이전트는 증거가 있을 때만

> L4(위임)는 가장 과설계되기 쉬운 레이어다. 원칙: **카드는 살(도메인 구체성)이 있을 때만,
> 공유 구조는 위임이 실제로 돌 때만.**

## 1. 위임이 정당화되는 조건 — 세 가지뿐

1. **컨텍스트 격리** — 메인 대화를 오염시키지 않고 큰 탐색/검증을 수행 (예: 수백 파일 전수 대조)
2. **관점 분리** — 작성자가 자기 산출물을 검증하면 안 되는 경우 (검토자 역할)
3. **병렬성** — 독립 작업의 동시 수행

이 셋에 해당하지 않으면 **직접 처리가 기본값**이다. 특히 1인 운영의 일상 작업
(단건 기록, 조회)은 위임이 오히려 비용이다.

## 2. 카드 생성 게이트

카드를 만들기 전 통과해야 하는 질문:

> **"이 역할을 언제 부르는가?"에 '~해줘' 발화 2~3개로 답할 수 있는가?**

- No → 만들지 않는다. generic 카드는 라우팅되지 않는 dead code + 상시 컨텍스트 비용이다.
- Yes → 그 발화가 description에 들어간다 (→ 03).

기본값은 **카드 0종**이고, 도메인 구체성이 채워진 역할만 추가한다.

## 3. 카드 본문 — 도메인 살 4종

description(라우팅)과 별개로, 호출된 뒤의 행동 품질은 본문이 결정한다. 4종을 채운다:

| 살 | 내용 | 예 (가상 — 베이커리 정산 검토 역할) |
|----|------|------|
| **정체성** | 도메인 + 성격 형용사 2~3개 | "주문·재고·정산 기록을 대조하는 검토자. 의심 많은, 증빙 없으면 통과 안 시키는." |
| **방법(methods)** | [상황, 방법, 이유] 3열 표 3~5행 | [월말 정산, 주문↔재고↔입금 3중 대조, 단일 출처는 오류를 못 잡음] |
| **실패 패턴** | 이 도메인의 안티패턴 2~4개 | "입금 '대기'를 확인 없이 '완료'로 추정" |
| **KPI** | 산출물 품질의 측정 기준 2~3개 | "대조 수행률 100%, 미확인 항목 0건" |

방법 표의 **상황 열은 트리거 발화의 원천**이기도 하다 — 상황을 구체적으로 쓰면
description 합성 재료가 된다.

## 4. 역할별 도구 정책 차등

역할의 성격이 도구 권한을 결정한다:

| 역할 성격 | read | write | 외부(web 등) |
|----------|------|-------|------|
| 탐색/검토 (critic류) | 전체 | **검토 보고서만** | 불필요 |
| 깊이 작업 (builder/writer류) | 전체 | 산출물 폴더 | 불필요 |
| 조사 (researcher류) | 전체 | 조사 노트만 | 허용 |

검토 역할에 쓰기 권한을 주면 "검토하다가 고쳐버리는" 월권이 생기고,
관점 분리(§1-2)가 무너진다.

## 5. 출력 계약

위임이 습관이 되려면 호출하는 쪽이 결과 형태를 예측할 수 있어야 한다. 카드마다 명시:
- **최종 답변 형식** (예: 이슈 목록 + 심각도 표)
- **근거 요구** (각 판정마다 파일:라인 또는 계산식)
- **핸드오프** (후속을 누구에게 — 수정은 builder 역할로, 판단 충돌은 사용자에게)

## 6. 공유 구조 — light가 기본, full은 증거가 생기면

| 모드 | 구성 | 언제 |
|------|------|------|
| **none** | 없음 | 카드 0종일 때 |
| **light** | 위임 로그 1파일 (자동 기록, 수동 갱신 의무 없음) | 카드가 있는 기본값 |
| **full** | 계획/진행/발견 공유 파일 + 검증 루프 + 위임 정책 문서 | 멀티에이전트가 **실제로 돌고 있다는 기록**이 쌓인 뒤 |

경고 — **못 지킬 의무를 만들지 마라**: "모든 위임 전후 공유 파일 갱신 의무" 같은 조항은
소규모 운영에서 지켜지지 않고, 사문화된 의무는 다른 규칙의 신뢰도까지 갉아먹는다.
의무 대신 자동 기록(훅)을, 자동이 안 되면 권장을.

## 7. 측정과 가지치기

- 위임 로그(또는 세션 기록)로 **카드별 실제 호출 횟수**를 주기적으로 본다.
- 일정 기간 0회인 카드는 축소/아카이브 후보다 — 단, **측정 도구가 켜져 있었는지부터 확인**
  (꺼진 로그의 0건은 무의미).
