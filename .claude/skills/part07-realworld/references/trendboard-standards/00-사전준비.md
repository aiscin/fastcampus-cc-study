# 00. 사전 준비 — 키 발급 + Supabase 연결

> clip-06 Phase 3에서 보는 문서. **진입장벽 구간이라 순서대로 같이 간다.**
> 손으로 받는 건 **YouTube·Gemini 2종뿐** — Supabase는 클로드코드가 MCP로 만든다.
> 전부 **각자 본인 계정·본인 프로젝트로.** (강사 것 공유 X — 내 대시보드는 내 DB에)

## 이 문서에서 받는 것

| # | 어디서 | 받는 것 | 어디에 넣나 | 방식 |
|:-:|---|---|---|---|
| 1 | Supabase | 프로젝트 + URL + anon 키 | `.env.local` (프론트) | **MCP 자동** |
| 2 | Google Cloud | `YOUTUBE_API_KEY` | Edge Function **Secret** | 손으로 |
| 3 | Google AI Studio | `GEMINI_API_KEY` | 앱 **Admin 페이지** (DB) | 손으로 |

> 걸리는 시간: 처음이면 ~10분. **이 클립에서 몰아서 끝내면 다신 안 한다.**

### 키가 사는 곳이 다르다 (헷갈리기 쉬운 지점)

```
프론트(.env.local, Vercel)  ← VITE_SUPABASE_URL, VITE_SUPABASE_PUBLISHABLE_KEY  (공개 OK)
Edge Function Secret        ← YOUTUBE_API_KEY                                    (비밀)
DB(admin_settings 테이블)   ← gemini_api_key                                     (비밀, Admin UI로 저장)
자동 주입(설정 불필요)      ← SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY            (비밀)
```

**Gemini 키만 유별나다** — env가 아니라 **DB에 저장**한다. Admin 페이지에서 키·모델·프롬프트를
코드 수정 없이 바꾸려고 그렇게 설계돼 있다 (`admin_settings` 테이블).

## 1. Supabase — MCP로 자동 생성

클로드코드에게 시킨다. 대시보드에서 손으로 만들 필요 없다.

**복사용 입력:**
```
Supabase에 trendboard 프로젝트를 MCP로 만들려는데, 어떻게 하면 될까? 만들고 나서 URL이랑 anon 키를 .env.local에 넣어줘.
```

| 확인 | 프로젝트 생성됨 / `.env.local`에 `VITE_SUPABASE_URL`·`VITE_SUPABASE_PUBLISHABLE_KEY` |

> `SUPABASE_SERVICE_ROLE_KEY`는 **Edge Function에 자동 주입**된다 — 따로 발급·설정하지 않는다.
> 프론트·깃엔 절대 들어가면 안 되는 키라, 손댈 일이 없는 게 정상이다.

### 폴백 — `/mcp`에 Supabase 커넥터가 없을 때

MCP 없이도 된다. 손으로 하고 값만 넘긴다.

1. [supabase.com](https://supabase.com) 가입 → **New project**
2. 이름 `trendboard`, DB 비밀번호 설정(메모), 리전은 **가까운 곳**(Northeast Asia)
3. 생성까지 ~2분 → **Project Settings → API**에서 2개 복사:
   - `Project URL` → `VITE_SUPABASE_URL`
   - `anon` / `public` 키 → `VITE_SUPABASE_PUBLISHABLE_KEY`
4. 스키마도 손으로: **SQL Editor** → `_spec/schema.sql` 전체 붙여넣고 **Run**

> anon 키는 **공개돼도 되는 키**다(브라우저에 그대로 나간다). RLS가 방어선이라 그렇다.
> 같은 화면의 `service_role` 키는 반대 — **절대 프론트·깃 금지.**

## 2. YouTube Data API v3 — `YOUTUBE_API_KEY`

가장 자주 막히는 곳: **키만 만들고 API를 Enable 안 해서 403.** 순서대로.

1. [console.cloud.google.com](https://console.cloud.google.com) 접속
2. 상단 프로젝트 선택 → **New Project** (이름 `trendboard`) → 만든 프로젝트로 전환
3. **APIs & Services → Library** → `YouTube Data API v3` 검색 → **Enable** ★
4. **APIs & Services → Credentials** → **Create Credentials → API key**
5. 키 복사 → 메모 (`YOUTUBE_API_KEY`)

> ★ 3번을 건너뛰면 키는 멀쩡한데 호출이 **403**으로 죽는다. 403이 뜨면 여기부터 다시 본다.

**(권장) 키 제한** — Credentials에서 키 클릭 → **API restrictions** → `YouTube Data API v3`만 체크.
키가 새도 다른 Google API로는 못 쓴다.

| 확인 | Library에서 YouTube Data API v3가 **Enabled** 표시 / 키 메모됨 |

### 쿼터 — 하루 10,000 units (무료)

| 호출 | 비용 | 쓰나 |
|---|:-:|---|
| `playlistItems.list` | **1** | ✅ 영상 목록은 이걸로 |
| `videos.list` | **1** | ✅ 50개 콤마조인으로 한 번에 |
| `channels.list` | **1** | ✅ 채널 등록 시 |
| `search.list` | **100** | ❌ **금지** |

> **`search.list` 금지**가 이 프로젝트의 1급 규칙이다. 100배 비싸서 채널 몇 개만 돌려도 하루 쿼터가 날아간다.
> 대신 `uploads` 재생목록을 쓴다 — 채널 ID `UCxxxx`의 앞 두 글자를 `UU`로 바꾸면(`UUxxxx`)
> 그게 그 채널의 전체 업로드 재생목록이다. 그래서 1 unit으로 끝난다.

채널 10개 × 매시간 수집이면 하루 대략 500~1,000 units — 무료 쿼터 안에서 넉넉하다.

## 3. Gemini — `GEMINI_API_KEY`

1. [aistudio.google.com/apikey](https://aistudio.google.com/apikey) 접속 (Google 로그인)
2. **Get API key → Create API key**
3. 키 복사 → 메모 (`GEMINI_API_KEY`)

> 이 키는 **`.env`에 넣지 않는다.** clip-07에서 앱을 띄운 뒤 **Admin 페이지 → 설정**에 저장한다
> (→ `admin_settings.gemini_api_key`). RLS가 anon 읽기를 막고 있어서 브라우저로는 안 새어나간다.

**쓰는 모델 (기본값):**

| 용도 | 모델 | temperature |
|---|---|:-:|
| 태깅·요약 (`generate-tags`) | `gemini-2.5-flash-lite` | 0.3 |
| 콘텐츠 전략 (`generate-strategy`) | `gemini-2.5-flash` | 0.5 |

모델명도 `admin_settings`(`ai_model_tags`·`ai_model_strategy`)라 **코드 수정 없이 Admin에서 교체**된다.

| 확인 | 키 메모됨 / 무료 티어로 충분(요약 없는 신규 영상만 태깅) |
