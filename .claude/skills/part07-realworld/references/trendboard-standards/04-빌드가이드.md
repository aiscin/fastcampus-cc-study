# Phase 1 — 생성

## STEP 1. `/show-me-the-prd` — PRD 4종 (6분)

PRD를 직접 쓰지 않고 스킬에 시킨다.

```
/show-me-the-prd 유튜브 채널들 영상 성과를 매시간 수집해서 '채널 평균 대비 터진 영상(Outlier)'을 자동으로 잡아내고, 트렌드 키워드까지 보여주는 분석 대시보드를 만들고 싶어.
```

질문 답 요지:
- **사용자** = 채널을 추적하는 나 (크리에이터·마케터·기획자)
- **기능** = 채널등록 · 매시간수집 · Outlier · AI요약 · 트렌드
- **안 만드는 것** = 로그인 · 결제 · 멀티플랫폼

| ✅ 검증 | PRD 4종 생성 / 범위가 **Outlier + 트렌드**로 좁혀짐(욕심 X) |

> 정답 레퍼런스: `01-PRD.md`. 똑같을 필요 없다 — 다음 STEP에서 수렴시킨다.

## STEP 2. 구체화 — 공통 사양으로 수렴 ★★ (6분)

> **강사 노하우 지점이자 이 프로젝트에서 제일 중요한 STEP.**

`/show-me-the-prd`는 **사람마다 다른 PRD**를 만든다. 잘 만든 **이상형**이지 실제 동작 사양이 아니다.
그대로 kkirikkiri에 넣으면 **누군 되고 누군 안 된다** (공식·함수·뷰가 제각각).
→ 미리 깔아둔 **공통 사양(`trendboard/_spec/`)** 으로 수렴시켜 모두 **동일한 컨텍스트**로 시작한다.

> **PRD 만드는 경험은 각자, 빌드 사양은 하나로.**

```
방금 만든 PRD에 trendboard/_spec/의 02·03·BUILD-PROCESS 실제 사양을 박아줘 — video_details 뷰(컬럼 video_id·snapshot_at), Outlier Score=(views/avg)*10+vsRatio, AI 태깅은 generate-tags(summarize 아님), 체인은 collect-data→generate-tags→extract-trends. kkirikkiri에 줄 통합 사양서로 정리해줘.
```

| ✅ 검증 | `+vsRatio` 포함 / `generate-tags` 명시 / 뷰 컬럼명 보존 / 화면 7 + Edge Function 7 + 체인 / 03의 DO/DON'T 포함 |

### 병렬로: 키 발급 + 스키마 (clip-06 Phase 3·4)

kkirikkiri가 도는 동안 하면 시간이 안 아깝다.

1. **키 2종** — YouTube·Gemini (`00-사전준비.md`). Supabase는 MCP 자동.
2. **스키마 적용**:
```
trendboard/_spec/schema.sql을 내 Supabase에 MCP 마이그레이션으로 적용해줘. 테이블 7개랑 video_details 뷰가 생겨야 해. 뷰 컬럼명(video_id·snapshot_at)은 바꾸지 마.
```

| ✅ 검증 | Table Editor 7테이블 + `video_details` 뷰 / `select video_id, snapshot_at from video_details` 에러 없음(빈 결과 OK) |

# Phase 2 — 연결·운영

## STEP 4. Supabase 연결 + 함수 배포 + 수집 ★ (12분)

**디버깅 핵심 구간.** 셋으로 나눠 간다.

**4-a. Supabase 연결**
```
사양대로 Supabase 연결하자. 프로젝트 없으면 MCP로 만들고, URL·anon 키를 .env.local에 넣어줘(.gitignore 추가). 클라이언트 확인하고, STEP 2에서 적용한 schema.sql 스키마가 video_details 뷰까지 맞는지 확인해줘.
```
| ✅ | `.env.local` gitignore 포함(공개 가능한 anon만) / 뷰 select 에러 없음 |

**4-b. Edge Function 배포 + Gemini 키**
```
Edge Function 6종을 MCP로 배포해줘. SUPABASE_URL·SERVICE_ROLE_KEY는 자동 주입이라 설정 불필요 — YOUTUBE_API_KEY만 Secret으로 넣는 법(대시보드/CLI) 알려줘. 앱 띄워서 Admin 로그인(기본 trendboard2026, 바로 변경)하고 Gemini 키 저장하자.
```
> ⚠️ **여기 한 곳만 수동**: `YOUTUBE_API_KEY` Secret 등록은 MCP로 안 된다.

| ✅ | 함수 6종 배포 / Secret 등록 / Admin 로그인 → **비번 변경** → Gemini 키 저장 / anon으로 `gemini_api_key` 안 보임(RLS) |

**4-c. 채널 등록 → 수집**
```
테스트 채널 5개 등록하고 collect-data를 수동 실행해줘. videos·video_snapshots 채워지고 avg_views>0, generate-tags 체인으로 content_summaries에 키워드 생기는지 확인. 안 되면 Edge Function 로그 보고 고쳐줘.
```
| ✅ | videos·snapshots 채워짐 / `avg_views > 0` / content_summaries에 키워드 / Shorts·Long 구분 |

> 채널당 **50개**(`max_videos_per_channel`) 초기 수집 → 이후 **증분**. `search.list` 쓰면 안 된다.
> 라이브 수집이 느리면 강사 결과 화면으로 결과만 보여주고, 내 수집은 **백그라운드**로 두고 진행.

## STEP 5. Outliers 대시보드 검증 (5분)

```
Outliers 페이지에서 Score 정렬·Ch.Avg 배율·필터가 맞는지 확인해줘. outlierScore가 (views/avg)*10+vsRatio인지(단순 배율×10 아님), Tier 랭킹, content_tags 컬러 배지까지.
```

| ✅ | Score 내림차순 / Ch.Avg 배율 초록 / Format 필터 / Tier "#1 of N" / 요약 카드 4 + 12컬럼 + 페이지네이션 + CSV |

> **Score가 전부 0이면** `avg_views`가 아직 0이다 — collect-data를 1~2회 더 돌리면 채워진다(신규 채널 정상).

## STEP 6. AI 태깅 — `generate-tags` (5분)

> ⚠️ `summarize-videos`가 **아니다** (레거시).

```
generate-tags로 요약 없는 영상에 한 줄 요약·키워드·태그를 생성해서 테이블에 컬러 배지로 보여줘. (개선) Mega Outlier(5x↑)는 빨간 배지, 썸네일 더 크게.
```

| ✅ | content_summaries에 행 / Outliers·Videos에 키워드 컬러 배지 / 말로 시킨 UI 개선 반영 |

> Gemini **429**: 15초 대기 후 재시도, 호출 간 1.5초 sleep. batch 기본 20. 요약 **없는** 영상만.

Trends 페이지랑 extract-trends 만들어줘. video_details에서 keywords 실시간 집계(24h/3D/7D/30D), 바 차트 + 랭킹. Overview도 카드·차트·최근 Outlier로 마무리.

| ✅ | 키워드 랭킹 + 성장률 색상 / 기간 탭 / 바 차트 / Overview 카드·7일 Score 라인차트·Recent Outliers |
