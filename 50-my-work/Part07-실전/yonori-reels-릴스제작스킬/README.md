## 무엇을 만들었나

@travel_yonori 인스타 계정 전용 릴스 제작 파이프라인. 톤앤매너 프로파일 문서 하나를 모든 단계가 참조해 통일성을 유지한다.

```
Step 0  프로파일 게이트 (30일 신선도 체크)
Step 1  인스타 API 수집 (fetch_profile.py — 차단 시 수동 폴백)
Step 2  톤앤매너 프로파일 (반말/해요체 이원화 · 성과 신호 지문 · 훅 라이브러리)
Step 3  주제 선정 (빠른/딥 모드 — 딥 모드는 deep-research 재호출 · 100점 루브릭)
Step 4  대본 생성 (9컬럼 타이밍표 · 발화 공식 · 리텐션 6장치)
Step 5  영상 조립 (build_reel.py — edge-tts→묵음트림→정규화→ASS 자막→1080x1920)
Step 6  성과 회수 (viral ratio 재수집 → 가설 카운터 갱신 — 쓸수록 똑똑해짐)
```

## 핵심 배운 것

- **전문가 팀 검증의 가치**: 설계 단계에서 4인 Agent Teams가 실측으로 찾아낸 것 — drawtext 필터 부재, edge-tts 뒤 0.9초 묵음, 24kHz 모노 출력, 발화속도 공식. 만들고 나서 발견했으면 전부 재작업이었다.
- **측정 루프**: "홍콩 모녀여행 45배 vs 캐리어 0.31배" → 송신형(DM 공유 유발) 우위 가설. n=1이라 확정하지 않고 Step 6이 지지/반증을 세게 했다.
- **과잉설계 방어**: 검수자가 "주제 하나 고르는데 에이전트 3명은 과잉" 반박 → 단일 패스 채점 + 조건부 스폰으로 축소.
