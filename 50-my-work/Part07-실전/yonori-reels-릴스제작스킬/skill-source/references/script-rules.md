# 작법 규칙 (Step 2·4에서 참조)

> 전문가 검증(2026-07-06, 실측 포함)으로 확정된 규칙. 수치는 근거 없이 바꾸지 않는다.

## §1. 발화시간 공식 (실측 검증 완료)

```
예상 장면초 = 음절수 ÷ 6.0 + 0.25 × 쉼표수 + 0.15
```

- 근거: edge-tts(SunHiNeural) 5문장 실측 + 조립 파이프라인 검증(2건, 오차 5~9%). 단일 상수(초당 5.5 또는 3.7음절)는 짧은 문장에서 최대 68% 오차
- build_reel.py가 edge-tts의 파일당 고정 묵음(~1.1초)을 트림하고 0.15초 간격을 넣으므로 위 공식이 **조립 후 실제 장면 길이**다. 트림 전 원본 mp3 길이는 `+1.1초`를 더한 값
- 적용: 타이밍표의 "예상발화(초)" 컬럼 자동 계산. 장면 길이와 ±20% 이상 벌어지면 ⚠️ 경고 표시
- 분량 상한 (공식 역산): 30초 릴스 ≈ 나레이션 총 140~160음절 / 60초 ≈ 290~320음절. 초과 = 과밀 대본(속도 붕괴) — 문장을 줄인다
- **개인화 (목소리 직접 녹음 체제)**: 6.0은 TTS 실측값. 사용자 녹음 파일을 받으면 실측 발화속도(총음절÷총발화초)를 계산해 `tone-profile.md`에 기록하고, **이후 대본부터 6.0 대신 개인 실측값**을 쓴다. 녹음이 쌓일 때마다 최근 3회 평균으로 갱신 (본인 속도와 장면 컷이 어긋나는 싱크 문제 방지)

## §2. 훅 (첫 1.5초)

- 길이: 8~10음절 이내
- 유형 3분류: 질문형("~한 적 있음?") / 반전·손실형("이 3천원이 여행을 살렸다") / 숫자형
- **송신 훅 우선**: 수신자를 호명해 시청자가 즉시 특정인을 떠올리게 한다 — "엄마랑 홍콩 갈 거면 이건 꼭 봐" (DM 공유 = 최상위 랭킹 신호)
- 금지: "우리 ~ 갔어요" 브이로그식 시작 (실측 사망 패턴, viral ratio 0.31x)
- 훅이 던진 질문은 본문이 반드시 회수한다 (낚시 인식 방지)

## §3. 리텐션 6장치 체크리스트 (Step 4 산출 전 통과 필수)

| # | 장치 | 확인 |
|---|------|------|
| 1 | 송신 훅 — 첫 1.5초에 수신자 호명 또는 공유 트리거 | ☐ |
| 2 | 3초 오픈루프 — 첫 줄에 질문/결과 프리뷰 | ☐ |
| 3 | 패턴 인터럽트 — 3~5초마다 장면 컷 + 자막 교체 | ☐ |
| 4 | 저장 미끼 프레임 — 중반에 가격·시간·주소 밀도 프레임 1장 | ☐ |
| 5 | 루프 클로징 — 끝 문장이 첫 프레임과 연결 (무봉합 재생) | ☐ |
| 6 | 송신 CTA — "같이 갈 사람 태그해" + 본문 연결 댓글 질문 1개 | ☐ |

- 각 장면의 타이밍표 "트리거 신호" 컬럼에 해당 장치 번호를 기록한다
- CTA 기준: 본문 정보와 연결된 구체 질문형만 (산 CTA: "너라면 몇 개까지 들고 다닐 듯?" / 죽은 CTA: "팔로우해줘")

## §4. TTS 전제 작문 규칙

- **숫자·외래어 한글 풀어쓰기**: 대본 "발음표기" 컬럼에 기록, TTS 입력에는 한글만 사용
  - "14,590회" → "만사천오백구십 번" / "Cairo" → "카이로"
- **이모지·특수문자 제거**: 나레이션 문장에서 삭제 (자막·캡션에는 허용)
- **호흡**: 5~7음절마다 쉼표. 예: "3발 캐리어를, 양손이랑 발로, 낑낑 끌고 다닌 적 있음?"
- **문장부호 생략 금지**: 반말체라도 마침표·물음표를 찍어 TTS가 문장 경계를 인식하게 한다
- 문장 1개 = 장면 1개 = 3~5초. 같은 어미 3연속 금지 (어미 분포표 참조)

## §5. 자막 상수

- 한 줄 12자 이내, 동시 표시 2줄 이하, 최소 표시 1.5초/줄
- 자막은 나레이션의 **축약형** — 100% 받아쓰기 금지 (화면이 텍스트로 덮이는 것 방지)
- 렌더링: ASS 전용 (이 환경 ffmpeg에 drawtext 없음 — 실측). Malgun Gothic, Alignment=2, MarginV=250(하단 세이프존: 인스타 UI 겹침 회피), PlayResX=1080/PlayResY=1920
