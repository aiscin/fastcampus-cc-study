# 주제 선정 절차 (Step 3 상세)

## 1. 모드 판정

| 조건 | 모드 |
|------|------|
| `data/trend-cache-{YYYYMM}.md` 존재 + 14일 이내 | **빠른 모드** (기본) |
| 캐시 없음/만료 | **딥 모드** 제안 |
| 사용자가 "제대로/깊이 조사해서" | 딥 모드 |
| 캐시에 없는 신규 목적지·테마 요청 | 딥 모드 |
| 직전 릴스 2개 연속 viral ratio < 3x (프로파일 확인) | 딥 모드 제안 — "새 각이 필요해 보여요" |

## 2. 빠른 모드

1. 트렌드 캐시 읽기 + 부족한 부분만 WebSearch 2~3회 즉석 확인
2. 프로파일의 성과 신호 지문·훅 라이브러리와 결합해 후보 3~5개 생성
3. 메인 스레드가 루브릭(§4) 단일 패스 채점 — 에이전트 스폰 없음

## 3. 딥 모드

1. **deep-research 스킬을 Skill 도구로 재호출**. 쿼리 템플릿 ("여행 트렌드" 같은 광의 쿼리 금지 — 계정 승리 신호를 쿼리에 박는다):
   ```
   2026년 {현재 계절/월} 한국 인스타그램에서 급상승 중인 '이색여행·데이트·놀거리' 소재 중,
   시청자가 특정인에게 DM으로 공유하고 싶어지는(관계형·감정 유발) 소재를 우선 조사.
   각 소재별: (1) 검색·언급 수요 신호 (2) 최근 릴스 실제 조회수 사례 (3) 왜 공유를 유발하는지 1줄.
   가성비·10년차 직장인 여행자 관점 필터 적용.
   ```
2. 결과 리포트를 `data/trend-cache-{YYYYMM}.md`로 저장 (다음 14일간 빠른 모드가 재사용)
3. **실패 폴백**: deep-research 실패 시 WebSearch 직접 조사로 강등하고 사용자에게 "간이 조사로 진행했어요" 고지. 캐시에는 `(간이)` 라벨
4. 루브릭 채점 (§4)

## 4. 루브릭 (100점)

| 축 | 배점 | 채점 질문 |
|----|------|-----------|
| 송신 유발력 | 40 | 시청자가 특정인에게 DM 보낼 소재인가? (검증된 45x 레버 — 최고 배점) |
| 계정 적합성 | 25 | 니치(이색·데이트·가성비)·반말톤·10년차 직장인 페르소나 부합? |
| 차별각 | 20 | 같은 소재 릴스 100개 대비 고유한 각이 있는가? |
| 제작 실현성 | 15 | 실제 촬영·제작 가능한가? (촬영 불가 소재는 90점이어도 가치 0) |

- **≥70 통과 / 55~69 각 수정 후 재채점 / <55 폐기**
- 후보 카드 필수 3필드: `트렌드 소스 / 이 계정만의 각 / 예상 송신 트리거` + 예상 훅 문장 (트렌드 복붙 방지 가드레일)
- 소프트 쿼터: 후보 중 송신형 ≥1 권장 (하드 쿼터는 가설 카운터 지지 3건+ 이후 격상 검토)

## 5. 검증 에이전트 스폰 (예외적)

기본은 단일 패스 채점. 아래일 때만 Agent 도구로 검증관을 스폰한다 (비용 대비 — "주제 하나 고르기"에 매번 3명은 과잉):

- (a) 상위 2후보 점수차 ≤ 5점 → 각 후보에 독립 재채점 1명씩 (블라인드)
- (b) 트렌드 검증과 계정 적합이 하드 충돌 (예: 소재는 뜨거운데 계정 톤과 정면 배치) → 그 쟁점 하나만 타깃 재질의 1회

다라운드 토론 금지 (동일 모델 간 쏠림·가짜 합의 방지). 결과는 점수 분해와 함께 사용자에게 제시.

## 6. 사용자 선택

후보를 점수 순으로 정렬해 AskUserQuestion으로 제시 — 각 옵션에 `점수 분해 + 예상 훅 + 이 계정만의 각` 포함. 최종 결정은 항상 사용자.
