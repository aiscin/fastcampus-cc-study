## 2. P0 — 반드시 고쳐야 빌더가 "안전하게" 동작함

### P0-1 · 강제 #1 "경계 모순" (guard·breach·audit 3인 합치)

설계도 line 34 "40-output **밖** 수정 차단" ↔ line 120 "실제 워크스페이스는 40-output **밖** 독립 경로에 생성".
→ 훅이 정직하면 **핵심 기능(찍어내기)이 죽고**, 예외 허용하면 `~/.claude/`·남의 워크스페이스에 "정상 산출물"로 **덮어쓴다**. 자기모순 가드.

**개선 (guard-doc 확정 문구)**: "밖 절대 차단"을 폐기하고 **per-run 쓰기 잠금(write-scope lock)**으로 대체.

> 빌드 착수 시 그 run의 쓰기 허용 경로를 잠근다: (a) 빌더 `30-drafts`·`40-output`, (b) 이번 요청이 선언한 독립 출력경로 **1개**, (c) 그 외 차단. 단 (b)가 **이미 존재하면** 쓰기 차단 → 타임스탬프 백업 후 진행. `10-standards`·`20-blueprints`·`00-inbox` 원본은 항상 읽기전용.

### P0-2 · 스킬 복사 오염 = 무가드 자동발동 (breach·instruction)

line 122 "스킬 라이브러리 16종에서 복사" + line 139 "description 유지" → 능력(외부발송·과금·PII)은 오는데 **가드는 안 오고**, description 유지로 **자동발동**까지. "뉴스 정리해줘" 한마디에 `naver-news`가 무가드로 외부 발송.

**개선 (2단 방어, instruction-doc 관할 판정)**:

> "복사·배치하는 능력이 **비가역/외부발송/과금**이면 — 라우팅은 그 스킬의 **자동발동을 끄고**(disable-model-invocation, 명시 호출만) + 강제층은 실행 직전 **게이트 훅**을 건다."
> 소유: 자동발동 on/off = 지시층, 실행 게이트 훅 = 강제층. 층으로 안 겹침. line 139를 "안전 능력은 description 유지, 위험 능력은 자동발동 차단+훅"으로 **위험 등급 분기** 추가.

### P0-3 · 인터뷰 우회 = 거짓 보증 세탁 (breach)

line 124·78 되묻기가 "빈칸/애매"에만 트리거 → **위험하지만 자신 있게 꽉 채운 요청서**(프로덕션 키를 프론트 번들에 주입 등)는 체크리스트 4개 다 통과, 콜백 0회. 빌더가 "검토·생성했다"는 **거짓 보증을 단 세탁 단계**로 전락.

**개선**: 되묻기를 "빈칸"이 아니라 **위험 신호**에 트리거. (프로덕션키+공개)·(외부발송+무확인)·(PII+커밋)·(출력경로 밖 삭제) 매칭 시 완성도 무관·우회 없음으로 **인간 확인 강제**(= 아래 G-F).

### P0-4 · 닥터·커맨드 트리거 계약 공백 (instruction)

line 31·104-109는 닥터 4종·`/new-workspace`·`/audit`의 **이름·"담당 결정"만** 있고 **트리거 발화·NOT-trigger 경계가 전무**(line 209에서 전량 Clip 2 위임). 표준 03 실측: description 계약 없는 스킬/카드는 수십 회 위임 기회에 **0회 선택**. instruction-tuner·harness-audit·guard-installer는 표면이 겹쳐 **"점검해줘" 한마디에 누가 뜰지 불명**.

**개선**: 닥터별 `트리거 발화 2-3개 + 산출물 계약 + "단 X는 /other" 경계`를 **설계도에 표로 박기**. (Clip 2 구현이 아니라 설계 결정 자체)

## 4. 강제층 통합 개선 스펙 — 가드 6종 (guard·breach 공동 확정)

> 겹치던 방어 3건을 6종으로 정리하고 중복을 제거. **경계 한 문장**: "무엇이 위험한가 = breach 소유 / 걸리면 어떻게 멈추나 = guard 소유". 시크릿·PII 탐지만 guard.

| 가드                                     | 트리거                         | 판정                                                                   | 차단·복구                   | 되돌림  |
| ---------------------------------------- | ------------------------------ | ---------------------------------------------------------------------- | ---------------------------- | ------- |
| **G-A** inbox 반입 스캔            | PostToolUse (00-inbox 착지)    | 시크릿·PII(guard) + 인젝션·악성훅 서명(breach), 한 훅 두 룰셋        | 격리 + 승격 차단             | X→강제 |
| **G-B** per-run 쓰기 잠금          | PreToolUse (Write·Edit·Bash) | 선언 경로 밖·기존본 존재 차단                                         | 백업 후 우회(GUARD_OK)       | X→강제 |
| **G-C** 스킬복사 매니페스트        | PreToolUse (cp)                | 능력 목록(breach) → 대응 가드(guard),**미지 능력=default-deny** | 미설치 능력 있으면 복사 차단 | X→강제 |
| **G-D** 자식 gitignore+init 순서   | PreToolUse (git init)          | 표준06 §2 순서 강제                                                   | 순서 위반 차단               | X→강제 |
| **G-E** 커밋 PII 게이트(자식 repo) | PreToolUse (add/commit)        | 정밀 grep, 우회 없음                                                   | exit 2                       | X→강제 |
| **G-F** 위험신호 강제 확인 게이트  | PreToolUse                     | 위험 패턴 집합 매칭, 완성도 무관                                       | exit 2 + 인간 확인           | X→강제 |

+ 베이스라인 rm-to-trash 유지 (G-B 덮어쓰기 백업과 상보).

**핵심 교정 2가지**:

- **altitude 교정**: PII 방어를 빌더 자기 repo(원래 위치)가 아니라 **자식 워크스페이스 생성(G-D)+커밋(G-E)** 2겹으로 이동 — inbox 재사용 자료의 시크릿이 자식에 박제되는 걸 막음.
- **default-deny fallback**: breach가 능력 항목을 빠뜨려도 우회로가 안 생기게 "미지 부작용=자동통과 금지".
