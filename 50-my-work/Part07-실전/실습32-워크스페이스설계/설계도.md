## 설계도 다섯 줄

**1. 목적**
대화로 **새 워크스페이스를 설계·점검·생성**하는 빌더(메타하네스).
카탈로그에서 아키타입을 골라 **뼈대 → 지시 층 → 강제 층** 순으로 쌓아, "내 의도대로 동작하는 워크스페이스"를 찍어낸다.
- *절대 안 하는 것*: 실제 업무(리서치·집필·분석 등)는 여기서 하지 않는다. 오직 **"환경을 찍어내는 일"** 하나만.

**2. 아키타입** → **개발(메타)**
빌더 자신은 아키타입 하나. 단 `20-blueprints/`에 **아키타입 카탈로그 10종**을 보유해, 요청에 맞는 아키타입을 골라 찍어낸다. (↓ 부록)

**3. 폴더 초안** (빌더 생애주기: 요청 들어옴 → 설계·점검 → 찍어냄 → 보관)
```
00-inbox/        새 요청·자료 투입 → 빌더가 분석      ★ intake
10-standards/    하네스 원리·표준 (두 층 / 7층, 불변 참조)
20-blueprints/   아키타입 카탈로그 · 설계도/요청서 템플릿 (재사용)
30-drafts/       진행 중 — 설계도 초안 · /audit 점검 리포트   ★ WIP
40-output/       찍어낸 완성 워크스페이스가 나가는 곳
90-archive/      완료된 요청 · 옛 버전
```
> logs 같은 건 따로 두지 않고 30-drafts/90-archive에 흡수 — 과설계 금지(표준 07·ANTI-PATTERNS).

**4. 지시 층** (CLAUDE.md 핵심 규칙 + 배치할 스킬·커맨드)
- **CLAUDE.md**: "워크스페이스는 두 층으로 쌓는다 — 지시 먼저, 강제 나중. 뼈대(목적·폴더)부터, 안전장치는 필요한 만큼."
- 배치: **`/new-workspace`** 커맨드, **하네스 닥터 스킬**(instruction-tuner · guard-installer · harness-audit · breach-test), **`/audit`**

**5. 강제 층** (막을 행동 — 가드 7종, 상세는 하단 "## 검토 반영")
- ~~`40-output/` 밖 파일 차단~~ → **per-run 쓰기 잠금(G-B)**: 이번 run이 선언한 출력경로 1개 + 빌더 작업폴더만 쓰기 허용. **신규 경로면 통과, 기존 경로면 차단** — 이 run이 만든 것이라는 소유 증명 + 인간 승인 있을 때만 우회 (v1의 "밖 절대 차단"은 line 120 독립폴더 출력과 모순 → 폐기 / "백업 후 진행"은 남의 폴더 덮어쓰기 합법화 위험 → 폐기)
- **시크릿·PII 차단**: 빌더 자기 repo가 아니라 **자식 워크스페이스 생성·커밋 시점**에 (inbox 재사용 자료의 키가 자식에 박제되는 것 차단)
- **위험 능력 실행 게이트** + **위험신호 강제 확인** + **inbox 반입 스캔** (가드 7종 참조)

## inbox 사용법 — 무엇을 던져넣나

빌더가 새 워크스페이스를 만들려면 `00-inbox/`에 **요청서 + 재사용 자료**를 넣는다.
요청 하나당 폴더 하나:
```
00-inbox/2026-07-04-리서치볼트/
├─ 요청.md          ← 아래 템플릿
└─ (참고파일들)      ← 재사용할 CLAUDE.md · 샘플데이터 · 브랜드가이드 등
```

**요청.md 템플릿** (`20-blueprints/요청서-템플릿.md`로도 보관)
```markdown
## 워크스페이스 요청서
- 목적:           (이 작업 환경으로 뭘 하나)
- 아키타입 힌트:   (카탈로그 10종 중 짐작 / "모르겠음"도 OK)
- 여기서 할 동작:  (예: 매주 보고서, 자료 축적·검색 …)
- 절대 안 할 것:
- 재사용 자료:     (첨부한 파일 목록)
- 막고 싶은 사고:  (강제 층 힌트)
```

**빌더 처리 흐름**: `00-inbox/` 읽기 → 아키타입 판정 → `30-drafts/`에 설계도 초안 + `/audit` → `40-output/`에 완성본 → 처리된 요청은 `90-archive/`로.

## 레이어 매칭 규칙 — 요구사항 → 알맞은 레이어

> 빌더의 핵심 결정 엔진. "모든 워크스페이스에 같은 레이어"를 깔지 않고, **요청 신호에 맞춰 레이어를 골라 쌓는다.** (Clip 2 닥터 4종의 알맹이)

### ① 요청서 칸 = 레이어 결정 신호

| 요청서 칸 | 뽑히는 신호 |
|---|---|
| 목적 / 안 할 것 | 뼈대 (에이전트 하나의 경계) |
| 아키타입 힌트 | 기본 로드아웃 (카탈로그) |
| 여기서 할 동작 | 반복 절차 → 스킬·커맨드 후보 |
| 도메인 규칙(톤·출처·재현성) | 지시 층 규칙 |
| 막고 싶은 사고 | 강제 층 후보 |

> 칸이 비었거나 애매하면 빌더가 **되묻는다(인터뷰)**. 빈칸 채우기 전엔 안 쌓음.

### ② 레이어별 결정 (아키타입 기본값 + 요구 델타)

| 요구 신호 | 쌓는 레이어 | 강도 |
|---|---|---|
| 목적 · 아키타입 | 뼈대 (폴더 + CLAUDE.md 머리) | 항상 (카탈로그 기본값) |
| 자주·길게 반복하는 동작 | 지시: **스킬** | 짧고 매세션 필요 → CLAUDE.md 한 줄 |
| 품질·스타일 규칙 | 지시: **CLAUDE.md 규칙** | 얇게 (표준 07) |
| 대량 탐색·병렬 작업 | 지시: **서브에이전트** | 증거 있을 때만 (표준 10) |
| 삭제·유출·과금 위험 | 강제: **훅 / 권한** | ↓ 테스트 통과 시 |

### ③ 핵심 판정 — "지시냐 강제냐" 한 문장 (표준 05)

> **"이 규칙을 어기면 되돌릴 수 있나?"**
> - 되돌림 O (품질·스타일·톤) → **지시 층** — CLAUDE.md 한 줄 (부탁)
> - 되돌림 X (삭제·시크릿 유출·외부 발송·과금) → **강제 층** — 훅 (장치)

### ④ 쌓는 순서 & 검증 (표준 08·09)

- **약 → 강**: 뼈대 → 지시 → 강제 → 검증. *훅을 규칙보다 먼저 걸지 않는다.*
- 안전장치는 **처음엔 최소, 사고 나면 늘린다.**
- `/audit`으로 **의도↔결과 갭** 점검 후 빠진 레이어를 되메꾼다 (내비 재안내).

### ⑤ 이 로직이 들어가는 곳 (Clip 2)

| 닥터 / 커맨드 | 담당 결정 |
|---|---|
| `instruction-tuner` | ②의 지시 층 (CLAUDE.md·스킬·서브에이전트·컨텍스트 예산) |
| `guard-installer` | ③의 강제 층 (훅·PII) — "되돌림 X" 판정 |
| `harness-audit` (`/audit`) | ④ 검증·갭 메우기 |
| `/new-workspace` | ①의 신호 읽기 + 뼈대 + 쌓는 순서 |

## 빌드 정의 — 인터뷰로 확정 (2026-07-04)

> 짓기 전에 애매한 정의를 인터뷰로 확정한다 (③ "빈칸이면 인터뷰"의 실제 적용). 확정된 4가지:

| 결정 | 확정 | 빌드에 미치는 영향 |
|---|---|---|
| 상호작용 | 요청서 파일 + 인터뷰 **하이브리드** | inbox 요청서를 읽되, 빈칸·애매하면 되묻고 진행 |
| 출력 위치 | **독립 폴더** | `40-output/`은 완성 후 이관 대기소, 실제 워크스페이스는 별도 경로에 독립 생성 → 클로드코드로 바로 열림 |
| 완성도 | **지시 + 강제 + 스킬**까지 | 뼈대 → CLAUDE.md → 훅/권한 → 스킬 배치. 단 강제는 **아키타입별 비가역 자원 기본값만** 처음에 깔고, 나머지 안전장치는 사고 시 승격 (line 99 "처음엔 최소"와 정합 — 검토 P1) |
| 스킬 재사용 | 라이브러리에서 **복사** | 아키타입이 요구하는 스킬을 내 스킬 라이브러리(16종)에서 복사 배치 |

**빌더가 반드시 되묻는 체크리스트** (하나라도 불명확하면 빌드 중단):
- 목적이 한 문장으로 좁혀지나? (섞이면 둘로 쪼갠다)
- 아키타입이 카탈로그 하나로 정해지나? (모호하면 "할 동작"으로 역추정)
- "막고 싶은 사고"가 비가역인가? (지시 vs 강제 판정 근거)
- 재사용 자료의 위치·형식이 확인되나?

> ⚠️ **되묻기 트리거는 "빈칸"이 아니라 "위험 신호"다** (검토 P0-3). 요청서를 자신 있게 꽉 채워도, 아래 위험 패턴이 잡히면 **완성도 무관·우회 없이 인간 확인을 강제**한다 — 그러지 않으면 빌더가 "검토했다"는 거짓 보증을 다는 세탁 단계가 된다.
> 위험 패턴: (프로덕션키 + 공개) · (외부발송 + 무확인) · (PII + 커밋) · (출력경로 밖 삭제). → 가드 G-F.

## 자연어 우선 활성 — "슬래시 없이 기능 호출"

> 찍어낸 워크스페이스에서 사용자가 `/커맨드` 목록을 외우지 않아도, **말 한마디로 적절한 기능이 떠야 한다.** fastcampus-cc가 이미 쓰는 방식을 빌더의 기본 규칙으로 승격.

**2단 노출 규칙 (표준 03 라우팅):**
1. **1순위 — 기능은 되도록 스킬로 만든다.** description에 트리거 문구를 풍부하게 넣으면 의도만 말해도 자동 호출(슬래시 불필요). 예: "여행 코스 짜줘" → trip-advisor 자동 발동.
2. **2순위 — 커맨드로 남아야 하면 자연어 트리거 표로 감싼다.** CLAUDE.md에 `트리거 문구 → 커맨드` 표를 둔다. 예: "막혔어요" → /stuck.
3. **슬래시는 "단축키"일 뿐, 유일한 문이 아니게 한다.**
4. **⚠️ 예외 — 위험 능력은 자동발동에서 뺀다** (검토 P0-2). 비가역·외부발송·과금 스킬은 description을 풍부하게 두되 **자동발동을 끄고(disable-model-invocation) 명시 호출만** 허용한다. "자동 편의"와 "안전"이 충돌하면 안전이 이긴다. 실행 자체는 강제층 게이트 훅(G-C)이 최후로 막는다 — **2단 방어**.

→ 빌더는 각 아키타입을 찍을 때 이 2단 규칙을 **CLAUDE.md 기본 골격에 심고**, 배치한 스킬·커맨드마다 **자연어 트리거를 자동 생성**한다.

## 부록 — 아키타입 카탈로그 (10종)

각 아키타입은 "무엇이 들어와서(입력) → 무엇이 나가나(출력)"로 구분하고,
찍어낼 때 **폴더 골격 + 지시 층 기본 + 강제 층 기본**이 세트로 깔린다.
(실제 템플릿 파일 채우기는 Clip 2 빌더에서 — 여기선 골격만 정의)

### 핵심 6종

| 아키타입 | 정체 (입력 → 출력) | 폴더 골격 | 지시 층 기본 | 강제 층 기본 |
|---|---|---|---|---|
| 💻 **개발** | 요구사항 → 동작하는 코드 | `src / tests / docs / archive` | 코드 컨벤션 | 시크릿 커밋 차단 · main 직push 방지 · 커밋 전 테스트 |
| 🔍 **리서치** | 질문 → 출처 붙은 보고서 (일회성) | `sources / notes / reports` | 출처 등급 · 교차검증 | 출처 없는 단정 경고 |
