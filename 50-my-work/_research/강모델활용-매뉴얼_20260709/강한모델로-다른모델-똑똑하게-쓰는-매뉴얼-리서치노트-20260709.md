# 리서치 노트 — 강한 모델로 다른 모델을 똑똑하게 쓰기

> 보고서와 짝. 보고서="무엇을 찾았나", 이 노트="어떻게 찾고 무엇을 판단했나". 작성 2026-07-09.

## 1) 리서치 설계 (⓪ 범위·목적·기준)

- **범위**: 강한 모델(Fable 5)로 다른/저렴한 모델을 더 잘 쓰는 기법군.
- **목적**(AskUserQuestion 응답): 자산 생성 + 멀티에이전트 오케스트레이션 + 타사 모델 포함 + 비용 최적화 = **4개 다** 선택 → 범용 종합.
- **적용 대상**: Claude Code + API 둘 다 / 일반 활용.
- **기준**: 재현 가능한 핵심 기법 위주, 각 기법을 Fable/Claude 구체 기능에 매핑, 과장 X-post는 근거로 안 씀.
- **결과 형식**: 마크다운 "매뉴얼"(고정).
- **초기 판단**: "Fable 5"라는 브랜드 자체를 검색하면 실질 근거가 얕음 → **저변의 검증된 기법군**(어드바이저/오케스트레이션/라우팅/메타프롬프팅·distillation)을 검색하고 Fable/Claude 기능에 연결하는 전략 채택.

## 3) 교차검증 판단 (채택/제외 근거)

- **어드바이저 패턴**: 공식 문서(A) + claude-api 내장 스킬(A) + 다수 블로그(C)가 **일치** → 패턴·모델호환은 A로 채택. 단, "19.7→41.2%·85%↓" 구체 수치는 **공식 문서에서 재확인 실패**(공식은 nudge +7%p 등 다른 튜닝 수치만) → 해당 숫자는 **C(스니펫 의존)**으로 강등 표기, "내 워크로드에서 실측" 한계 명시.
- **멀티에이전트 90.2%/15배**: Anthropic 자사 블로그 단일 출처지만 공식 → A/B로. "자사 eval·코딩엔 부적합" 조건을 한계에 그대로 반영(네거티브 채택).
- **라우팅 45~85%**: 실무 블로그 + arxiv 스니펫이 방향 일치하나 수치 편차 큼 → C, "참고치" 표기.
- **distillation 수치**: 파인튜닝·오픈소스 소형모델 대상 연구 → 개인 Claude 활용과 결이 달라, **few-shot/프롬프트 자산화 부분만** 매뉴얼에 반영하고 파인튜닝은 한계로 분리.
- **제외**: "Fable 5" 브랜드 직검색 결과의 개별 X-post류(검증 불가·과장) → 근거 미채택, 저변 기법으로 대체.

## 후속 리서치 (2026-07-09) — 타사 혼합 라우팅 + "모델 자동선택 훅" 아이디어 검토

fetch 3건(성공, 차단 없음):
- ✅ openrouter.ai 모델 라우팅 → 2층 구조(모델선택 vs 프로바이더선택), 역제곱 가격 가중, auto-router(Not Diamond, 0~10 비용-품질 다이얼), 폴백체인(프로바이더+`models` 배열), 실패요청 무과금. (B/C)
- ✅ techsy.io Claude Code Router → 로컬 프록시, `config.json` 슬롯매핑(haiku→싼모델·sonnet→Sonnet·opus→Opus), 태스크기반 자동 분배. **Claude Code 사용자에게 가장 직접 적용.** (C)
- ✅ portkey.ai 페일오버 → 폴백체인(GPT 503→Claude), 상태코드 라우팅(429 재라우팅/500 재시도/가드레일→대체모델), 지연 임계 헤징(2초), 70/30 프로액티브 분산, 조건부 라우팅. (C)

### 교차검증 판단
- 라우팅 85%↓ 수치는 **RouteLLM(UC버클리, ICLR 2025)** 이 1차 근거로 확인 → 이전 보고서의 C 등급을 이 항목만 **B**로 상향 가능.
- "silent quality regression(싼 모델로 바꿀 때 조용한 품질 저하)" + "pre-merge eval gate 50~500케이스" 는 복수 실무 블로그가 일치 → 검증 방법론으로 채택(C).
- 태스크별 강점(코딩=Claude / 순수추론=GPT o-series / 최장컨텍스트·멀티모달=Gemini 1M)은 다수 비교글 일치하나 벤치 편차 커서 C, "성향"으로만 기술.

### 사용자 "모델 자동선택 훅" 아이디어 판단 (핵심)
- 방향(업무→최적모델 매칭)은 라우팅 그 자체 = **맞는 직관**.
- 단, **Claude Code 훅은 '지정(모델 전환)'엔 부적합** — 훅은 컨텍스트 주입·차단·조언·로깅은 되지만 세션 모델을 태스크별로 바꾸진 못함(현 이해, Part05 Hook 클립으로 검증 권장).
- '지정'의 올바른 도구: **Claude Code Router**(config.json 자동 라우팅) / **서브에이전트 `model` 핀** / (API) LiteLLM·OpenRouter·Not Diamond.
- '검증': 실시간 자기확신도는 부정확(앞선 라운드 근거) → **오프라인 eval 게이트(50~500케이스)** 로 규칙 정할 때 1회 검증이 정석.
- 결론 = **하이브리드**: 라우터/서브에이전트가 '지정', 훅은 '분류·조언·로깅'.
