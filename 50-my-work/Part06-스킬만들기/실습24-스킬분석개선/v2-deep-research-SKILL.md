# deep-research v2 — 딥리서치 보고서 스킬

주제 하나를 받아 **출처가 달린 마크다운 보고서 한 장**으로 정리한다.
검색만 하는 얕은 리서치가 아니라, 유망 URL을 골라 **본문을 직접 가져와(fetch)** 읽고
서로 다른 출처를 **교차검증**한 뒤, 주장마다 **신뢰도 등급**을 달아 보고서를 쓴다.

> v1(레퍼런스) 대비 변경점: ④ 출처 신뢰도 등급(A~E) 도입, ③ 로컬 주제 본문 fetch 필수화,
> ⑤ 수록 항목 상한(과밀 방지), 대형 주제 한정 경량 진행 메모. 흐름과 경량성은 v1을 유지한다.

---

## ⓪ 범위 확인 — 모호하면 먼저 묻는다

주제를 받으면 **범위 / 목적 / 기준** 3개가 명확한지 점검한다.
모호한 항목만 `AskUserQuestion`으로 묻는다. **혼자 가정하고 검색을 시작하지 않는다.**

- 결과형식은 **마크다운 보고서로 고정** — 묻지 않는다.
- 3개가 이미 주제에 다 드러나면 질문 생략하고 ①로.
- **목적 미입력(질문 건너뜀) 시 → "범용 종합" 기본 골격으로 안전 동작.** *(v2 추가: 기본값 보장)*

---

## ① 보고서 뼈대 잡기

⓪에서 좁혀진 범위로 **최종 보고서 목차를 먼저** 만든다. 이 뼈대가 검색의 길잡이.
(예: 지역별 / 기준별 / 후보 리스트 / 한계)

---

## ② 키워드 분해 + 검색 쿼리

**분해** — 주제를 3~5개 축으로 (넓은 축 → 좁은 축, 겹치지 않게, ⓪ 답변 근거 축만).
**쿼리** — (1) 넓은 쿼리 → (2) 축×축 조합 → (3) 검증용 쿼리.
한국 주제는 **검색어를 한국어로** 명시(WebSearch는 미국 기준).

---

## ③ 반복 검색·요약 루프 ⟲

```
③-a  WebSearch   후보 URL + 스니펫 수집
③-b  URL 선별    깊이 볼 URL 2~4개 (출처 종류 섞기: 블로그·뉴스·공식)
③-c  WebFetch    고른 URL 본문 가져와 핵심 추출
```

**★ v2 추가 — 로컬/실용 주제는 본문 fetch를 "기본"으로 한다.**
맛집·장소·가격·영업시간처럼 **스니펫에 안 나오는 디테일**이 핵심인 주제는,
다이닝코드·네이버·공식 프로필 등 **1차 정보원 본문 fetch를 선택이 아닌 필수 단계**로 돈다.

**루프 종료(포화 기준, 카운터 X)**: ①새 정보<20% ②핵심 주장마다 출처 2개+ ③상한(기본4/깊게6) 중 하나.
막히는 사이트(로그인·봇차단·JS)는 건너뛰고 다른 URL로 대체.

---

## ④ 교차 검증 + 신뢰도 등급 ★ (v2 강화)

가져온 본문들을 **서로 대조**한다.
- 충돌 정보 식별 → 재확인하거나 "확인 필요" 표시 (예: 위치 모순 발견 시 해소 과정을 기록).
- 한 출처에만 있는 주장은 다른 출처로 재확인.

**★ v2 추가 — 주장/출처에 신뢰도 등급(A~E)을 단다.** *(gptaku에서 차용)*

| 등급 | 의미 | 예 |
|---|---|---|
| A | 공식·1차 데이터 | 정부·공공기관·공식 사이트 |
| B | 신뢰 매체·플랫폼 본문 | 다이닝코드 프로필, 주요 언론 |
| C | 일반 매체·블로그 | 여행·맛집 기사 |
| D | 기업/홍보 글 | 자사 블로그 |
| E | 일화·SNS | 커뮤니티 후기 |

단일 출처거나 본문 확보 실패면 표/항목에 **그 사실을 명시**한다.

## ⑤ 마크다운 보고서 + 출처

①에서 짠 뼈대에 검증된 내용을 채워 **보고서 한 장**으로. 한국어, **시각화 없음.**

보고서 구성:
1. **핵심 한 줄 요약**
2. **카테고리별 표** (뼈대 목차대로) — ★ 표에 **신뢰도 등급 열** 포함
3. **출처 목록** — 모든 주장 추적 가능 (마크다운 링크 + 등급)
4. **한계** — 정보 기준 시점, 못 가져온 출처, 확인 필요 항목, 단일출처 항목

**★ v2 추가 — 수록 항목 상한**: 한 표/카테고리당 과밀 방지(예: 가게 20~30곳 이내). 1장 가독성 유지.

저장: 작업 폴더(또는 지정 위치)에 `{주제}-{날짜}.md`.

## 현실 한계 (보고서 "한계"에 반영)

- WebSearch 미국 기준 → 한국어 검색어 명시, 부족분 fetch 보완.
- WebFetch 차단 가능(로그인·봇·JS) → 다른 URL로 대체, 보고서에 명시.
- 실시간성 한계 → "정보 기준 시점" 명시, 방문/사용 전 재확인 권장.
