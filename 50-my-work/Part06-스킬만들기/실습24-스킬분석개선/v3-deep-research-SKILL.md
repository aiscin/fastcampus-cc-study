# deep-research v3 — 딥리서치 보고서 스킬

주제 하나를 받아 **출처가 달린 마크다운 보고서 한 장**으로 정리한다.
검색만 하는 얕은 리서치가 아니라, 유망 URL을 골라 **본문을 직접 가져와(fetch)** 읽고
서로 다른 출처를 **교차검증**한 뒤, 주장마다 **신뢰도 등급**을 달아 쓴다.

산출물은 **두 개** — ⑤ 보고서(결과)와 ⑥ 리서치 노트(과정 기록). 노트는 빼먹지 말 것.

> 이 흐름은 "어떻게 워크플로우를 구성해야 할까?"를 먼저 조사한 뒤 사용자가 보강해 정의한 것이다. 단계를 임의로 건너뛰지 말 것.

> **v3 변경점**: ④ 신뢰도 등급(A~E) · ③ 로컬 주제 본문 fetch 필수 · ⑤ 수록 상한 · ⓪ 기본값 보장 ·
> **「현장 메모」 살아있는 로그** · **결정론 반복 → scripts 분리 트리거** · fetch 예산.

---

## ⓪ 범위 확인 — 모호하면 먼저 묻는다

**범위 / 목적 / 기준** 3개가 명확한지 점검. 모호한 것만 `AskUserQuestion`으로 묻는다. **혼자 가정하고 시작하지 않는다.**

- 결과형식은 **마크다운 보고서로 고정** — 묻지 않는다.
- 3개가 이미 드러나면 질문 생략하고 ①로.
- **목적 미입력(질문 생략) 시 → "범용 종합" 기본 골격으로 진행(멈추지 않음).** *(v2)*
- 동일 주제 재리서치면, 직전 노트의 "미해결/다음 보강"을 **먼저 로드**해 출발점으로 삼는다. *(v3, ⑥과 연결)*

---

## ① 보고서 뼈대 잡기

⓪ 범위로 **최종 목차를 먼저** 만든다. (예: 지역별 / 기준별 / 후보 리스트 / 한계)
- **좁은 주제 → 단일 표** 유지. **넓은 주제만 권역/섹션 골격** 사용. *(v3, 과설계 방지)*

---

## ② 키워드 분해 + 검색 쿼리

**분해** — 3~5개 축(넓은→좁은, 안 겹치게, ⓪ 근거 축만).
**쿼리** — (1) 넓은 쿼리 → (2) 축×축 조합 → (3) 검증용 쿼리.
한국 주제는 **검색어를 한국어로** 명시(WebSearch 미국 기준).

---

## ③ 반복 검색·요약 루프 ⟲

```
③-a WebSearch  후보 URL+스니펫 수집
③-b URL 선별   깊이 볼 URL 2~4개 (블로그·뉴스·공식 섞기)
③-c WebFetch   본문 가져와 핵심 추출
```

**★ 로컬/실용 주제는 본문 fetch가 "기본"** *(v2)* — 맛집·장소·가격·영업시간처럼 스니펫에 없는 디테일이 핵심이면, 프로필성 1차 정보원(다이닝코드·네이버 등) 본문 fetch를 **필수 단계**로 돈다.

**★ 결정론 반복은 스크립트로 분리** *(v3, 강의 원칙)* — 같은 형태의 추출(예: 프로필 URL → 영업시간·가격·평점)이 **반복되면** `scripts/fetch_profile.py`로 빼서 같은 입력=같은 출력 보장. *단, 반복이 실제로 생긴 뒤 분리(미리 만들면 과설계).*

**루프 종료(포화, 카운터 X)**: ①새 정보<20% ②핵심 주장마다 출처 2개+ ③상한(기본4/깊게6) 중 하나.
**fetch 예산** *(v3)*: 한 바퀴 2~4개 + 세션 총 12개 상한(폭주·비용 방지).
막히면 → 다른 URL(다이닝코드 프로필 등) 대체 → 끝내 스니펫만이면 **"스니펫 의존" 표시** + 다른 출처로 1회 교차.

## ④ 교차 검증 + 신뢰도 등급 ★ *(v2)*

본문들을 **서로 대조**한다.
- 충돌은 표시로 끝내지 말고 **해소 시도**: 그 항목만 겨냥한 검증 쿼리 1회 → 안 풀리면 "확인 필요" 명시.
- 핵심 후보가 단일 출처 → 추가 검색 1회로 2번째 출처 탐색, 못 찾으면 "교차검증 미완료" 태그.

**신뢰도 등급(A~E)** — 주장/출처에 등급을 단다:

| 등급 | 의미 | 예 |
|---|---|---|
| A | 공식·1차 데이터 | 정부·공공기관·공식 사이트 |
| B | 신뢰 매체·플랫폼 본문 | 다이닝코드 프로필, 주요 언론 |
| C | 일반 매체·블로그 | 여행·맛집 기사 |
| D | 기업/홍보 글 | 자사 블로그 |
| E | 일화·SNS | 커뮤니티 후기 |

## ⑤ 마크다운 보고서 + 출처

①에서 짠 뼈대에 검증된 내용을 채워 **보고서 한 장**으로. 한국어, **시각화 없음.**

구성: ① 핵심 한 줄 요약 / ② 카테고리별 표(**「신뢰도」 열 포함**) / ③ 출처 목록(링크+등급) / ④ 한계(기준 시점·미확보·확인 필요).
**수록 항목 상한**: 카테고리당 20~30개 이내(1장 가독성). *(v2)*
저장: 작업 폴더(또는 지정 위치)에 `{주제}-{날짜}.md`.

---

## ⑥ 리서치 노트 (항상 함께)

보고서와 **별도 파일**로 *과정*을 남긴다. 보고서="무엇을 찾았나", 노트="어떻게 찾고 무엇을 판단했나".

구성: 1) 리서치 설계(⓪ 범위·목적·기준) 2) 검색 로그(쿼리별 수확 + fetch 성공·차단) 3) 교차검증 판단(채택/제외 근거, 네거티브 포함) 4) **미해결/다음 보강**(← 다음 동일주제 리서치 시 ⓪ 직후 로드) 5) 한 줄 회고.
저장: 보고서와 같은 폴더에 `{주제}-리서치노트-{날짜}.md`.
