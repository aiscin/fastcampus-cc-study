# 두 딥리서치 결과물 — 정확히 무엇이 달랐나

> 주제: "서울 김밥 맛집" (동일) · 비교 시점 2026-06-22
> A = 기존 gptaku 스킬 결과 (`서울김밥맛집_20260622/`)
> B = 레퍼런스(직접) 스킬 결과 (`서울-유명-김밥집-2026-06-19.md`)

---

## 1. 가게 커버리지 — 다룬 가게 자체가 달랐다

**A가 다룬 가게(고유명): 5곳** · **B가 다룬 가게(고유명): 약 20곳**

| 구분 | 가게 |
|---|---|
| **양쪽 공통 (4곳)** | 오월의김밥 · 한입소반 · 키친봄날 · 수아당 |
| **A에만 있음 (1곳)** | **모녀김밥(광장시장)** ← A의 간판 사례 |
| **B에만 있음 (16곳)** | 통통김밥 · 식물원김밥 · 루비떡볶이 · 연희김밥 · 서호김밥 · 해남원조김밥 · 호랑이김밥 · 올바른김밥 · 채식주의자 · 오토김밥 · 그집김밥 · 오미마리 · 커피가머무르는곳 · 뽁 · 보슬보슬 · 우엉 |

➡️ **핵심 차이 1**: A는 "광장시장 마약김밥(모녀김밥)"을 중심에 뒀지만, B가 발굴한 **화제성 1군(통통김밥·식물원김밥·연희김밥)을 A는 통째로 누락**. 반대로 B는 광장시장/모녀김밥을 다루지 않음. → 같은 주제인데 **겹치는 가게가 4곳뿐**.

---

## 2. 같은 가게라도 — 정보 깊이가 달랐다

공통 4곳 중 대표 2곳 비교:

| 가게 | A (기존 gptaku) | B (레퍼런스) |
|---|---|---|
| **오월의김밥** | "낙성대, 팔뚝 김밥, 08~14시·포장 전용, 재료 소진 마감" | "낙성대역 1번출구 도보 1분, 08:00~14:00, **월 휴무**, 재료 소진 마감, 테이크아웃 전용, 밥도둑·샐러드김밥" + **다이닝코드 프로필 출처** |
| **수아당** | "성신여대, 육회·치즈 등 이색 속재료, 키토김밥" | "성북구 성신여대, **마당/우엉/향버섯(비건)/육회** 메뉴, 다양한 야채 골라먹기, **카페 겸업**" |

➡️ **핵심 차이 2**: B는 **휴무일·정확한 동선·메뉴명·겸업 여부**까지. A는 같은 가게도 한 줄 요약 수준. (B가 다이닝코드 본문을 fetch한 결과)

## 3. 조직 기준 — 묶는 축이 달랐다

| | A (기존 gptaku) | B (레퍼런스) |
|---|---|---|
| 1차 분류 | **지역(권역)별** — 종로/관악/용산/서대문/성북/성수 | **목적·기준별** — 화제성 / 노포 / 프리미엄 / 가성비 |
| 결과 성격 | "어디에 무엇이 있나" 지도 | "콘텐츠로 어떻게 쓰나" 기획안 |
| 부가 산출 | 없음 | **콘텐츠 제작 앵글 5종** (청와대 500줄·오전한정 등) |

➡️ **핵심 차이 3**: A는 위치 중심, B는 **쓸 목적(콘텐츠 제작)에 맞춰** 묶음. B는 처음에 "목적·기준"을 물었기 때문.

## 4. 검증 방식 — 어디까지 파고들었나

| | A (기존 gptaku) | B (레퍼런스) |
|---|---|---|
| 주 방법 | WebSearch 스니펫 위주 (WebFetch 1건) | WebSearch + **다이닝코드 프로필 본문 fetch 다수** |
| 가격 정보 | 개략 ("6줄 3천 원대 / 5~9천 원") | **가게별 개별 가격** (연희 매운오징어 5,000원, 보슬보슬 9,500원 …) |
| 정보 충돌 | 없음 | 식물원김밥 위치 모순 **발견 → 2지점으로 해소** 기록 |
| 출처 신뢰도 | **A~E 등급 표기** | 등급 없음, "본문 차단/단일출처"는 명시 |

➡️ **핵심 차이 4**: A는 **신뢰도 라벨(A~E)**이 강점, B는 **본문까지 파서 디테일·충돌 해소**가 강점.

## 5. 결과물 형태

| | A (기존 gptaku) | B (레퍼런스) |
|---|---|---|
| 형태 | 폴더 패키지(state.json·요약·5섹션·참고문헌) | 단일 .md 1장 |

➡️ **핵심 차이 5**: A는 절차·재현·재개가 강하지만 무겁고, B는 1장으로 가볍지만 상태관리는 없음.
