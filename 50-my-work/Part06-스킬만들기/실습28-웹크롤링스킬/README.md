## 무엇을 했나

1. **웹 크롤링 (insane-search)** — API 없는 KREAM에서 봇 차단을 뚫고 상품 수집
   - `curl_cffi` + **safari TLS 위장** + self-referer → 첫 시도에 strong_ok (Playwright 불필요)
   - 키워드 하나로 남/여 인기순 URL 2개 조립 → 6컬럼(브랜드·상품명·가격·관심·리뷰·거래) 추출
2. **전문가 팀 토론 (kkirikkiri Agent Teams)** — 패션·트렌드 전문가 4인
   - MD/바이어 · 트렌드 애널리스트 · 소비자행동 전문가 + 적대 검토(크리틱, build≠review)
   - "비교 분석 가능한 것 / 종합 분석 가능한 것" 도출
   - **크리틱이 핵심 착시 적발**: "남녀 거래량 동일=젠더리스"는 발견이 아니라 데이터 항등식
3. **트렌드 리포트** — 검증된 유효 분석만 담은 다크 에디토리얼 HTML

## 핵심 배운 점 — 데이터 구조부터 확인

KREAM의 관심·리뷰·거래는 **상품별 누적 총량**이라 성별로 분리되지 않는다. 정렬(남/여 인기순)만 다를 뿐 같은 상품의 숫자는 동일. 그래서:
- **성별 신호로 유효한 건 ① 순위 위치 ② TOP50 등장 구성(집합)** 둘뿐
- 남녀 전환율·퍼널 비교는 불가 (여성 관심/리뷰도 결국 상품 총량)
- 화려한 "젠더리스/전환율" 해석은 대부분 데이터 착시였음 → **적대 검토가 막아줌**

→ 혼자 한 번에 분석했으면 자신 있게 틀렸을 결론. **토론·검증을 끼운 값어치.**
