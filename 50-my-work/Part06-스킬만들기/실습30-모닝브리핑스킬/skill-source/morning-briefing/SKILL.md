# morning-briefing — 아침 한 번에 오늘 파악

> 메일·일정·뉴스 3소스를 모아 **짧게** 브리핑하고 md로 남긴다.
> 스킬이 다른 스킬(daily-news)을 부르는 구조.
> 결정론 영역(메일·일정 수집 + 광고 필터)은 `scripts/gather.py`가 처리하고,
> 이 문서(AI)는 **답장 우선순위 판단·요약·톤 조정**만 맡는다.

## 사전 준비 (처음 한 번만)
- **gws CLI + Google OAuth** — Part 5 실습20에서 연동 완료. `which gws`로 확인.
- **네이버 검색 API 키** — 프로젝트 루트 `.env` (daily-news와 공용).

## 무엇을 고정하고 무엇을 맡기나
매일 같아야 하는 것(어떤 API를 어떻게 부를지, 광고 필터 규칙)은 스크립트에 박혀 있다.
그날그날 달라지는 판단(어떤 메일부터 답할지, 뉴스 중 뭐가 중요한지, 어떻게 요약할지)만 AI가 한다.
그래서 매일 같은 모양으로 나오면서 내용은 오늘에 맞는다.

## 실행 흐름

### STEP 1 — 메일·일정 수집 (결정론, 스크립트)
```bash
cd <프로젝트 루트>
python3 .claude/skills/morning-briefing/scripts/gather.py --mail-limit 10
```
출력 JSON: `weather(오늘 날씨), unread_total(안 읽은 총 개수), mails[](광고 제외 최근 N건), ads_filtered(거른 광고 수), events[](오늘 일정), errors[]`.
각 mail: `from, subject, date, snippet`. 각 event: `time, summary, location`.
`weather`: `now_temp, high, low, condition, rain_prob`. 날씨 위치는 기본 서울 — `gather.py`의 `WEATHER_LAT/LON`으로 변경. (Open-Meteo 무료 API, 키 불필요)
`is_monday`: 오늘이 월요일이면 true. `week_events`: 월요일일 때만 이번 주(월~일) 전체 일정. `conflicts`/`week_conflicts`: 시간이 겹치는 일정 쌍(자동 감지).

**에러 처리** — `errors[]`에 "gws 인증/네트워크" 메시지가 있으면, 브리핑을 멈추고
사용자에게 "구글 연동(gws)이 만료됐을 수 있어요 — 재인증이 필요합니다"라고 안내한다.
메일만 실패하고 일정은 오면, 되는 것만 브리핑하고 실패한 소스는 한 줄로 알린다.

### STEP 2 — 뉴스 (daily-news 스킬 재호출)
daily-news 스킬을 호출해 오늘 종합 뉴스 5건을 받는다(스킬이 스킬을 부르는 지점). 내부적으로:
```bash
set -a; source .env; set +a
python3 .claude/skills/daily-news/scripts/fetch_daily_news.py --top 5
```
뉴스 수집이 실패해도 브리핑 전체를 멈추지 않는다 — "오늘 뉴스는 못 가져왔어요" 한 줄로 대체한다.

### STEP 3 — 정리·판단 (AI 영역)
1. **날씨 + 옷차림 한 줄** — `weather`의 기온·강수확률을 보고 옷차림을 한 줄로 표현한다(예: "낮 28도 맑음, 반팔 OK / 소나기 대비 우산"). 숫자는 스크립트가 준 것 그대로, 옷차림 표현만 AI가.
2. **답장 우선순위 TOP 3** — `mails`에서 발신자·제목으로 "오늘 답해야 할 것"을 최대 3건 골라 앞에 세운다.
   견적·계약·긴급·상사/거래처 신호를 우선한다. 나머지는 아래에 목록으로.
3. **일정** — 시간순. 일정이 0건이면 "오늘 잡힌 일정은 없어요"라고 명확히.
   - **주간 일정(월요일만)** — `is_monday`가 true면 `week_events`로 이번 주(월~일) 일정을 요일별로 한 번 정리한다. "이번 주 미리 보기" 블록으로. 화~일요일엔 이 블록을 넣지 않는다.
   - **일정 겹침 알림** — `conflicts`(월요일은 `week_conflicts`도)가 비어있지 않으면 "⚠️ 일정이 겹쳐요"로 겹치는 두 일정과 시각을 보여주고 "하나를 옮기거나 시간 조정이 필요해요"라고 제안한다. 겹침이 없으면 이 알림을 만들지 않는다(없는데 있다고 하지 말 것).
4. **메일 → 일정 등록 제안** — `mails`의 제목·`snippet`에서 날짜·시간·미팅·마감·약속 신호가 보이면, "이 메일은 일정 등록이 필요해 보여요"로 최대 3건 제안한다(제목 + 추정 일시 + "캘린더에 추가할까요?"). 신호가 애매하면 넣지 않는다 — 억지 제안 금지.
5. **뉴스** — daily-news가 준 5건을 한 줄씩.

### STEP 4 — 출력 (대화 + md 저장)
**대화창**에 아래 순서로, **짧고 친근하되 군더더기 없이**(인사말 남발 금지, 존댓말 유지):
```
☀️ 오늘의 브리핑 (YYYY-MM-DD 요일)

한 줄 요약: {날씨 · 첫 일정 · 답장할 메일 N건 · 뉴스 핵심 1개}   ← 5초 안에 파악되게 맨 위

🌤️ 날씨: {현재 X도 · 낮 최고 Y / 최저 Z · 상태} — {옷차림 한 줄}

📧 답장 먼저 (안 읽은 메일 총 {unread_total}건 · 광고 {ads_filtered}건 제외)
  ⭐ 1. {발신자} — {제목} : {왜 우선인지 한 줄}
  ⭐ 2. ...
  · 그 외: {제목들 간단히}

📅 오늘 일정 ({N}건)
  - {시간} {일정} {장소}

📰 오늘 뉴스
  1. {제목 핵심} — {한 줄}  (N개 매체)
  ...
```
그다음 md로 저장:
- 경로: `50-my-work/Part06-스킬만들기/실습30-모닝브리핑스킬/brief-{YYYY-MM-DD}.md` (폴더 없으면 `mkdir -p`).
- **멱등성** — 같은 날짜 파일이 이미 있으면 덮어쓴다(하루 여러 번 호출해도 한 파일). 저장 후 경로 한 줄만 알린다.

## 자동 실행 미리보기 (선택)
Part 5 SessionStart hook을 변형하면 아침 첫 세션에 이 브리핑을 자동 안내할 수 있다.
단 무인 실행은 AskUserQuestion을 못 쓰므로 옵션을 `.env`에 고정하고, 같은 날 재실행 시 기존 md가 있으면 건너뛴다.

## Settings (가변 요소)
| 설정 | 기본값 | 변경 방법 |
|------|--------|-----------|
| 메일 건수 | 10 | `gather.py --mail-limit N` |
| 뉴스 건수 | 5 | daily-news `--top N` |
| 광고 필터 규칙 | 발신자·제목·프로모션 카테고리 (보안/결제/배송 키워드는 보존) | `scripts/gather.py`의 `AD_*`/`KEEP_HINT` |
| 저장 위치 | 실습30 폴더 | STEP 4 경로 |

## Scripts
- **`scripts/gather.py`** — Gmail 안 읽은 메일(광고 필터) + 오늘 캘린더 일정을 gws CLI로 수집. 광고 오탐 방지(보안/결제/배송 발신은 보존).

## 연계 스킬
- **daily-news** — 뉴스 칸 재료 (STEP 2에서 재호출).
