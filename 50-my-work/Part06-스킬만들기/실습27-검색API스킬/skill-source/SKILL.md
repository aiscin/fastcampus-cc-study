# naver-news — 반도체 종목 일일 뉴스레터

네이버 검색 API로 모은 뉴스를 **종목별 호재/악재 브리핑 HTML**로 만든다.
결정론 영역(수집·기간계산·태깅·정제·등급)은 `scripts/fetch_news.py`가 처리하고,
이 문서(AI)는 **관련성 최종 판단·호재/악재 분류·요약·HTML 디자인**만 맡는다.

> ⚠️ 본 뉴스레터는 공개 뉴스를 AI로 요약한 **정보 제공용**이다. 투자 권유·매수/매도 추천이 아니다.

---

## 사전 준비 (처음 한 번만)

**1. 네이버 검색 API 키 발급** — [developers.naver.com](https://developers.naver.com)
   - 로그인 → Application → 애플리케이션 등록
   - **사용 API = '검색'** 선택 / 비로그인 오픈 API **'WEB 설정'** / URL `http://localhost`
   - 발급된 **Client ID + Secret** 확보

**2. `.env` 파일** — 프로젝트 루트(이 워크스페이스 최상위 폴더)에 `.env` 생성/추가:
   ```
   NAVER_CLIENT_ID=발급받은_ID
   NAVER_CLIENT_SECRET=발급받은_Secret
   ```
   - 따옴표 없이 값만 붙여넣는다. `.env`는 git에 올라가지 않는다(.gitignore).

키가 없거나 틀리면 `fetch_news.py`가 `has_error: true`로 알려준다 → STEP 1 참조.

**3. (권장) 공휴일 판정 라이브러리** — 정확한 휴장일 자동 판정용:
   ```
   python3 -m pip install --user -r .claude/skills/naver-news/requirements.txt
   ```
   미설치여도 동작하지만 공휴일을 못 거른다(주말·근로자의날·연말폐장만 판정).

## 워치리스트 (외부 config)

종목·매크로 키워드는 **`watchlist.json`** 에 있다. 종목을 바꾸려면 **이 파일만** 수정한다(코드 수정 불필요).
```json
{ "watchlist": [ {"code":"098120","name":"마이크로컨텍솔"}, ... ],
  "macro_keywords": ["반도체 현물가", ...], "macro_max_items": 20 }
```
현재: 마이크로컨텍솔(098120)·씨엠티엑스(388210)·이노테크(469610).

## 실행 흐름

### STEP 0 — 호출 방식 판별 (수동 vs 자동)
- **수동 호출**(사람이 직접 요청): AskUserQuestion 2단으로 옵션을 묻는다.
  - Q1 정렬: 최신순(date) / 관련도순(sim)
  - Q2 기간: 자동(요일 기반) / 전일만(1일) / 주말포함(3일)
- **자동 루틴**(무인 스케줄 실행): AskUserQuestion 못 씀 → **`--sort date` + 기간 자동** 고정값으로 직행.

### STEP 1 — 뉴스 수집 (결정론, 스크립트)
```bash
cd <프로젝트 루트>
set -a; source .env; set +a
python3 .claude/skills/naver-news/scripts/fetch_news.py [--sort date|sim] [--days N]
```
출력 JSON: `market_closed, market_closed_reason, holiday_check, period_label, total_unique, stocks[](name/code/items), macro[], errors[], has_error`.
각 item: `title, summary, link, pubDate, hours_ago, source_domain, media_grade(A/B/C), unverified, tags`.

**휴장일 처리(필수)** — JSON의 `market_closed`가 `true`이면 (주말·공휴일·근로자의날·연말폐장):
- **자동 루틴**: 발송하지 않고 종료한다 (장이 안 열린 날은 종목 뉴스가 의미 없음). `market_closed_reason`을 로그로 남긴다.
- **수동 호출**: 사용자에게 "오늘은 {사유}로 증시 휴장입니다 — 그래도 만들까요?"라고 확인 후 진행.
- `holiday_check`가 `"weekday-only-fallback"`이면 holidays 라이브러리 미설치 상태 → 공휴일을 못 거른 것. 사용자에게 "정확한 공휴일 판정을 위해 `pip install -r requirements.txt`를 권장"한다고 한 줄 안내.

**에러 처리(필수)** — JSON의 `has_error`가 `true`이거나 `errors[]`가 비어있지 않으면:
- 발행 전에 사용자에게 **원인과 해결책을 한국어로 먼저 안내**한다 (키 오류/호출한도/네트워크 등 errors 메시지 그대로 번역).
- exit code 1(데이터 0건 + 에러)이면 수집 자체가 실패한 것 — 발행하지 말고 중단.
- 일부 종목만 실패(데이터는 있음)면 "일부 수집 실패" 경고를 뉴스레터 상단에 한 줄 표시하고 진행.

### STEP 2 — 종목별 분석 (AI — 판단 영역)
각 종목의 `items` 를 읽고:
1. **관련성 최종 판단** — 스크립트가 경계매칭으로 1차 거른 상태. 그래도 맥락상 무관하면 제외.
2. **호재/악재 태그** — 근거가 된 **기사 원문 문장 1개를 반드시 인용**한다. 인용 못 하면 태그 금지.
   - 🟢 호재 / 🔴 악재 / 🟡 복합(혼재) / ⚪ 중립(인사·MOU·단순동향, **또는 판단 애매할 때**)
   - 단정 금지. 단일 기사뿐이면 "호재로 해석되는 경향" 같은 강한 표현도 자제.
3. **이슈 요약** — 종목당 2~3줄. 사실 위주.
4. **신뢰도 표시** — item의 `unverified: true`(C급 단독)면 "(미확인)" 표기.
5. **뉴스 0건 종목** — 빼지 말고 "오늘 관련 뉴스 없음" 회색 카드.

컴플라이언스 표현: 금지("지금 사야"·"목표가 OO"), 허용("기사 기준 긍정적 요인으로 분류"). 호재/악재 = **뉴스 논조 분류**이지 투자 판단 아님.

### STEP 3 — 시장 요약 + 오늘의 PICK (AI)
- `macro[]` 근거로 **TL;DR 3줄**: ① 호재 요인 ② 악재 요인 ③ 이번 주 관전 포인트.
- **오늘의 PICK 1종목**: (뉴스 건수 × 호재/악재 강도)로 선정 + 이유 1줄.
  - 전 종목 ⚪/0건이면 PICK 생략하고 "오늘은 주목 종목 없음"으로.

### STEP 4 — HTML 렌더링 (AI)
모바일 반응형. 섹션 순서:
```
[헤더]   날짜 + 기간 + 한 줄 시황  →  ★헤더 바로 아래 면책 한 줄 고정
[PICK]   오늘 주목 종목 1개 (강조)  →  PICK 카드 안에 소형 면책("뉴스량 기준 선정, 투자 권유 아님")
[TL;DR]  반도체 시장 3줄
[종목 카드]  중요순(뉴스 많은 순→호재→악재→복합→중립→0건)
          카드: 종목명 | 🟢/🔴/🟡/⚪ | 뉴스 N건 / 이슈 2줄 / 원문 링크 1개
[아카이브]  ../../index.html
[푸터]   발행 시각 | 출처(네이버 뉴스) | 면책 문구 + 생성시각 HTML 주석
```
🟡 복합 카드 배경색도 정의한다(예: 앰버). 면책 문구는 **헤더·PICK·푸터 3곳**에 둔다(모바일에서 푸터까지 안 내려도 보이도록).
면책(고정): "본 뉴스레터는 공개 뉴스를 AI로 요약한 정보 제공용입니다. 투자 권유·매수/매도 추천이 아니며, 정확성을 보장하지 않습니다. 투자 판단과 손익은 본인에게 귀속됩니다."

### STEP 5 — 저장 + 누적 (멱등성 주의)
- 저장: `50-my-work/Part06-스킬만들기/실습27-검색API스킬/newsletter/{YYYY}/{MM}/{YYYYMMDD}-반도체.html` (폴더 없으면 `mkdir -p`).
- 누적: `newsletter/index.html`(root) 맨 위에 오늘치 링크 추가.
  - **upsert** — 같은 날짜 링크가 이미 있으면 추가하지 말고 교체(중복 링크 금지). 같은 날 재실행해도 목록은 한 줄만 유지.
- HTML 상단 주석에 `<!-- generated_at: ... -->` 를 넣어 재발행 버전을 추적.
- 저장 완료 후 사용자에게 **파일 경로 + "브라우저에서 열기" 안내 한 줄**(WSL이면 `explorer.exe` / `wslview`)을 출력한다.
- 전체 0건인 날도 "뉴스 없는 날" 템플릿으로 정상 발행(발송 중단 금지). 단 exit 1(수집 실패)은 발행 안 함.

## 발송 주기 + 휴장일 자동 판정
평일(월~금) 아침 기준. 월요일판이 금·토·일 포함.
휴장일(주말·법정공휴일·근로자의날·연말폐장)은 `fetch_news.py`의 `market_closed_info()`가 자동 판정한다.
- 법정공휴일은 `holidays` 라이브러리로 **음력 명절·대체공휴일·임시공휴일까지 미래연도 자동 계산** → 매년 손볼 필요 없음.
- 근로자의날(5/1)·연말폐장(12/31)은 법정공휴일이 아니지만 증시는 휴장 → 보충 규칙으로 처리.
- 라이브러리 미설치 시 주말+증시전용만 판정(`holiday_check=weekday-only-fallback`).

## 2차 확장 포인트 (전문가 토론 — 나중에)
- **DART 공시 연동**: 종목코드로 당일 공시 조회 → 뉴스 0건이어도 1차 출처 보완
- **전일 미국장**: NVDA·MU 등락률 한 줄 → 국내 시초가 선행지표
- **관리종목/거래정지 사전 체크**, 실적 캘린더 배지, RSS `feed.xml`
- **HTML 템플릿 분리**(assets/) — 면책·카드 구조 고정으로 재현성↑
- **자동 루틴 알림**: 무인 실행 중 `has_error`/exit 1이면 알림(조용한 실패 방지)
- 밸류체인 연동, TrendForce 현물가 파싱

## Scripts
- **`scripts/fetch_news.py`** — 수집·기간계산·휴장일판정·종목태깅(경계매칭)·정제·등급·매크로상한. watchlist.json + .env 로드.

## Config
- **`watchlist.json`** — 종목·매크로 키워드(코드 수정 없이 종목 변경).
- **`requirements.txt`** — `holidays`(공휴일 자동 판정). 미설치 시 graceful fallback.
