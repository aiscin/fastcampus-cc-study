# 실습29 — audio-to-doc (오디오/영상 → 문서) 스킬

> Part 06 · Clip 8 · 완료 2026-06-26

## 무엇을 만들었나

녹음 파일/유튜브 영상의 음성을 **전사 + 화자 분리**하고, 내용 유형을 판별해 **유형별 양식으로 요약 문서**를 만드는 스킬.

- 활성 스킬: `.claude/skills/audio-to-doc/`
- 포트폴리오 사본: `skill-source/`
- 호출: "회의록 만들어줘", "녹음 정리해줘", "유튜브 전사해줘" 등 자연어

## 메타 흐름 (그냥 "만들어줘" 대신)

1. **워크플로우 잡기** — "녹음을 텍스트로 전환·요약하려는데 어떻게 구성?" 으로 출발
2. **공식문서 검토** — ai.google.dev/gemini-api/docs/audio 확인 → 화자분리는 전용 파라미터 없이 프롬프트로, 20MB 기준 inline/Files API 분기, 1초=토큰 32개
3. **skillers-suda 4인 토론** — 기획/사용자/전문가/검수자 보완점 반영
4. **테스트·검증** — 실제 유튜브 클립으로 끝까지 실행

## 5 유형 자동 판별

| 유형 | 판별 신호 | 출력 양식 |
|------|----------|----------|
| 회의 | 다수 화자 + 안건·결정 키워드 | 안건/결정/액션아이템(체크박스) |
| 강의·컨퍼런스 | 단일 화자 + 개념 설명 | 핵심개념 + 섹션 요약 |
| 팟캐스트·대담 | 2-3 화자 + 자유 대화 | 주제별 구간 + 화자별 의견 |
| 인터뷰 | 질문자 ↔ 답변자 Q&A | Q&A + 발언 하이라이트 |
| 발표·세미나 | 단일 화자 + 주장-근거 | 주장-근거-Q&A |

## 스크립트 분리 (결정론은 코드로)

- `scripts/extract_audio.sh` — yt-dlp 음성 추출(URL) + ffmpeg 16kHz mono 변환, 로컬/URL 자동 분기
- `scripts/transcribe_gemini.py` — 20MB 기준 inline/Files API 분기, gemini-3.5-flash 전사+화자분리, 구조화 JSON + 원문 보존
- 유형 판별·요약은 SKILL.md 본문(AI)으로 → 자유도 확보

## 도구 선택 근거

- **Gemini gemini-3.5-flash** — 설치 0(REST), 화자 분리(프롬프트), 한국어 정확, 긴 영상(9.5h) 한 번에. 단점은 오디오가 Google로 전송(클라우드) → 대외비는 로컬 대안 권장.
- 모델명은 공식문서 + **라이브 API 모델 목록 조회**로 교차검증 (gemini-3.5-flash 실존 확인).

## 실제 시연 (검증)

- 입력: 유튜브 61초 시사 클립(거리의 만찬 공익제보자 몽타주)
- 결과: 전사 + MM:SS 타임스탬프 정상, 인터뷰 양식으로 요약 발행
- 한계 확인: 짧은 컷 다화자 몽타주는 화자 과분리(15명) — 깨끗한 단일 회의 녹음이 본래 타깃
- 산출물: `demo/transcript_raw.txt`, `demo/공익제보자-증언정리-20260626.md`
