# 레퍼런스 스캔 가이드 (Step 2)

후보를 내기 전에 "이미 비슷한 게 있나?"를 훑어, 바닥부터 만드는 낭비를 막고 차별점을 잡기 위한 단계다.

## 1. 스캔 범위

| 범위 | 정확한 glob | 기본 |
|------|--------|------|
| 로컬 스킬 | `.claude/skills/*/SKILL.md` | ✅ 항상 |
| 플러그인 스킬 | `~/.claude/plugins/cache/*/*/*/skills/*/SKILL.md` | ✅ 항상 |
| 기존 커맨드/에이전트 | `.claude/commands/*.md`, `.claude/agents/*.md` (역할 겹칠 때만) | 필요 시 |
| 웹 사례 | WebSearch 1회 — 외부 도구/접근법 | ⛔ 사용자가 "웹도 찾아줘" 할 때만 |

각 SKILL.md의 frontmatter(`name`, `description`)만 Read한다. **본문 전체는 읽지 않는다** — 컨텍스트 윈도우를 아끼기 위해서다.

> **플러그인 경로 주의**: 플러그인 스킬은 `cache/{마켓}/{플러그인}/{버전}/skills/{스킬}/` 구조라 중간에 **마켓·플러그인·버전 폴더 3겹**이 낀다. 그래서 `*/*/*`를 써야 걸린다. `cache/*/skills/...`처럼 얕게 잡으면 하나도 안 잡히니 주의. Glob 결과가 0건이면 깊이를 의심한다.

## 1-1. 자기 자신·코스 가이드 제외

스캔 결과에서 다음은 후보 비교 대상에서 뺀다 — 사용자 아이디어와 무관한 노이즈이기 때문이다:
- **ideation-funnel 자신** — 자기 스킬이 잡히는 건 당연하므로 무시.
- **코스 가이드 스킬**(`part0N-*` 등 강의 진행용) — 사용자가 만들 도구가 아니라 학습 안내용이므로 유사도 비교에서 제외.

## 2. 유사도 판단 기준

Step 1에서 좁힌 의도(목적·대상·해결할 불편)와 비교한다:

- **역할 중복**: 거의 같은 일을 하는 스킬이 이미 있다 → "재활용" 또는 "기존 걸 개선" 제안.
- **부분 겹침**: 일부 기능이 겹친다 → 그 부분은 기존 걸 호출하고, 새로운 건 차별점으로.
- **무관**: 겹치는 게 없다 → 새로 만들기.

## 3. 결과 정리 형식

사용자에게 한 묶음으로 보여준다:

```
🔎 레퍼런스 스캔 결과

이미 있는 비슷한 것:
- {스킬명} — {description 한 줄} → {재활용 / 차별점}

겹치지 않는 부분(= 새로 만들 가치):
- {차별 포인트 1-2개}

→ 결론: {재활용} / {차별점 잡아 새로 만들기} / {완전히 새로 만들기}
```

겹치는 게 없으면 솔직하게 "비슷한 건 없었어요 — 새로 만들 가치가 있어요"로 정리한다. 억지로 연결하지 않는다.

## 4. 외부 레퍼런스 딥리서치 (deep-research 방법론 적용)

로컬 스캔이 "이미 있나"라면, 이건 "잘되는 건 어떻게 하나"다. 검색 스니펫만 긁으면 근거 없는 레퍼런스가 되므로, 외부 검토는 `deep-research` 스킬의 방법론을 그대로 빌린다.

**적용 절차:**
1. **무엇을 리서치할지 정하기** — 의도에서 도출. 보통: 이 도메인에서 잘되는 사례·구조·템플릿 / 흔한 실패 / 필요한 도구. (예: "잘되는 정보형 여행 릴스의 훅·씬 구성")
2. **본문 fetch** — WebSearch로 후보 URL을 모으되, 유망 URL 2~4개는 **WebFetch로 본문을 직접 읽는다**. 스니펫에서 멈추지 않는다.
3. **교차검증 + 신뢰도 등급(A~E)** — 서로 다른 출처를 대조하고 충돌은 해소 시도. 주장·출처마다 등급:

   | 등급 | 의미 |
   |---|---|
   | A | 공식·1차 데이터 (플랫폼 공식 가이드 등) |
   | B | 신뢰 매체·플랫폼 본문 |
   | C | 일반 매체·블로그 |
   | D | 기업/홍보 글 |
   | E | 일화·SNS 후기 |

4. **fetch 예산** — 한 바퀴 2~4개, 가벼운 아이디어면 생략 가능("딥리서치 건너뛰기").

**산출물 저장 (resource/):**
- `resource/{주제}-research-{날짜}.md` — 보고서(무엇을 찾았나): 핵심 요약 + 표(신뢰도 열 포함) + 출처 목록(등급) + 한계
- `resource/{주제}-research-note-{날짜}.md` — 리서치 노트(과정): 검색 로그·채택/제외 근거·미해결
- `resource/reference-scan-{날짜}.md` — 로컬 스캔 + 외부 리서치를 합친 최종 정리본

draft에는 요약만 적고, 원자료는 resource/에 둔다 — 컨텍스트를 가볍게 유지하면서 근거는 보존하기 위해서다. 같은 주제를 다시 아이데이션하면 이전 resource/를 먼저 로드해 출발점으로 삼는다.
